# 设备参数显示空值问题修复报告

## 问题描述
点击设备的"编辑"按钮后，编辑设备页面上所有的设备参数全部显示为空，即使设备在数据库中有参数值。

## 根本原因

### 数据格式不匹配
数据库中存储的 `key_params` 格式与前端代码期望的格式不一致：

**数据库实际格式（简单格式）**:
```json
{
  "检测对象": "温度",
  "安装位置": "室内墙装",
  "温度信号类型": "NTC 10K",
  "温度量程": "-20~60℃"
}
```

**前端代码期望格式（嵌套格式）**:
```json
{
  "检测对象": {
    "value": "温度",
    "raw_value": "温度",
    "data_type": "string",
    "unit": null,
    "confidence": 1.0
  }
}
```

### 错误的访问方式
原来的 `getParamValue` 函数尝试访问 `formData.key_params[paramName].value`，但实际数据是 `formData.key_params[paramName]` 直接就是字符串值，导致返回 `undefined`。

## 修复方案

### 1. 改进 getParamValue 函数

支持两种数据格式的读取：

```javascript
const getParamValue = (paramName) => {
  if (!formData.key_params[paramName]) {
    return ''
  }
  
  // 处理两种数据格式：
  // 1. 简单格式: {参数名: "参数值"}
  // 2. 嵌套格式: {参数名: {value: "参数值", raw_value: ..., ...}}
  const param = formData.key_params[paramName]
  
  if (typeof param === 'string') {
    // 简单格式：直接返回字符串值
    return param
  } else if (typeof param === 'object' && param !== null) {
    // 嵌套格式：返回 value 字段
    return param.value || ''
  }
  
  return ''
}
```

**改进点**:
- 检查参数的类型
- 如果是字符串，直接返回
- 如果是对象，返回 `value` 字段
- 兼容两种格式

### 2. 改进 setParamValue 函数

支持两种数据格式的写入：

```javascript
const setParamValue = (paramName, value, param) => {
  if (!formData.key_params[paramName]) {
    // 如果参数不存在，使用简单格式初始化
    formData.key_params[paramName] = value
  } else {
    // 如果参数已存在，检查其格式
    const existingParam = formData.key_params[paramName]
    
    if (typeof existingParam === 'string') {
      // 简单格式：直接更新值
      formData.key_params[paramName] = value
    } else if (typeof existingParam === 'object' && existingParam !== null) {
      // 嵌套格式：更新 value 和 raw_value
      formData.key_params[paramName].value = value
      formData.key_params[paramName].raw_value = value
    } else {
      // 其他情况：使用简单格式
      formData.key_params[paramName] = value
    }
  }
}
```

**改进点**:
- 检查现有参数的格式
- 根据格式选择正确的更新方式
- 新参数使用简单格式
- 保持与数据库格式一致

### 3. 统一参数初始化格式

#### onDeviceTypeChange 函数
```javascript

  value: '',
  raw_value: '',
  data_type: param.data_type,
  unit: param.unit || null,
  confidence: 1.0

// 修改后（简单格式）

  formData.key_params[param.name] = {
    value: '',
    raw_value: '',
    data_type: param.data_type || 'string',
    unit: param.unit || null,
    confidence: 1.0
  }
}

// 修改后（简单格式）

  formData.key_params[param.name] = ''
}
```

## 数据格式说明

### 为什么使用简单格式？

1. **数据库存储效率**: 简单格式占用更少空间
2. **易于理解**: 直接的键值对，更直观
3. **兼容性**: 与现有数据库数据格式一致
4. **简化逻辑**: 减少数据转换的复杂度

### 简单格式 vs 嵌套格式

| 特性 | 简单格式 | 嵌套格式 |
|------|---------|---------|
| 存储大小 | 小 | 大 |
| 可读性 | 高 | 中 |
| 扩展性 | 低 | 高 |
| 复杂度 | 低 | 高 |
| 当前使用 | ✓ | ✗ |

### 何时需要嵌套格式？

如果将来需要以下功能，可以考虑迁移到嵌套格式：
- 参数置信度评分
- 参数来源追踪
- 参数验证状态
- 参数历史记录
