## 实现的功能

### 1. 白名单配置

**文件**：`data/static_config.json`

添加了 `whitelist_features` 字段：

```json
{
  "intelligent_extraction": {
    "feature_quality_scoring": {
      "enabled": true,
      "min_quality_score": 50,
      "whitelist_features": [
        "水",
        "气",
        "阀"
      ],
      "scoring_rules": {
        ...
      }
    }
  }
}
```

**说明**：
- 白名单中的特征会跳过质量评分检查
- 即使评分低于阈值也不会被过滤
- 默认包含：水、气、阀

### 2. 后端实现

**文件**：`backend/modules/text_preprocessor.py`

修改了 `_extract_features_with_detail()` 方法中的步骤5（过滤逻辑）：

```python
# 获取白名单配置
whitelist_features = quality_scoring_config.get('whitelist_features', [])

for feature in enhanced_features:
    # 检查是否在白名单中
    is_whitelisted = feature in whitelist_features
    
    # 计算质量评分
    quality_score = self._calculate_feature_quality(feature)
    
    # 检查过滤条件
    if quality_enabled and quality_score < min_quality_score and not is_whitelisted:
        # 白名单特征不受质量评分限制
        filter_reason = 'low_quality'
```

**关键改进**：
- 在过滤前检查特征是否在白名单中
- 白名单特征跳过质量评分检查
- 其他过滤条件（长度、重复等）仍然适用

## 白名单使用建议

### 建议添加到白名单的特征

1. **介质类型**（单字）：
   - 水、气、油、汽

2. **设备类型**（单字）：
   - 阀、泵、表、柜、箱

3. **其他重要的短词**：
   - 根据实际业务需求添加

### 不建议添加到白名单的特征

1. **量词**：个、台、套、只等
2. **无意义的单字**：的、了、是等
3. **纯数字**：1、2、3等
4. **纯标点**：、。！等

### 使用原则

1. **优先使用关键词列表**：
   - `medium_keywords`：介质关键词（+15分）
   - `device_type_keywords`：设备类型关键词（+15分）
   - `brand_keywords`：品牌关键词（+15分）

2. **白名单作为补充**：
   - 只有在关键词列表无法解决的情况下才使用白名单
   - 避免过度使用，保持白名单简洁

3. **定期审查**：
   - 定期检查白名单中的特征是否仍然需要
   - 移除不再需要的特征

## 技术细节

### 白名单保护机制

```python
# 伪代码
for feature in features:
    is_whitelisted = feature in whitelist
    quality_score = calculate_quality(feature)
    
    if is_whitelisted:
        # 白名单特征直接保留
        keep(feature)
    elif quality_score >= threshold:
        # 质量评分达标，保留
        keep(feature)
    else:
        # 质量评分不达标，过滤
        filter(feature)
```

### 质量评分机制

**"水"的评分**：
- 基础分：50
- 在设备关键词库中（medium_keywords）：+15
- 总分：65 ✓

**"阀"的评分**：
- 基础分：50
- 太短(<2)：-20
- 总分：30 ✗（需要白名单保护）

### 前端数据流

```

  ↓ 读取 fullConfig.intelligent_extraction.feature_quality_scoring.whitelist_features
  ↓ 用户修改白名单
  ↓ 更新 fullConfig
  ↓ emit('change')
ConfigManagementView
  ↓ 保存配置
Backend API
```
