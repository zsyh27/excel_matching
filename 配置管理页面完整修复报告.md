## 技术细节

### 组件设计原则

所有新创建的编辑器组件遵循统一的设计原则：

1. **Props**：
   - `modelValue`: 配置数据（支持 v-model）
   - `fullConfig`: 完整配置对象（可选，用于复杂场景）

2. **Emits**：
   - `update:modelValue`: 更新配置数据
   - `change`: 通知父组件配置已变更

3. **UI结构**：
   - 编辑器头部：标题 + 描述
   - 编辑器主体：配置表单
   - 信息提示：使用说明和示例

4. **样式规范**：
   - 统一的颜色方案
   - 一致的间距和布局
   - 响应式设计

### 配置数据结构

#### WhitelistEditor
```javascript
modelValue: string[]  // 白名单特征列表
// 示例：['水', '气', '阀', 'co', 'co2']
```

#### QualityScoreEditor
```javascript
modelValue: {
  enabled: boolean,
  min_length_chinese: number,
  min_length_english: number,
  threshold: number  // 0-1之间
}
```

#### UnitRemovalEditor
```javascript
modelValue: {
  enabled: boolean,
  units: string[]  // 单位列表（小写）
}
// 示例：{ enabled: true, units: ['ppm', 'ma', 'v'] }
```

#### MatchThresholdEditor
```javascript
modelValue: {
  value: number  // 匹配阈值（0-100分）
}
// 推荐值：5-10分
```
