## 三、配置生效机制

### 配置读取流程

```
1. 后端启动
    ↓
2. 加载配置（从数据库 configs 表）
    ↓
3. 初始化 RuleGenerator
    ↓
4. 读取 feature_weight_config
    ↓
5. 设置 self.brand_weight, self.device_type_weight 等
    ↓
6. 添加设备时调用 generate_rule()
    ↓
7. 调用 _assign_feature_weight()
    ↓
8. 使用配置值分配权重
    ↓
9. 生成规则并保存到数据库
```

### 配置优先级

```python
# 权重值的优先级（从高到低）
1. self.device_type_weight（从 feature_weight_config 读取）
2. strategy['device_type_weight']（从 feature_weight_strategy 读取）
3. 默认值（20.0）
```

- key_params权重：2.5（较低）

**修改后**：
- key_params权重：15.0（次高）
- 与设备类型同等重要

## 九、常见问题

### Q1：修改配置后需要重新生成规则吗？

**A**：是的。修改权重配置后，需要批量重新生成所有设备的规则，新权重才能生效。

**执行命令**：
```bash
cd backend
python simple_regenerate_rules.py
```

### Q2：如何验证权重配置是否生效？

**A**：可以通过以下方式验证：

1. 检查配置：
```bash
python check_weight_config.py
```

2. 添加测试设备，查看生成的规则权重

3. 上传Excel测试匹配，检查匹配得分

### Q3：如果效果不理想，如何调整权重？

**A**：可以通过以下方式调整：

1. 通过前端配置管理页面修改
2. 通过脚本修改数据库配置
3. 重新生成规则
4. 测试验证

### Q4：权重配置会影响已有的匹配结果吗？

**A**：不会。权重配置只影响：
- 新添加设备时生成的规则
- 批量重新生成的规则

已有的规则不会自动更新，需要手动重新生成。
