# 批量导入和规则权重修复说明

## 问题总结

1. **批量导入时没有自动生成规则**
   - 原因：后端服务使用的是旧代码，RuleGenerator初始化缺少preprocessor参数
   
2. **规则权重不正确**
   - 设备类型"电动调节阀"权重为5分（应该是20分）
   - 规格型号权重为3-5分（应该是15分）
   - 原因：后端服务使用的是旧的权重分配逻辑

3. **设备详情页面关键参数为空**
   - 需要检查前端如何显示key_params字段

## 需要执行的操作

### 步骤1：重启后端服务 ⚠️ 重要！

**必须重启后端服务才能加载新代码！**

```bash
# 停止当前运行的后端服务（Ctrl+C）
# 然后重新启动
cd backend
python app.py
```

### 步骤2：清理Python缓存（可选但推荐）

```bash
cd backend
# Windows PowerShell
Remove-Item -Path "modules/__pycache__" -Recurse -Force -ErrorAction SilentlyContinue
Remove-Item -Path "__pycache__" -Recurse -Force -ErrorAction SilentlyContinue

# 或者 Windows CMD
rmdir /s /q modules\__pycache__
rmdir /s /q __pycache__
```

### 步骤3：重新生成现有设备的规则

有两种方式：

**方式A：使用数据一致性检查（推荐）**
1. 在设备管理页面点击"数据一致性检查"按钮
2. 系统会自动为没有规则的设备生成规则
3. 新生成的规则将使用正确的权重

**方式B：使用脚本批量重新生成**
```bash
cd backend
python regenerate_imported_device_rules.py
```

### 步骤4：测试批量导入

1. 在设备管理页面点击"批量导入"
2. 上传Excel文件
3. 确保勾选"自动为导入的设备生成匹配规则"
4. 点击"确认导入"
5. 检查导入的设备是否有规则，权重是否正确
