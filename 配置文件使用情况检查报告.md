## 二、配置系统架构分析

### 当前架构

```
配置系统
├── 主配置系统 (需要迁移)
│   ├── static_config.json ⚠️ 
│   ├── 数据库 configs表 ✅
│   └── 用途: 文本预处理、匹配引擎等
│
└── 智能设备录入系统 (独立,不迁移)
    ├── device_params.yaml ✅
    └── 用途: 设备参数定义、解析规则
```

### 关键发现

#### 1. 主配置系统的问题 ⚠️
- **双重存储**: JSON + 数据库
- **启动依赖**: app.py启动时先读JSON,再读数据库
- **测试依赖**: 大量测试脚本直接读JSON

#### 2. 智能设备系统的设计 ✅
- **独立配置**: 使用YAML文件
- **明确职责**: 只负责设备参数定义
- **不冲突**: 与主配置系统分离

## 四、迁移计划

### 阶段1: 修改app.py启动逻辑 (核心)

**当前代码** (app.py 第68-72行):
```python
# 临时加载配置用于初始化预处理器
from modules.data_loader import ConfigManager
temp_config_manager = ConfigManager(Config.CONFIG_FILE)
config = temp_config_manager.get_config()
preprocessor = TextPreprocessor(config)
```

**修改为**:
```python
# 直接从数据库加载配置
# 注意: 此时data_loader还未初始化,需要特殊处理
from modules.database_loader import DatabaseLoader
from modules.database import DatabaseManager

# 初始化数据库管理器
db_manager = DatabaseManager(Config.DATABASE_URL)
temp_loader = DatabaseLoader(db_manager)

# 从数据库加载配置
config = temp_loader.load_config()
preprocessor = TextPreprocessor(config)
```
