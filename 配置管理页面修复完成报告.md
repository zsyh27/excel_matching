## 问题详情

### 问题 1：分隔符统一 vs 处理分隔符（重复页面）

**症状**：两个菜单项显示完全相同的页面

**根本原因**：
- 两个页面都使用 `SplitCharsEditor` 组件
- 两个页面都编辑 `feature_split_chars` 配置

**解决方案**：
- 创建新组件 `SeparatorMappingEditor.vue`
- "分隔符统一" 使用新组件，编辑 `text_cleaning.separator_mappings`
- "处理分隔符" 继续使用 `SplitCharsEditor`，编辑 `feature_split_chars`

**功能区别**：
- **分隔符统一**（文本清理阶段）：转换分隔符格式（`；` → `;`）
- **处理分隔符**（特征提取阶段）：使用分隔符拆分文本

### 问题 2：智能拆分页面无法打开

**错误信息**：
```
[Vue warn]: Invalid prop: type check failed for prop "modelValue". 
Expected Object, got Undefined
```

**根本原因**：
- 配置键映射错误：`smart-split` → `intelligent_extraction`（不存在）
- 正确映射：`smart-split` → `intelligent_splitting`

**解决方案**：
- 更新配置键映射
- 添加默认值处理，避免 `undefined` 错误

### 问题 3：单位删除页面无法打开

**错误信息**：同上

**根本原因**：
- 配置键映射错误：`unit-remove` → `metadata_keywords`
- 正确映射：`unit-remove` → `unit_removal`

**解决方案**：
- 更新配置键映射
- 添加默认值处理

### 问题 4：其他配置键映射错误

发现并修复了多个配置键映射错误：

| 菜单项 | 错误映射 | 正确映射 |
|--------|----------|----------|
| 元数据处理 | `metadata_keywords` | `text_cleaning` |
| 复杂参数分解 | `metadata_keywords` | `intelligent_extraction` |
| 质量评分 | `metadata_keywords` | `feature_quality_scoring` |
| 白名单 | `metadata_keywords` | `feature_whitelist` |
| 匹配阈值 | `metadata_keywords` | `match_threshold_config` |
