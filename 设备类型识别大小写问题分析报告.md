**报告时间**: 2026-03-11  
**问题发现**: 用户在设备类型模式中添加的前缀关键词区分大小写

### 2. 设备类型识别器的实现

查看 `backend/modules/intelligent_extraction/device_type_recognizer.py`：

```python
def recognize(self, text: str) -> DeviceTypeInfo:
    """
    识别设备类型
    
    Args:
        text: 输入文本
        
    Returns:
        DeviceTypeInfo: 设备类型信息
    """
    if not text or not text.strip():
        return DeviceTypeInfo(...)
    
    # 1. 精确匹配
    result = self._exact_match(text)
    if result and result.confidence >= 0.95:
        return result
    
    # 2. 模糊匹配
    result = self._fuzzy_match(text)
    if result and result.confidence >= 0.85:
        return result
    
    # 3. 关键词匹配
    result = self._keyword_match(text)
    if result and result.confidence >= 0.75:
        return result
    
    # 4. 类型推断
    result = self._type_inference(text)
    return result if result else DeviceTypeInfo(...)
```

**关键发现**：
- ❌ `recognize` 方法直接使用原始文本 `text`
- ❌ 没有调用任何文本预处理或归一化函数
- ❌ 没有应用 `unify_lowercase` 配置

## 🎯 问题根因

### 根本原因

**设备类型识别器没有应用全局配置中的文本预处理功能**

具体表现：
1. ❌ 没有调用 `TextPreprocessor.normalize_text()` 方法
2. ❌ 没有应用 `unify_lowercase` 配置
3. ❌ 直接使用原始文本进行匹配，导致区分大小写

### 设计缺陷

**设备类型识别器与文本预处理器分离**：
- 设备类型识别器 (`DeviceTypeRecognizer`) 是独立模块
- 文本预处理器 (`TextPreprocessor`) 是另一个独立模块
- 两者之间没有集成

**API处理器没有协调两者**：
- API处理器直接调用识别器
- 没有先调用预处理器进行文本归一化

## 🔧 解决方案

### 方案1：在识别器中集成文本预处理（推荐）

**优点**：
- ✅ 彻底解决问题
- ✅ 与匹配阶段保持一致
- ✅ 应用所有全局配置

**缺点**：
- ⚠️ 需要修改识别器初始化逻辑
- ⚠️ 需要传递完整配置

**实现方案**：

```python
class DeviceTypeRecognizer:
    """设备类型识别器"""
    
    def __init__(self, config: Dict[str, Any]):
        self.config = config
        self.device_types = config.get('device_types', [])
        self.prefix_keywords = config.get('prefix_keywords', {})
        self.main_types = config.get('main_types', {})
        
        # 新增：初始化文本预处理器
        self.preprocessor = TextPreprocessor(config)
        
        # 预编译正则模式
        self.patterns = self._build_patterns()
    
    def recognize(self, text: str) -> DeviceTypeInfo:
        """识别设备类型"""
        if not text or not text.strip():
            return DeviceTypeInfo(...)
        
        # 新增：应用文本归一化
        normalized_text = self.preprocessor.normalize_text(text, mode='matching')
        
        # 使用归一化后的文本进行匹配
        result = self._exact_match(normalized_text)
        if result and result.confidence >= 0.95:
            return result
        
        result = self._fuzzy_match(normalized_text)
        if result and result.confidence >= 0.85:
            return result
        
        result = self._keyword_match(normalized_text)
        if result and result.confidence >= 0.75:
            return result
        
        result = self._type_inference(normalized_text)
        return result if result else DeviceTypeInfo(...)
```

## ⚠️ 注意事项

### 1. 配置结构

确保传递给识别器的配置包含全局配置：

```python
config = {
    'intelligent_extraction': {
        'device_type_recognition': {
            'device_types': [...],
            'prefix_keywords': {...},
            'main_types': {...}
        }
    },
    'global_config': {
        'unify_lowercase': True,
        'remove_whitespace': True,
        'fullwidth_to_halfwidth': True
    }
}
```

### 2. 模式预编译

如果在 `_build_patterns()` 中预编译正则表达式，需要考虑：
- 是否需要对模式也进行归一化？
- 还是在匹配时对文本进行归一化？

**推荐**：在匹配时对文本进行归一化，保持模式不变。

### 3. 性能影响

每次调用 `recognize()` 都会进行文本归一化：
- 对于单次调用，性能影响可忽略
- 对于批量调用，可以考虑缓存归一化结果

### 4. 向后兼容性

修改后需要确保：
- 现有的测试用例仍然通过
- 现有的功能不受影响
- 配置格式保持兼容
