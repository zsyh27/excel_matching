## 技术说明

### 参数配置结构

```json
{
  "device_types": {
    "蝶阀": {
      "keywords": ["蝶阀", "butterfly valve"],
      "params": [
        {
          "name": "公称通径",
          "pattern": "DN\\s*([0-9]+)",
          "required": true,
          "data_type": "string",
          "unit": "mm"
        }
      ]
    }
  }
}
```

### 字段说明

- `keywords`: 设备类型的关键词列表，用于识别设备类型
- `params`: 参数列表
  - `name`: 参数名称
  - `pattern`: 正则表达式模式，用于从文本中提取参数值
  - `required`: 是否必填
  - `data_type`: 数据类型（string, number, range等）
  - `unit`: 单位

## 注意事项

1. **参数值可以为空**: 除了标记为必填的参数外，其他参数值都可以为空
2. **正则模式可选**: 如果参数没有固定格式，可以不设置正则模式（pattern: null）
3. **关键词匹配**: 系统会使用关键词来识别设备类型，建议添加多个常用的关键词
4. **配置更新**: 修改配置后，前端需要刷新页面才能看到最新配置
