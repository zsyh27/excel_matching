# 实时预览功能设计

## 1. 功能概述

实时预览功能是配置管理页面的核心功能之一，允许用户在配置规则后立即看到处理效果。该功能需要保留并根据新的五步业务流程进行更新。

## 2. 当前实时预览功能分析

### 2.1 当前展示内容

**预处理结果**：
- 原始文本
- 清理后文本
- 归一化文本
- 提取特征列表

**匹配结果**：
- 匹配状态（成功/失败）
- 匹配设备名称
- 匹配得分

### 2.2 当前问题

1. **流程不清晰**：只展示预处理和匹配两个阶段，无法看到中间步骤
2. **信息不足**：缺少置信度、评分明细、参数对比等关键信息
3. **调试困难**：无法看到每一步的处理结果，难以定位问题
4. **缺少上下文**：无法理解为什么匹配成功或失败

## 3. 新实时预览功能设计

### 3.1 设计原则

1. **流程可视化**：按五步流程清晰展示每一步的结果
2. **信息完整性**：提供置信度、评分明细、参数对比等详细信息
3. **易于调试**：显示处理日志和生成的正则表达式
4. **用户友好**：使用视觉指示器（图标、颜色）增强可读性

#### 3.4.2 图标使用

- ✓ 成功/匹配
- ✗ 失败/未匹配
- ~ 部分匹配
- 🎯 置信度
- 📋 设备类型
- 📊 参数
- 🏷️ 辅助信息
- 🎯 匹配
- 🖥️ 界面预览
- 🔧 调试
- 🥇 第一名
- 🥈 第二名
- 🥉 第三名

#### 3.4.3 排名图标

```javascript
function getRankIcon(index) {
  const icons = ['🥇', '🥈', '🥉', '4️⃣', '5️⃣'];
  return icons[index] || `${index + 1}️⃣`;
}
```
