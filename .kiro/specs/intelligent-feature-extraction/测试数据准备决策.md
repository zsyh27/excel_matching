# 测试数据准备决策

## 1. 当前状况

### 1.1 数据库现状

**设备库**：
- 总设备数：137个
- 设备类型分布：

**问题**：
- ❌ 设备类型单一（只有传感器）
- ❌ 缺少探测器、阀门、控制器等常见设备
- ❌ 设备数量偏少，无法充分验证系统

### 1.2 测试文件现状

**文件**：`data/(原始表格)建筑设备监控及能源管理报价清单(3).xlsx`

**内容**：

  * CO浓度探测器

- ❌ 无法充分验证系统功能
- ❌ 无法达到 >90% 匹配准确率目标

## 2. 三种方案对比

### 方案A：扩充设备库（推荐）

**目标**：
- 增加设备库到300+个设备
- 覆盖更多设备类型（探测器、阀门、控制器等）
- 保持 >90% 匹配准确率目标

**实施步骤**：
1. 从现有数据源提取设备（如果有）
2. 手动添加常见设备（200个）
3. 生成规则并验证

**时间投入**：2-3天

**准确率目标**：
- 设备类型识别：>95%
- 参数提取：>90%
- 匹配准确率：>90%
- Top-3准确率：>95%

**优点**：
- ✅ 保持高准确率目标
- ✅ 系统更实用，覆盖更多场景
- ✅ 为未来扩展打好基础
- ✅ 长期价值高

**缺点**：
- ❌ 需要2-3天数据准备工作
- ❌ 需要验证新增设备的正确性

### 方案B：使用现有数据

**目标**：
- 使用现有137个设备
- 调整匹配准确率目标为 >70%
- 重点验证核心功能

**实施步骤**：
1. 使用现有137个设备作为设备库
2. 创建测试用例（从现有设备中选择）
3. 调整验收标准

**时间投入**：0天（无需额外准备）

**准确率目标**：
- 设备类型识别：>85%
- 参数提取：>80%
- 匹配准确率：>70%
- Top-3准确率：>85%

**优点**：
- ✅ 无需额外数据准备
- ✅ 可以快速开始测试

**缺点**：
- ❌ 准确率目标降低
- ❌ 设备类型覆盖不全（只有传感器）
- ❌ 系统实用性受限
- ❌ 无法验证探测器、阀门等设备的匹配

### 方案C：混合方案

**目标**：
- 增加设备库到200个左右
- 匹配准确率目标 >80%
- 平衡数据准备工作和系统质量

**实施步骤**：
1. 保留现有137个设备
2. 手动添加60-80个常见设备（探测器、阀门等）
3. 生成规则并验证

**时间投入**：1-2天

**准确率目标**：
- 设备类型识别：>90%
- 参数提取：>85%
- 匹配准确率：>80%
- Top-3准确率：>90%

**优点**：
- ✅ 数据准备工作适中
- ✅ 设备类型覆盖较全
- ✅ 准确率目标合理

**缺点**：
- ❌ 仍需1-2天数据准备工作
- ❌ 准确率目标略有降低

## 4. 推荐方案及理由

### 推荐：方案A（扩充设备库）

**理由**：

1. **长期价值**
   - 扩充的设备库可以持续使用
   - 为未来功能扩展打好基础
   - 一次投入，长期受益

2. **系统质量**
   - 保持高准确率目标（>90%）
   - 确保系统实用性
   - 建立用户信任

3. **用户体验**
   - 覆盖更多设备类型
   - 满足实际使用需求
   - 减少用户选择时间

4. **投入产出比**
   - 虽然需要2-3天数据准备
   - 但可以获得高质量的系统
   - 避免后期返工

5. **风险控制**
   - 充分的测试数据
   - 降低上线风险
   - 提高用户满意度

### 脚本2：生成规则

```python
# scripts/generate_rules_for_devices.py

# 为所有设备生成匹配规则
devices = get_all_devices_from_database()

for device in devices:
    rule = generate_rule_from_device(device)
    save_rule_to_database(rule)
```

### 脚本3：验证数据

```python
# scripts/verify_devices_and_rules.py

# 验证设备和规则的正确性
devices = get_all_devices_from_database()
rules = get_all_rules_from_database()

# 检查每个设备是否有对应的规则
for device in devices:
    rule = find_rule_for_device(device)
    if not rule:
        print(f"警告：设备 {device['device_name']} 没有对应的规则")
    else:
        # 验证规则是否正确
        if not validate_rule(rule, device):
            print(f"警告：设备 {device['device_name']} 的规则不正确")
```
