# 智能特征提取和匹配系统需求文档

## 1. 项目概述

### 1.1 背景

当前系统在处理设备描述文本时存在以下问题：
- 过度依赖文本清理和预处理，配置复杂（17个配置页面）
- 特征提取不够智能，主要依赖分隔符拆分
- 对文本格式变化敏感，鲁棒性不足
- 维护成本高，需要频繁调整配置 

### 1.2 目标

构建一个智能的特征提取和匹配系统，实现：
- **设备类型识别准确率 >85%**（最重要！）
- **匹配准确率 >70%**
- 用户选择时间减少 70%（从 >2分钟 降至 <30秒）
- 配置页面减少 47%（从 17个 降至 9个）
- 维护成本降低 60%
- 用户无需编写正则表达式，通过可视化界面配置规则

**注**：由于当前设备库规模有限（137个设备，仅传感器类型），准确率目标相应调整。系统架构设计支持未来扩展到更高准确率。

### 1.3 核心设计原则

1. **设备类型识别优先**：即使参数不完全匹配，也要确保设备类型识别准确
2. **结构化信息提取**：提取结构化信息而非碎片化特征
3. **智能评分和排序**：多维度评分，按评分排序，默认选中最高分设备
4. **配置化方案**：用户通过界面配置，系统自动生成正则表达式

## 3. 功能需求

### 3.1 设备类型识别器

#### 3.1.1 主类型识别
- 识别基础设备类型：传感器、探测器、阀门、控制器、变送器等
- 支持多种识别模式：
  * 精确匹配：CO浓度探测器 → CO浓度探测器
  * 模糊匹配：CO探测器 → CO浓度探测器 (90%)
  * 关键词匹配：CO + 探测器 → CO浓度探测器 (80%)
  * 类型推断：CO + 浓度 → CO浓度探测器 (70%)

#### 3.1.2 子类型识别
- 结合前缀词和主类型识别子类型
- 支持组合模式：前缀 + 修饰词 + 类型
- 提取完整设备名称

#### 3.1.3 置信度评估
- 计算识别置信度（0-1）
- 根据匹配模式调整置信度
- 提供置信度解释

### 3.2 参数提取器

#### 3.2.1 量程提取
- 识别量程标签：量程、范围、测量范围
- 提取量程值：0~250ppm, -40~80℃, 0-10bar
- 归一化：提取起始值、结束值、单位

#### 3.2.2 输出信号提取
- 识别输出标签：输出、输出信号
- 提取信号值：4~20mA, 0~10V, RS485
- 归一化：标准化表达

#### 3.2.3 精度提取
- 识别精度标签：精度、准确度
- 提取精度值：±5%, ±0.5℃, ±1%FS
- 归一化：提取数值

#### 3.2.4 规格提取
- 识别规格代码：DN50, PN16, PT100, G1/2
- 保持原始格式

### 3.3 辅助信息提取器

#### 3.3.1 品牌提取
- 词库匹配：西门子、施耐德、ABB、霍尼韦尔等
- 支持品牌别名

#### 3.3.2 介质提取
- 词库匹配：水、气、油、蒸汽、冷媒等

#### 3.3.3 型号提取
- 模式匹配：字母+数字组合
- 识别常见型号格式


### 3.4 智能匹配器

#### 3.4.1 设备类型过滤
- 从设备库中筛选同类型设备
- 如果类型不确定，扩大筛选范围

#### 3.4.2 多维度评分
- **设备类型得分（50%）**：
  * 完全匹配：50分
  * 主类型匹配 + 关键词匹配：40-45分
  * 主类型匹配：30-35分
  * 相近类型：20-25分

- **参数得分（30%）**：
  * 量程完全匹配：15分
  * 量程范围包含：10分
  * 输出信号匹配：10分
  * 精度匹配：5分

- **品牌得分（10%）**：
  * 品牌一致：10分

- **其他得分（10%）**：
  * 介质匹配：5分
  * 型号匹配：5分

#### 3.4.3 参数模糊匹配
- 量程范围匹配：0-250ppm 可匹配 0-200ppm
- 数值容差：±5% 可匹配 ±3%
- 信号类型等价：4-20mA ≈ 0-10V

#### 3.4.4 多阶段匹配策略
- 严格匹配（90+分）
- 宽松匹配（70-89分）
- 模糊匹配（50-69分）
- 兜底匹配（30-49分）

#### 3.4.5 智能排序
- 按总分降序排列
- 同分情况下，设备类型匹配度高的优先
- 提供设备类型分组视图

### 3.5 规则生成器

#### 3.5.1 设备类型规则生成
- 根据配置的关键词生成正则表达式
- 支持前缀词 + 类型组合
- 支持多种组合模式
- 自动优化正则表达式

#### 3.5.2 参数规则生成
- 根据标签关键词和值模式生成正则
- 支持预定义值模式模板
- 自动处理特殊字符转义

#### 3.5.3 规则缓存
- 缓存生成的正则表达式
- 配置变更时自动更新缓存

### 3.6 配置管理

#### 3.6.1 设备类型模式配置
- 配置基础设备类型关键词
- 配置前缀词库（CO, CO2, 温度, 湿度等）
- 配置组合模式（前缀+类型、前缀+修饰词+类型等）
- 设置优先级

#### 3.6.2 参数提取模式配置
- 配置参数类型（量程、输出、精度、规格）
- 配置标签关键词
- 选择值模式模板
- 添加示例值

#### 3.6.3 辅助信息模式配置
- 配置品牌词库
- 配置介质词库
- 配置型号识别模式

#### 3.6.4 同义词映射配置

- 配置技术术语映射
- 支持单位归一化（℃=°C=度）
- 支持自动变体生成（未来增强）

#### 3.6.5 全局配置

  * 拆分复合词（室内墙装→室内+墙装）
  * 拆分技术规格（DN15→dn+15）
  * 按空格拆分

## 5. 配置保留分析结论

基于详细分析，我们确定了以下配置保留方案：

### 5.1 保留的配置（6个）

1. **品牌关键词**（BrandKeywordsEditor）
   - 用途：品牌识别
   - 位置：设备信息录入前配置
   - 原因：品牌识别是辅助信息提取的一部分

2. **设备参数配置**（DeviceParamsEditor）
   - 用途：参数定义
   - 位置：设备信息录入前配置
   - 原因：定义设备参数结构，核心配置

3. **特征权重**（FeatureWeightEditor）
   - 用途：评分权重
   - 位置：设备信息录入前配置
   - 原因：直接影响匹配结果

4. **设备行识别**（DeviceRowRecognitionEditor）
   - 用途：Excel导入
   - 位置：数据导入阶段
   - 原因：与特征提取无关的独立功能

5. **同义词映射**（SynonymMapEditor）⭐
   - 用途：词汇扩展
   - 位置：智能提取配置（新阶段）
   - 原因：提高匹配召回率的关键，无法完全自动化
   - 增强：添加自动变体生成功能

6. **全局配置**（GlobalConfigEditor）
   - 用途：全局设置
   - 位置：全局配置
   - 扩展：添加智能拆分、匹配阈值等选项

### 5.2 新增的配置（3个）

1. **设备类型模式**（DeviceTypePatternEditor）
   - 用途：设备类型识别规则
   - 位置：智能提取配置（新阶段）
   - 功能：配置设备类型关键词、前缀词库、组合模式

2. **参数提取模式**（ParameterPatternEditor）
   - 用途：技术参数提取规则
   - 位置：智能提取配置（新阶段）
   - 功能：配置量程、输出、精度、规格提取规则

3. **辅助信息模式**（AuxiliaryPatternEditor）
   - 用途：品牌/介质/型号提取规则
   - 位置：智能提取配置（新阶段）
   - 功能：配置辅助信息识别规则

### 5.3 删除的配置（8个）

1. **噪音过滤**（IgnoreKeywordsEditor）
   - 原因：智能提取自动处理，容易误删有用信息

2. **复杂参数分解**（ComplexParamEditor）
   - 原因：与智能拆分重复，智能提取器自动处理

3. **质量评分**（QualityScoreEditor）
   - 原因：规则复杂难配置，智能提取器内置质量评估

4. **白名单**（WhitelistEditor）
   - 原因：与质量评分耦合，智能提取器自动识别重要特征

5. **设备类型关键词**（DeviceTypeEditor）
   - 原因：简单的关键词列表不够智能，由设备类型模式替代

6. **智能拆分**（IntelligentCleaningEditor）
   - 原因：功能重复，移到全局配置

7. **单位删除**（UnitRemovalEditor）
   - 原因：可能导致信息丢失，参数提取器智能处理

8. **匹配阈值**（MatchThresholdEditor）
   - 原因：不需要单独页面，移到全局配置

### 5.4 简化的配置（3个）

1. **元数据处理**（MetadataRulesEditor）
   - 简化方案：系统内置常用标签（型号、品牌、规格等）
   - 迁移：常用标签内置，扩展标签移到全局配置

2. **归一化映射**（NormalizationEditor）
   - 简化方案：保留单位归一化
   - 迁移：单位归一化合并到同义词映射

3. **处理分隔符**（SplitCharsEditor）
   - 简化方案：保留智能拆分功能
   - 迁移：智能拆分选项移到全局配置
