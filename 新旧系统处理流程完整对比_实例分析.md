# 新旧系统处理流程完整对比 - 实例分析

## 测试输入

**原始文本**：
```
1.名称:CO浓度探测器2.规格参数：工作原理：电化学式； 量程 0~ 250ppm ；输出信号 4~20mA / 2~10VDC；精度±5%  @25C. 50% RH（0~100  ppm）3.施工要求:按照图纸、规范及清单要求配置并施工调试到位，并达到验收使用要求。
```

**场景**：用户从Excel导入此文本，系统需要自动匹配设备库中的设备

### 两套并行系统

| 系统 | 名称 | 用途 | API端点 |
|------|------|------|---------|
| **系统A（旧）** | MatchEngine | Excel批量匹配（生产环境） | `/api/excel/match` (line 556) |

**关键发现**：两套系统都在使用中，互不干扰！

## 旧系统（MatchEngine）处理流程

### 阶段0：文本预处理

**调用路径**：`app.py` → `MatchEngine.match()` → `TextPreprocessor.preprocess()`

**输入**：原始文本
```
1.名称:CO浓度探测器2.规格参数：工作原理：电化学式； 量程 0~ 250ppm ；输出信号 4~20mA / 2~10VDC；精度±5%  @25C. 50% RH（0~100  ppm）3.施工要求:按照图纸、规范及清单要求配置并施工调试到位，并达到验收使用要求。
```

**处理步骤**：

#### 步骤0.1：智能清理（已删除）
- ⚠️ **注意**：智能清理功能已在2026-03-12删除
- 现在直接跳过此步骤，`cleaned_text = original_text`

#### 步骤0.2：归一化处理

**层次1：精准映射**（normalization_map）
- 应用配置中的映射规则
- 例如：`"℃" → "c"`, `"：" → ":"`, `"～" → "~"`

**层次2：通用归一化**
- 全角转半角：`"０～２５０" → "0~250"`
- 删除空格：`"量程 0~ 250ppm" → "量程0~250ppm"`
- 统一小写：`"CO" → "co"`, `"VDC" → "vdc"`

**归一化结果**：
```
1.名称:co浓度探测器2.规格参数:工作原理:电化学式;量程0~250ppm;输出信号4~20ma/2~10vdc;精度±5%@25c.50%rh(0~100ppm)3.施工要求:按照图纸、规范及清单要求配置并施工调试到位，并达到验收使用要求。
```

#### 步骤0.3：特征提取

**拆分规则**：使用配置中的 `feature_split_chars`
- 分隔符：`;`, `,`, `、`, `\n`, `+` 等

**括号处理**：
- `"50%rh(0~100ppm)"` → 提取 `["50", "0~100"]`
- 删除单位后缀：`"250ppm"` → `"250"`, `"4~20ma"` → `"4~20"`

**元数据关键词处理**：
- 删除前缀：`"名称:co浓度探测器"` → `"co浓度探测器"`
- 删除前缀：`"量程0~250"` → `"0~250"`

**智能拆分**（如果特征长度>4）：
- `"co浓度探测器"` → `["co", "浓度", "探测器"]`

**提取的特征列表**：
```python
[
    "co",
    "浓度",
    "探测器",
    "电化学式",
    "0~250",
    "4~20",
    "2~10",
    "±5",
    "25",
    "50",
    "0~100"
]
```


**特征质量评分**（如果启用）：
- 对每个特征计算质量分数（0-100）
- 过滤低质量特征（分数<50）
- 白名单特征直接通过

**PreprocessResult 对象**：
```python
PreprocessResult(
    original="1.名称:CO浓度探测器...",
    cleaned="1.名称:CO浓度探测器...",  # 智能清理已删除
    normalized="1.名称:co浓度探测器2.规格参数:工作原理:电化学式;量程0~250ppm;...",
    features=["co", "浓度", "探测器", "电化学式", "0~250", "4~20", "2~10", "±5", "25", "50", "0~100"]
)
```

### 阶段1：规则匹配

**调用路径**：`MatchEngine.match()` → `calculate_weight_score()`

**输入**：特征列表
```python
features = ["co", "浓度", "探测器", "电化学式", "0~250", "4~20", "2~10", "±5", "25", "50", "0~100"]
```

**匹配算法**：

#### 步骤1.1：遍历所有规则

对每条规则（Rule）：
- 规则包含：`rule_id`, `target_device_id`, `auto_extracted_features`, `feature_weights`, `match_threshold`

#### 步骤1.2：计算权重得分

对于每个输入特征：
1. **直接匹配**：检查特征是否在规则的 `auto_extracted_features` 中
2. **同义词匹配**：检查特征的同义词是否在规则特征中
3. **累加权重**：如果匹配，累加对应的 `feature_weights` 值

**示例计算**（假设匹配到某个CO浓度探测器规则）：
```python
# 规则特征和权重
rule.auto_extracted_features = ["co", "浓度", "探测器", "0~250", "4~20", "电化学式"]
rule.feature_weights = {
    "co": 5.0,
    "浓度": 3.0,
    "探测器": 10.0,  # 设备类型，权重高
    "0~250": 8.0,
    "4~20": 8.0,
    "电化学式": 5.0
}

# 匹配计算
weight_score = 5.0 + 3.0 + 10.0 + 8.0 + 8.0 + 5.0 = 39.0
matched_features = ["co", "浓度", "探测器", "0~250", "4~20", "电化学式"]
```

#### 步骤1.3：阈值判断

**第一轮匹配**：使用规则自己的 `match_threshold`
- 如果 `weight_score >= rule.match_threshold`，加入候选列表

**第二轮匹配**（兜底）：使用 `default_match_threshold`（默认5.0）
- 如果第一轮没有候选，使用默认阈值再次判定

#### 步骤1.4：选择最佳匹配

从候选列表中选择权重得分最高的规则：
```python
best_candidate = max(candidates, key=lambda c: c.weight_score)
```

### 步骤1：设备类型识别

**调用路径**：`DeviceTypeRecognizer.recognize(text)`

**输入**：**原始文本**（不是归一化文本！）
```
1.名称:CO浓度探测器2.规格参数：工作原理：电化学式； 量程 0~ 250ppm ；输出信号 4~20mA / 2~10VDC；精度±5%  @25C. 50% RH（0~100  ppm）3.施工要求:按照图纸、规范及清单要求配置并施工调试到位，并达到验收使用要求。
```

**识别模式**：

#### 模式1：精确匹配（置信度100%）
- 完全匹配设备类型名称
- 例如：文本中包含"CO浓度探测器"

#### 模式2：模糊匹配（置信度90%）
- 部分匹配设备类型
- 例如：文本中包含"CO探测器"

#### 模式3：关键词匹配（置信度80%）
- 关键词组合匹配
- 例如：文本中包含"CO" + "探测器"

#### 模式4：类型推断（置信度70%）
- 根据关键词推断类型
- 例如：文本中包含"浓度" + "探测器"

**识别结果**：
```python
DeviceTypeInfo(
    main_type="探测器",
    sub_type="CO浓度探测器",
    keywords=["co", "浓度", "探测器"],
    confidence=1.0,  # 100%
    mode="exact"  # 精确匹配
)
```

#### 提取输出信号

pattern = r'输出[信号]*[：:]\s*([\d~\-/\s]+[a-zA-Z]+)'

#### 提取精度

pattern = r'精度[：:]*\s*[±]?(\d+)%'

# 从@符号后提取温度和湿度条件

### 步骤4：智能匹配和评分

**调用路径**：`IntelligentMatcher.match(extraction, top_k=5)`

**输入**：ExtractionResult（包含步骤1-3的结果）

**评分算法**：

#### 总分 = 设备类型(50%) + 参数(30%) + 品牌(10%) + 其他(10%)

**设备类型评分**（最高50分）：
- 完全匹配：50分
- 主类型匹配：30-45分
- 不匹配：0分

**参数评分**（最高30分）：
- 量程匹配：15分
- 输出匹配：10分
- 精度匹配：5分

**品牌评分**（最高10分）：
- 品牌一致：10分
- 品牌不一致：0分

**其他评分**（最高10分）：
- 介质匹配：5分
- 型号匹配：5分

**示例计算**（假设匹配到某个CO浓度探测器）：
```python
# 候选设备1
device = {
    "device_id": "HON_CO_001",
    "device_type": "CO浓度探测器",  # 完全匹配
    "range": "0~250ppm",  # 匹配
    "output": "4~20mA",  # 匹配
    "accuracy": "±5%",  # 匹配
    "brand": "霍尼韦尔"  # 未提取到品牌，无法匹配
}

# 评分
device_type_score = 50  # 完全匹配
range_score = 15  # 量程匹配
output_score = 10  # 输出匹配
accuracy_score = 5  # 精度匹配
brand_score = 0  # 品牌未知
total_score = 80  # 总分
```

## 关键差异对比

| 维度 | 旧系统（MatchEngine） | 新系统（IntelligentExtractionAPI） |
|------|---------------------|--------------------------------|
| **输入处理** | 预处理 → 特征提取 → 匹配 | 原始文本 → 直接提取 → 评分 |
| **特征来源** | 归一化文本拆分 | 正则表达式直接提取 |
| **匹配方式** | 特征权重累加 | 多维度评分 |
| **设备类型** | 作为特征之一（权重10-20） | 独立识别（权重50%） |
| **参数提取** | 作为特征（权重1-15） | 结构化提取（权重30%） |
| **评分透明度** | 权重得分 | 分项得分 + 总分 |
| **结果数量** | 1个（最佳匹配） | Top-K个（默认5个） |

## 总结

### 旧系统特点
- ✅ 简单直接，基于特征权重匹配
- ✅ 性能高效，适合批量处理
- ❌ 评分不透明，难以理解匹配原因
- ❌ 只返回一个结果，缺少备选方案

### 新系统特点
- ✅ 结构化提取，信息更完整
- ✅ 多维度评分，结果更透明
- ✅ 返回多个候选，用户可选择
- ✅ 六步流程可视化，便于调试
- ❌ 处理流程更复杂
- ❌ 性能开销略高

### 两套系统并存的原因
- 旧系统：用于生产环境的Excel批量匹配（稳定、高效）
- 新系统：用于智能提取和预览（功能丰富、透明度高）
