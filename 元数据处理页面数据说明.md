# 元数据处理页面数据说明

## 问题描述
用户反馈：元数据处理页面中原来的数据不显示了，是否在菜单迁移过程中删除了？

## 调查结果

### 1. 数据检查
检查了所有配置备份文件（从 2026-02-27 到 2026-03-06），发现：
- **所有备份中 `text_cleaning.metadata_rules` 都是空数组 `[]`**
- 没有发现任何包含元数据规则数据的备份文件

### 2. 配置路径
- 配置文件路径：`data/static_config.json`
- 数据结构：`config.text_cleaning.metadata_rules`
- 当前值：`[]`（空数组）

### 3. 前端映射
- 菜单ID：`metadata`
- 配置键：`text_cleaning`
- 编辑器组件：`MetadataRulesEditor.vue`
- 数据绑定：正常，已修复兼容性问题

## 结论

**原来就没有数据**，不是在菜单迁移过程中删除的。

可能的情况：
1. 用户从未添加过元数据规则
2. 数据在更早的时间点（2026-02-27之前）就已经丢失
3. 用户可能记错了，或者混淆了其他配置项

## 修复内容

### 前端代码优化
修改了 `ConfigManagementView.vue` 中的 `getEditorValue` 函数，增强了对元数据配置的处理：

```javascript
// 处理 text_cleaning.metadata_rules 的嵌套结构（元数据处理）
if (menuId === 'metadata') {
  // 如果 value 是 text_cleaning 对象
  if (value && typeof value === 'object' && 'metadata_rules' in value) {
    return value.metadata_rules || []
  }
  // 如果 value 直接是数组（兼容旧格式）
  if (Array.isArray(value)) {
    return value
  }
  // 默认返回空数组
  return []
}
```

这个修复确保了：
1. 正确读取 `text_cleaning.metadata_rules` 的数据
2. 兼容旧格式（如果直接是数组）
3. 默认返回空数组，避免 undefined 错误
