## 问题1: 智能提取API设备格式错误

### 问题描述
在配置管理页面的"设备类型模式"中添加新设备类型并进行实时测试时，出现错误：
```
ERROR: 'str' object has no attribute 'get'
```

### 根本原因
- `DataLoader.get_all_devices()` 返回 `Dict[str, Device]` 格式（device_id -> Device对象）
- `IntelligentMatcher` 期望 `List[Dict]` 格式（字典列表）
- 在兜底匹配阶段，直接对 Device 对象调用 `.get()` 方法导致错误

## 解决方案

### 修改的文件
- `backend/modules/intelligent_extraction/intelligent_matcher.py`

### 添加的方法
1. `_get_all_devices_as_list()`：统一获取设备列表
2. `_convert_devices_to_list()`：设备格式转换（支持多种输入格式）

### 更新的方法
1. `_filter_by_device_type()`：使用格式转换
2. `_filter_by_main_type()`：使用格式转换
3. `_fallback_match()`：使用统一的设备获取方法

## 问题2: 设备类型配置保存后消失

### 问题描述
用户在"设备类型模式"页面添加新设备类型后，刷新页面发现之前添加的设备类型消失了。

### 根本原因
配置结构不匹配：
- **前端期望**：`intelligent_extraction.device_type_recognition`
- **数据库实际**：`intelligent_extraction` 配置中没有 `device_type_recognition` 子键
- **结果**：前端加载时返回空对象，用户添加的数据无法持久化

### 问题流程
```
用户添加设备类型
    ↓
前端保存到 intelligent_extraction.device_type_recognition
    ↓
保存成功（创建了新的子键）
    ↓
用户刷新页面
    ↓
前端加载：deviceTypeConfig = ieConfig.device_type_recognition || {}
    ↓
配置路径不存在，返回空对象 {}
    ↓
设备类型列表为空，数据消失 ❌
```

## 解决方案

### 修复脚本
创建并运行 `fix_device_type_config_structure.py`，在数据库中初始化 `device_type_recognition` 配置结构。

### 初始化的配置
```json
{
  "device_types": [
    "传感器", "探测器", "变送器", "控制器",
    "执行器", "阀门", "流量计", "能量计"
  ],
  "prefix_keywords": {
    "温度": ["传感器", "探测器", "变送器"],
    "湿度": ["传感器", "探测器", "变送器"],
    // ... 11 个前缀关键词
  },
  "main_types": {
    "传感器": ["温度传感器", "湿度传感器", ...],
    "探测器": ["CO探测器", "CO2探测器", ...],
    // ... 8 个主类型
  }
}
```

### 执行修复
```bash
python fix_device_type_config_structure.py
```
