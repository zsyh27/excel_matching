## 一、后端API路由注册

### 1.1 修改的文件

**文件**: `backend/app.py`

**添加的内容**:

1. **智能提取API初始化函数**
   ```python
   def init_intelligent_extraction_api():
       """初始化智能提取API"""
       global intelligent_extraction_api
       try:
           from modules.intelligent_extraction.api_handler import IntelligentExtractionAPI
           config = data_loader.load_config()
           intelligent_extraction_api = IntelligentExtractionAPI(config, data_loader)
           logger.info("智能提取API初始化成功")
       except Exception as e:
           logger.error(f"智能提取API初始化失败: {e}")
   ```

2. **三个API端点**:
   - `/api/intelligent-extraction/extract` - 提取设备信息
   - `/api/intelligent-extraction/match` - 智能匹配设备
   - `/api/intelligent-extraction/preview` - 五步流程预览

### 1.2 API功能说明

#### API 1: 提取设备信息

**端点**: `POST /api/intelligent-extraction/extract`

**请求示例**:
```json
{
  "text": "CO浓度探测器 量程0~250ppm 输出4~20mA 精度±5%"
}
```

**响应示例**:
```json
{
  "success": true,
  "data": {
    "device_type": {
      "main_type": "探测器",
      "sub_type": "CO浓度探测器",
      "confidence": 1.0,
      "mode": "exact"
    },
    "parameters": {
      "range": "0~250ppm",
      "output": "4~20mA",
      "accuracy": "±5%"
    },
    "auxiliary": {
      "brand": null,
      "medium": null,
      "model": null
    }
  }
}
```

#### API 2: 智能匹配设备

**端点**: `POST /api/intelligent-extraction/match`

**请求示例**:
```json
{
  "text": "温度传感器 量程-20~60℃",
  "top_k": 5
}
```

**响应**: 返回前5个匹配的候选设备及评分

#### API 3: 五步流程预览

**端点**: `POST /api/intelligent-extraction/preview`

**请求示例**:
```json
{
  "text": "室内温度传感器 量程-20~60℃"
}
```

**响应**: 返回五步流程的完整信息和调试数据

## 三、前端对话框组件

### 3.1 设备类型对话框

**文件**: `frontend/src/components/ConfigManagement/DeviceTypeDialog.vue`

**功能**:
- 添加新的设备类型
- 编辑现有设备类型
- 表单验证（必填、长度、重复检查）

**字段**:
- 类型名称（必填，2-50字符，不可重复）
- 描述（可选）

**特性**:
- 创建后设备类型名称不可修改
- 自动检查设备类型是否已存在
- 表单验证和错误提示

### 3.2 前缀关键词对话框

**文件**: `frontend/src/components/ConfigManagement/PrefixKeywordDialog.vue`

**功能**:
- 添加新的前缀关键词
- 编辑现有前缀关键词
- 关联多个设备类型

**字段**:
- 前缀词（必填，1-20字符，不可重复）
- 关联设备类型（必填，多选）

**特性**:
- 创建后前缀词不可修改
- 自动检查前缀词是否已存在
- 支持多选设备类型
- 表单验证和错误提示

**现状态**:
- 点击"添加前缀词"按钮打开对话框
- 输入前缀词和选择关联设备类型
- 自动验证是否重复
- 保存后更新列表

#### 功能4: 编辑前缀关键词

**原状态**: 显示"编辑前缀关键词功能待实现"

**现状态**:
- 点击"编辑"按钮打开对话框
- 预填充当前前缀词和关联设备类型
- 可以修改关联设备类型（前缀词不可修改）
- 保存后更新列表

#### 功能5: 实时测试

**原状态**: 显示"测试功能待实现"

**现状态**:
- 输入设备描述文本
- 点击"测试"按钮或按Enter键
- 调用 `/api/intelligent-extraction/preview` API
- 显示识别结果：
  - 主类型
  - 子类型
  - 置信度（带颜色标签：绿色≥90%，黄色≥70%，红色<70%）
  - 匹配模式

### 4.3 新增的计算属性

1. `existingDeviceTypes` - 现有设备类型列表（用于重复检查）
2. `existingPrefixes` - 现有前缀词列表（用于重复检查）

### 4.4 新增的方法

1. `handleDeviceTypeConfirm` - 处理设备类型对话框确认
2. `handlePrefixKeywordConfirm` - 处理前缀关键词对话框确认
