## 常见问题

### Q1: 为什么配置数量是12个而不是9个？

A: 阶段1完成了废弃配置的删除（8个），还有3个配置待简化合并（阶段2工作）：
- 元数据处理 → 合并到全局配置
- 归一化映射 → 合并到同义词映射
- 处理分隔符 → 合并到全局配置

### Q2: 同义词映射为什么移动了位置？

A: 根据需求文档，同义词映射应该在"智能提取配置"阶段，而不是"匹配配置阶段"。这样更符合业务流程。

### Q3: 废弃的配置文件还在吗？

A: 是的，废弃的组件文件仍然保留在代码库中，但已被注释掉不会被使用。这样做是为了：
- 便于回退
- 保留历史记录
- 方便调试

### Q4: 如何验证系统正常工作？

A: 运行测试脚本：
```bash
python test_runtime_functionality.py
```
