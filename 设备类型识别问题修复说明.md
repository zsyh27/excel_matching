# 设备类型识别问题修复说明

## 问题描述

**用户反馈**：添加新设备类型"压力变送器"后，实时测试显示：
- 主类型：未识别
- 子类型：未识别
- 置信度：0.0%
- 匹配模式：none

**测试输入**：`压力变送器 0-10ma`

## 问题分析

### 配置检查结果

运行 `test_device_type_recognition.py` 发现配置是完整的：

```
✅ '压力变送器' 在设备类型列表中
✅ '压力' 前缀映射到 ['传感器', '探测器', '变送器']
✅ '变送器' 主类型包含 '压力变送器' 子类型
```

### 根本原因

**配置没有重新加载到智能提取API**

1. `intelligent_extraction_api` 在应用启动时初始化
2. 初始化时加载的是启动时的配置
3. 保存新配置后，`save_config` 接口重新加载了其他组件：
   - ✅ `config`
   - ✅ `preprocessor`
   - ✅ `match_engine`
   - ✅ `device_row_classifier`
   - ❌ **但没有重新加载 `intelligent_extraction_api`**

4. 因此实时测试使用的仍然是旧配置，无法识别新添加的设备类型

## 解决方案

### 方案1：修复代码（已完成）✅

修改 `backend/app.py` 中的 `save_config` 接口，添加智能提取API的重新加载：

```python
if success:
    # 重新加载配置和组件
    global config, preprocessor, match_engine, device_row_classifier, intelligent_extraction_api
    config = data_loader.load_config()
    preprocessor = TextPreprocessor(config)
    data_loader.preprocessor = preprocessor
    match_engine = MatchEngine(rules=rules, devices=devices, config=config)
    device_row_classifier = DeviceRowClassifier(config)
    
    # 重新初始化智能提取API（重要！）
    try:
        from modules.intelligent_extraction.api_handler import IntelligentExtractionAPI
        intelligent_extraction_api = IntelligentExtractionAPI(config, data_loader)
        logger.info("智能提取API已重新加载")
    except Exception as api_error:
        logger.error(f"智能提取API重新加载失败: {api_error}")
    
    return jsonify({'success': True, 'message': message}), 200
```

### 设备类型识别逻辑

智能提取API使用以下配置进行设备类型识别：

1. **device_types**：设备类型列表
   ```json
   ["传感器", "探测器", "变送器", "控制器", "执行器", "阀门", "流量计", "能量计", "压力变送器"]
   ```

2. **prefix_keywords**：前缀关键词映射
   ```json
   {
     "压力": ["传感器", "探测器", "变送器"],
     "温度": ["传感器", "探测器", "变送器"],
     ...
   }
   ```

3. **main_types**：主类型和子类型映射
   ```json
   {
     "变送器": ["温度变送器", "湿度变送器", "压力变送器", "压差变送器"],
     "传感器": ["温度传感器", "湿度传感器", "压力传感器", "压差传感器"],
     ...
   }
   ```

识别流程：
1. 检查输入文本是否包含 `device_types` 中的任何类型
2. 检查输入文本是否包含 `prefix_keywords` 中的前缀
3. 根据前缀和主类型，确定子类型
4. 计算置信度和匹配模式
