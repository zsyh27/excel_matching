# 设备库管理页面使用指南

## 概述

设备库管理页面提供了完整的设备CRUD（创建、读取、更新、删除）功能，支持动态表单、智能录入和设备类型管理。

## 功能特性

### 1. 设备列表查看

设备列表表格显示以下信息：
- **设备ID**: 唯一标识符
- **品牌**: 设备品牌
- **设备类型**: 设备分类（新增字段）
- **设备名称**: 设备的名称
- **规格型号**: 设备的规格型号
- **单价**: 设备不含税单价
- **是否有规则**: 显示该设备是否已生成匹配规则

### 2. 搜索和筛选

#### 关键词搜索
在搜索框中输入关键词，可以搜索：
- 设备ID
- 品牌
- 设备名称
- 规格型号

#### 高级筛选
- **品牌筛选**: 下拉选择特定品牌
- **设备类型筛选**: 下拉选择特定设备类型（新增）
- **价格范围**: 设置最低价格和最高价格

### 3. 添加新设备

点击"添加设备"按钮，打开设备表单对话框。

#### 基础信息（必填）
1. **设备ID**: 输入唯一的设备标识符
2. **品牌**: 输入设备品牌
3. **设备类型**: 从下拉列表选择设备类型（重要！）
4. **设备名称**: 输入设备名称
5. **规格型号**: 输入规格型号
6. **单价**: 输入不含税单价

#### 动态参数表单（根据设备类型自动显示）

当选择设备类型后，表单会自动显示该类型设备的特定参数字段。

**示例：选择"水泵"类型**
- 流量 (单位: m³/h)
- 扬程 (单位: m)
- 功率 (单位: kW)
- 转速 (单位: r/min)

**示例：选择"风机"类型**
- 风量 (单位: m³/h)
- 风压 (单位: Pa)
- 功率 (单位: kW)
- 转速 (单位: r/min)

**示例：选择"阀门"类型**
- 公称直径 (单位: mm)
- 公称压力 (单位: MPa)
- 阀体材质
- 密封材质

#### 可选信息
- **详细参数**: 可以补充其他特殊参数
- **自动生成规则**: 勾选后会自动为该设备生成匹配规则

### 4. 编辑设备

点击设备行的"编辑"按钮，可以修改设备信息。

**注意事项**：
- 设备ID不可修改（唯一标识符）
- 修改设备类型会清空之前填写的参数
- 可以选择是否重新生成匹配规则

### 5. 删除设备

点击设备行的"删除"按钮，会弹出确认对话框。

**警告**：
- 删除设备会同时删除关联的匹配规则
- 此操作不可撤销

### 6. 批量导入

点击"批量导入"按钮，可以从Excel文件批量导入设备。

### 7. 数据一致性检查

点击"数据一致性检查"按钮，系统会检查：
- 设备数据完整性
- 规则关联完整性
- 数据格式正确性

## 设备类型配置

系统支持的设备类型由后端配置文件定义（`backend/config/device_params.yaml`）。

### 当前支持的设备类型

根据配置文件，系统支持以下设备类型（示例）：
- 水泵
- 风机
- 阀门
- 冷水机组
- 空调机组
- 新风机组
- 热交换器
- 水箱
- 控制器
- 传感器
- 执行器

### 添加新设备类型

如需添加新的设备类型，需要：
1. 编辑 `backend/config/device_params.yaml`
2. 定义设备类型名称和参数配置
3. 重启后端服务

## 常见问题

### Q1: 为什么看不到设备类型字段？

**A**: 设备类型字段已经在最新版本中添加。如果看不到，请：
1. 刷新浏览器页面（Ctrl+F5 强制刷新）
2. 清除浏览器缓存
3. 确认前端代码已更新

### Q2: 设备类型下拉列表为空怎么办？

**A**: 可能的原因：
1. 后端配置文件 `device_params.yaml` 未正确配置
2. 后端服务未正常启动
3. 前端无法连接到后端API

解决方法：
1. 检查后端日志
2. 访问 `http://localhost:5000/api/device-types` 查看API是否正常
3. 确认配置文件路径正确

### Q3: 修改设备类型后参数丢失了？

**A**: 这是正常行为。不同设备类型有不同的参数配置，切换类型时会清空之前的参数，避免数据混乱。

### Q4: 如何查看设备的完整信息？

**A**: 点击设备行的"查看"按钮，可以查看设备的所有详细信息，包括：
- 基础信息
- 关键参数
- 原始描述
- 置信度评分
- 录入方式
- 创建和更新时间

### Q5: 设备类型筛选不工作？

**A**: 确保：
1. 后端API已更新（支持 `device_type` 查询参数）
2. 数据库中的设备有 `device_type` 字段
3. 前端代码已更新

## 技术说明

### 前端实现
- **组件**: `frontend/src/components/DeviceManagement/DeviceList.vue`
- **表单**: `frontend/src/components/DeviceManagement/DeviceForm.vue`
- **API**: `frontend/src/api/database.js`

### 后端实现
- **路由**: `backend/app.py` - `/api/devices`
- **模型**: `backend/modules/data_loader.py` - `Device` 类
- **配置**: `backend/config/device_params.yaml`

### 数据库字段
```sql
CREATE TABLE devices (
    device_id TEXT PRIMARY KEY,
    brand TEXT NOT NULL,
    device_type TEXT,           -- 新增字段
    device_name TEXT NOT NULL,
    spec_model TEXT NOT NULL,
    key_params TEXT,            -- JSON格式
    detailed_params TEXT,
    unit_price REAL NOT NULL,
    raw_description TEXT,
    confidence_score REAL,
    input_method TEXT DEFAULT 'manual',
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);
```
