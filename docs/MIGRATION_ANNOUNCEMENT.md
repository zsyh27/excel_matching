# 规则管理功能迁移公告

## 发布日期
2026年3月4日

## 概述

我们很高兴地宣布，规则管理功能已经完成重构并成功迁移！新版本将规则管理功能更好地集成到设备管理和统计仪表板中，为您提供更流畅、更高效的使用体验。

## 主要变更

### 🎯 功能位置调整

**规则管理功能已从独立页面迁移到：**

1. **设备管理页面** (`/device-management`)
   - 在设备列表中直接查看规则摘要
   - 在设备详情页面管理规则
   - 更直观的规则编辑体验

2. **统计仪表板** (`/statistics`)
   - 匹配日志查看和分析
   - 规则统计数据展示
   - 匹配成功率趋势图

### ✨ 新功能亮点

1. **规则摘要显示**
   - 设备列表中直接显示规则状态
   - 一目了然地查看特征数量、匹配阈值和总权重
   - 快速筛选有规则/无规则的设备

2. **上下文化的规则编辑**
   - 在设备详情页面直接编辑规则
   - 实时查看匹配难度评估
   - 智能的规则建议

3. **统一的统计视图**
   - 所有统计数据集中在一个仪表板
   - 标签页切换，操作更便捷
   - 支持数据导出和刷新

4. **性能大幅提升**
   - 查询响应时间提升90%+
   - 使用智能缓存机制
   - 更流畅的用户体验

### 🔄 URL重定向

为了保持向后兼容，旧的URL会自动重定向到新位置：

| 旧URL | 新URL |
|-------|-------|
| `/rule-management` | `/device-management` |
| `/rule-management/logs` | `/statistics?tab=logs` |
| `/rule-management/statistics` | `/statistics?tab=rules` |

**建议：** 请更新您的书签到新的URL。

### 📊 API变更

#### 新增API端点

```
# 设备规则管理
GET  /api/devices              # 包含规则摘要
GET  /api/devices/{id}         # 包含完整规则信息
PUT  /api/devices/{id}/rule    # 更新设备规则
POST /api/devices/{id}/rule/regenerate  # 重新生成规则

# 统计数据
GET /api/statistics/match-logs           # 匹配日志
GET /api/statistics/rules                # 规则统计
GET /api/statistics/match-success-rate   # 匹配成功率
```

#### 废弃API端点

以下API端点将在 **2026年6月4日** 停止支持：

```
GET  /api/rules
GET  /api/rules/{id}
PUT  /api/rules/{id}
POST /api/rules/regenerate
```

**重要：** 如果您的代码使用了这些API，请尽快迁移到新的API端点。

### 常见问题

**Q: 我的数据会丢失吗？**
A: 不会。所有数据都保持不变，只是访问方式改变了。

**Q: 旧的URL还能用吗？**
A: 可以。旧URL会自动重定向到新位置，但建议更新书签。

**Q: 批量操作功能去哪了？**
A: 批量规则生成功能已移除。建议在设备详情页面单独管理每个设备的规则，这样更精确和安全。

**Q: 我需要做什么？**
A: 对于普通用户，无需任何操作，系统会自动处理。对于开发者，建议尽快迁移API调用。

**Q: 如果遇到问题怎么办？**
A: 请查看文档或联系技术支持团队。
