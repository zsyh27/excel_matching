# 规则管理功能迁移指南

## 概述

规则管理功能已经从独立页面迁移到设备管理和统计仪表板中，提供更加集成和便捷的用户体验。

## 功能位置变更

### 之前（旧版本）
- 规则管理独立页面：`/rule-management`
- 规则列表和编辑
- 匹配日志查看
- 统计数据展示

### 现在（新版本）
- **设备管理页面**：`/device-management`
  - 设备列表显示规则摘要
  - 设备详情页面查看和编辑规则
  - 直接在设备上下文中管理规则

- **统计仪表板**：`/statistics`
  - 匹配日志查看和分析
  - 规则统计数据
  - 匹配成功率趋势

## 新功能位置

### 1. 查看设备规则

**位置**: 设备管理 > 设备列表

**操作步骤**:
1. 打开设备管理页面
2. 在设备列表中查看"规则状态"列
3. 显示规则摘要信息：
   - 特征数量
   - 匹配阈值
   - 总权重

**筛选功能**:
- 可以按"有规则"或"无规则"筛选设备
- 快速找到需要生成规则的设备

### 2. 编辑设备规则

**位置**: 设备管理 > 设备详情 > 匹配规则标签页

**操作步骤**:
1. 在设备列表中点击设备名称
2. 切换到"匹配规则"标签页
3. 查看完整的规则信息：
   - 匹配阈值
   - 总权重
   - 特征列表（按权重排序）
4. 点击"编辑规则"按钮
5. 在对话框中调整：
   - 特征权重（使用滑块或输入框）
   - 添加/删除特征
   - 调整匹配阈值
6. 实时查看总权重和匹配难度
7. 点击"保存"应用更改

### 3. 重新生成规则

**位置**: 设备管理 > 设备详情 > 匹配规则标签页

**操作步骤**:
1. 在设备详情的"匹配规则"标签页
2. 点击"重新生成"按钮
3. 系统自动分析设备信息生成新规则
4. 显示新旧规则对比
5. 确认后应用新规则

### 4. 查看匹配日志

**位置**: 统计仪表板 > 匹配日志标签页

**操作步骤**:
1. 打开统计仪表板页面
2. 切换到"匹配日志"标签页
3. 查看所有匹配记录
4. 使用筛选功能：
   - 按日期范围筛选
   - 按匹配状态筛选（成功/失败）
   - 按设备类型筛选
5. 点击日志查看详细信息
6. 导出日志数据

### 5. 查看规则统计

**位置**: 统计仪表板 > 规则统计标签页

**操作步骤**:
1. 打开统计仪表板页面
2. 切换到"规则统计"标签页
3. 查看统计信息：
   - 总规则数
   - 平均匹配阈值
   - 平均总权重
   - 权重分布图表
   - 阈值分布图表
4. 点击"刷新"更新数据

### 6. 查看匹配成功率

**位置**: 统计仪表板 > 匹配统计标签页

**操作步骤**:
1. 打开统计仪表板页面
2. 切换到"匹配统计"标签页
3. 查看成功率趋势图
4. 选择日期范围查看特定时期的数据
5. 查看摘要统计：
   - 总匹配次数
   - 成功次数
   - 失败次数
   - 平均成功率

## URL重定向

为了保持向后兼容，旧的URL会自动重定向到新位置：

| 旧URL | 新URL | 说明 |
|-------|-------|------|
| `/rule-management` | `/device-management` | 规则管理主页 |
| `/rule-management/logs` | `/statistics?tab=logs` | 匹配日志 |
| `/rule-management/statistics` | `/statistics?tab=rules` | 规则统计 |

**注意**: 请更新您的书签和链接到新的URL。

## API变更

### 新增API

#### 设备规则API
```
GET  /api/devices              # 包含规则摘要
GET  /api/devices/{id}         # 包含完整规则信息
PUT  /api/devices/{id}/rule    # 更新设备规则
POST /api/devices/{id}/rule/regenerate  # 重新生成规则
```

#### 统计API
```
GET /api/statistics/match-logs           # 匹配日志
GET /api/statistics/rules                # 规则统计
GET /api/statistics/match-success-rate   # 匹配成功率
```

### 废弃API

以下API将在3个月后（2026-06-04）停止支持：

```
GET  /api/rules                # 使用 /api/devices 替代
GET  /api/rules/{id}           # 使用 /api/devices/{id} 替代
PUT  /api/rules/{id}           # 使用 /api/devices/{id}/rule 替代
POST /api/rules/regenerate     # 使用 /api/devices/{id}/rule/regenerate 替代
```

**迁移建议**: 请尽快更新您的代码使用新的API端点。

## 常见问题

### Q: 为什么要迁移规则管理功能？
A: 为了提供更好的用户体验。将规则管理集成到设备管理中，用户可以在设备上下文中直接管理规则，减少页面跳转，提高工作效率。

### Q: 旧的规则管理页面还能访问吗？
A: 旧的URL会自动重定向到新位置。独立的规则管理页面已经移除。

### Q: 我的数据会丢失吗？
A: 不会。所有设备和规则数据都保持不变，只是访问方式改变了。

### Q: 批量操作功能去哪了？
A: 批量规则生成功能已移除。建议在设备详情页面单独管理每个设备的规则，这样更精确和安全。

### Q: API还能继续使用吗？
A: 旧的API端点会继续工作3个月（到2026-06-04），但建议尽快迁移到新的API。

### Q: 如何快速找到没有规则的设备？
A: 在设备管理页面，使用"规则状态"筛选器，选择"无规则"即可。

### Q: 性能有提升吗？
A: 是的。新版本使用了缓存优化，查询响应时间提升了90%以上。
