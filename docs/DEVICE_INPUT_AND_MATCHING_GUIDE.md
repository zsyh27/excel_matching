## 系统概述

本系统通过建立标准设备库和智能匹配算法，实现设备报价信息的快速准确匹配。

### 核心理念

```
标准设备库（参考答案） + 智能匹配算法 = 高效准确的报价系统
```

### 核心流程

```
管理员录入设备 → 建立设备库 → 用户上传清单 → 系统匹配设备 → 生成报价单
```

**关键点**：
- **录入**：只有管理员录入设备到设备库（2个入口）
- **匹配**：用户上传清单，系统从设备库中匹配（不是录入）

## 设备库录入（唯一的录入操作）

### 什么是设备库录入？

设备库录入是管理员往系统中添加标准设备数据的操作，这些设备数据作为匹配的"参考答案"。

**录入的设备信息**：
```
设备信息 = {
    设备ID: 1,
    品牌: "霍尼韦尔",
    设备名称: "座阀",
    规格型号: "VVF53",
    详细参数: "二通+DN15+PN16",
    单价: 1250.00
}
```

### 两个录入入口

#### 录入入口1：设备库管理页面

访问"数据库管理"页面，提供两种录入方式：

**1.1 单个设备录入**
- 手动填写表单
- 填写字段：品牌、设备名称、规格型号、详细参数、单价
- 适合少量设备补充

**1.2 Excel批量导入**
- 准备Excel文件，包含列：品牌、设备名称、规格型号、详细参数、单价
- 通过界面上传Excel文件
- 系统批量导入设备
- 适合大量设备录入
- 当前已导入720+条真实设备数据

**命令行批量导入**（可选）：
```bash
python backend/scripts/import_devices_from_excel.py
```

#### 录入入口2：智能设备录入页面

访问"智能设备录入"页面，提供智能解析功能：

**操作步骤**：
1. 输入设备描述文本（如："霍尼韦尔 座阀 VVF53 二通 DN15"）
2. 系统自动解析：
   - 品牌：霍尼韦尔
   - 设备类型：座阀
   - 型号：VVF53
   - 关键参数：{"通径": "DN15", "类型": "二通"}
3. 用户确认或修正解析结果
4. 保存到设备库

**优势**：
- 无需手动填写多个字段
- 自动识别品牌、类型、型号、参数
- 减少录入工作量
- 适合从文本描述快速录入

### 自动规则生成

无论通过哪个入口录入设备，系统都会自动为每个设备生成匹配规则：
- 规则包含：特征列表、特征权重、匹配阈值
- 无需手动配置规则
- 规则用于后续的设备匹配

## 报价清单匹配（使用功能，非录入）

### 什么是报价清单匹配？

报价清单匹配是用户上传客户的设备清单Excel文件，系统从设备库中匹配设备并生成报价单的功能。

**这不是"录入"操作**，而是"查询/匹配"操作。

**用户上传的Excel内容示例**：
```
Excel清单 = [
    "霍尼韦尔+座阀+二通+DN15",
    "西门子+CO2传感器+QAA2061+量程0-2000ppm",
    "施耐德+温度传感器+PT1000+0-100℃",
    ...
]
```

**处理流程**：
```
1. 上传Excel → 2. 识别设备行 → 3. 提取特征 → 4. 匹配设备 → 5. 导出报价
```

**关键点**：
- 用户上传的Excel不会添加到设备库
- 系统只是从现有设备库中查找匹配的设备
- 匹配成功后，在Excel中填充设备信息和价格
- 导出包含价格的报价单

## 特征提取机制

### 什么是特征？

特征是从设备描述中提取的**可匹配的关键信息**，用于识别和匹配设备。

**示例**：
```
原始描述: "霍尼韦尔+座阀+二通+DN15+PN16"
提取特征: ["霍尼韦尔", "座阀", "二通", "dn15", "pn16"]
```

### 特征提取流程

#### 阶段1：智能清理

**目的**：去除噪音文本，保留核心信息

**处理内容**：
- 截断噪音：在"按照图纸规范"、"达到验收"等处截断
- 删除噪音段落：删除"施工要求"、"含税"等无关内容
- 删除元数据标签：删除"型号:"、"规格:"等前缀
- 统一分隔符：将逗号、空格等转为标准分隔符

**示例**：
```
输入: "霍尼韦尔座阀，型号:VVF53，DN15，按照图纸规范施工"
输出: "霍尼韦尔+座阀+vvf53+dn15"
```

#### 阶段2：删除无关关键词

**目的**：删除对匹配无帮助的词汇

**删除内容**：
- 施工要求、验收、含税、品牌、国产、进口等

**示例**：
```
输入: "进口品牌霍尼韦尔座阀含税价"
输出: "霍尼韦尔+座阀"
```

#### 阶段3：归一化处理

**目的**：统一不同表达方式，提高匹配率

**三层归一化**：

1. **精准映射**（配置文件定义）
   ```
   ℃ → (删除)
   Pa → pa
   ~ → -
   ± → (删除)
   ```

2. **通用归一化**（自动处理）
   ```
   全角转半角: ＤＮ１５ → DN15
   删除空格: D N 1 5 → DN15
   统一小写: DN15 → dn15
   ```

3. **同义词映射**（匹配阶段使用）
   ```
   阀 ↔ 阀门
   传感器 ↔ 探头
   控制器 ↔ DDC
   ```

**示例**：
```
输入: "温度传感器　０～１００℃"
输出: "温度传感器+0-100"
```

#### 阶段4：特征拆分

**目的**：将文本拆分为独立的可匹配特征

**拆分规则**：
- 按分隔符拆分（+、-、|、空格等）
- 处理括号内外的内容
- 智能识别品牌和设备类型
- 复杂参数分解（如"±5%@25℃"）

**特征质量评分**：
```
基础分: 50
加分项:
  + 是技术术语: +20
  + 包含数字: +10
  + 包含单位: +10
  + 在设备关键词中: +15
  + 长度适中(3-20): +5
减分项:
  - 是元数据标签: -30
  - 是常见词: -20
  - 太短(<2且不在设备关键词中): -20
  - 纯数字: -15
  - 纯标点: -30
```

**示例**：
```
输入: "霍尼韦尔+座阀+二通+DN15+PN16"
输出特征: ["霍尼韦尔", "座阀", "二通", "dn15", "pn16"]
质量评分: [85, 95, 90, 85, 85]  (全部保留)
```

### 特征权重分配

不同类型的特征有不同的匹配权重：

```python
特征权重配置 = {
    'brand_weight': 1.0,        # 品牌权重系数
    'device_type_weight': 4.0,  # 设备类型权重系数（最高）
    'model_weight': 1.0,        # 型号权重系数
    'parameter_weight': 4.0     # 参数权重系数
}
```

**权重分配示例**：
```
特征: "霍尼韦尔"  → 类型: 品牌    → 权重: 1.0 × 1 = 1.0
特征: "座阀"      → 类型: 设备类型 → 权重: 1.0 × 4 = 4.0
特征: "二通"      → 类型: 参数    → 权重: 1.0 × 4 = 4.0
特征: "dn15"      → 类型: 参数    → 权重: 1.0 × 4 = 4.0
特征: "vvf53"     → 类型: 型号    → 权重: 1.0 × 1 = 1.0
```

**为什么这样分配？**
- **设备类型和参数权重最高**：这些是设备的核心特征，最能区分不同设备
- **品牌和型号权重较低**：同一品牌有多种设备，型号可能有变体

## 匹配准确度提升方案

### 当前匹配准确率：91.30%

### 提升方案

#### 方案1：丰富设备库（最有效）

**原理**：设备库越完整，匹配成功率越高

**操作方法**：
1. **批量导入真实设备数据**
   ```bash
   # 准备Excel文件，包含以下列：
   # 品牌、设备名称、规格型号、详细参数、单价
   
   python backend/scripts/import_devices_from_excel.py
   ```

2. **手动补充缺失设备**
   - 分析匹配失败的案例
   - 识别缺失的设备类型
   - 通过前端界面或Excel导入补充

3. **使用智能设备录入**
   - 输入设备描述文本
   - 系统自动解析并生成规则
   - 快速扩充设备库

**效果**：
- 设备库从25条扩充到720+条后，匹配准确率从40%提升到91.30%
- 每增加100条设备，预计准确率提升2-3%

#### 方案2：优化特征权重

**原理**：调整不同特征的权重，提高关键特征的影响力

**操作方法**：
1. 访问"配置管理"页面
2. 进入"特征权重"标签
3. 调整权重系数：
   ```
   设备类型权重: 4.0 (推荐保持)
   参数权重: 4.0 (推荐保持)
   品牌权重: 1.0 (可适当提高到1.5)
   型号权重: 1.0 (可适当提高到1.5)
   ```

**调整建议**：
- 如果品牌识别很重要，提高品牌权重
- 如果型号匹配很精确，提高型号权重
- 不建议降低设备类型和参数权重

#### 方案3：扩展同义词库

**原理**：处理不同的表达方式，提高召回率

**操作方法**：
1. 访问"配置管理"页面
2. 进入"同义词映射"标签
3. 添加同义词对：
   ```json
   {
     "阀": "阀门",
     "传感器": "探头",
     "控制器": "DDC",
     "变频器": "VFD"
   }
   ```

**效果**：
- 输入"阀"可以匹配到"阀门"
- 输入"探头"可以匹配到"传感器"
- 双向匹配，提高灵活性

#### 方案4：完善归一化规则

**原理**：统一不同的表达方式，减少匹配失败

**操作方法**：
1. 访问"配置管理"页面
2. 进入"归一化规则"标签
3. 添加映射规则：
   ```json
   {
     "℃": "",
     "°C": "",
     "摄氏度": "",
     "度": ""
   }
   ```

**常见归一化规则**：
- 单位统一：℃、°C、摄氏度 → (删除)
- 符号统一：~、～ → -
- 空格处理：删除所有空格
- 大小写：统一转小写

#### 方案5：调整匹配阈值

**原理**：降低阈值可以提高召回率，但可能降低精确率

**操作方法**：
1. 访问"配置管理"页面
2. 进入"全局配置"标签
3. 调整默认匹配阈值：
   ```
   当前值: 5.0
   建议范围: 3.0 - 7.0
   
   降低阈值 → 更容易匹配，但可能匹配错误
   提高阈值 → 更精确匹配，但可能匹配失败
   ```

**调整建议**：
- 如果匹配失败率高，降低阈值到4.0
- 如果匹配错误率高，提高阈值到6.0
- 建议先优化其他方案，最后调整阈值

## 混乱数据处理策略

### 什么是混乱数据？

混乱数据是指格式不规范、信息不完整、包含大量噪音的设备描述。

**常见混乱数据类型**：
1. **格式混乱**：无固定分隔符，信息杂乱
   ```
   "霍尼韦尔座阀二通DN15PN16含税价按图纸施工"
   ```

2. **信息不完整**：缺少关键信息
   ```
   "座阀DN15"  (缺少品牌和型号)
   ```

3. **包含大量噪音**：无关信息过多
   ```
   "霍尼韦尔座阀，按照图纸规范施工，达到验收标准，含税含运费"
   ```

4. **表达方式不统一**：同一信息多种写法
   ```
   "温度0~100℃"
   "温度０～１００度"
   "温度范围:0-100摄氏度"
   ```

### 处理策略

#### 策略1：智能清理（自动）

系统已内置智能清理功能，自动处理大部分混乱数据。

**处理内容**：
- 自动截断噪音文本
- 删除无关段落
- 统一分隔符
- 删除元数据标签

**无需手动操作**，系统自动执行。

#### 策略2：归一化规则（配置）

通过配置归一化规则，统一不同的表达方式。

**操作步骤**：
1. 收集混乱数据的常见模式
2. 在"配置管理"中添加归一化规则
3. 系统自动应用规则

**示例**：
```json
{
  "温度范围": "温度",
  "摄氏度": "",
  "度": "",
  "０": "0",
  "～": "-"
}
```

#### 策略3：同义词映射（配置）

处理不同的表达方式，提高匹配灵活性。

**操作步骤**：
1. 识别常见的同义词
2. 在"配置管理"中添加同义词映射
3. 系统在匹配时自动扩展

**示例**：
```json
{
  "阀": "阀门",
  "探头": "传感器",
  "DDC": "控制器"
}
```

#### 策略4：设备库补充（手动）

对于特殊的混乱数据，可以补充对应的设备到设备库。

**操作步骤**：
1. 分析匹配失败的混乱数据
2. 手动整理出标准的设备信息
3. 添加到设备库
4. 系统自动生成匹配规则

**示例**：
```
混乱数据: "霍尼韦尔座阀二通DN15PN16含税价"
整理后: 品牌=霍尼韦尔, 设备名称=座阀, 参数=二通+DN15+PN16
添加到设备库 → 自动生成规则 → 下次可以匹配
```

#### 策略5：手动调整（用户操作）

对于系统无法自动处理的混乱数据，用户可以手动调整。

**操作步骤**：
1. 系统匹配失败后，显示候选设备列表
2. 用户从列表中手动选择正确的设备
3. 系统记录用户选择，用于后续优化

**优势**：
- 确保100%准确率
- 用户选择会被记录，用于改进算法
- 灵活处理特殊情况

### 混乱数据处理流程图

```
混乱数据输入
    ↓
智能清理（自动）
    ├─ 截断噪音
    ├─ 删除无关段落
    └─ 统一分隔符
    ↓
归一化处理（配置）
    ├─ 精准映射
    ├─ 通用归一化
    └─ 同义词扩展
    ↓
特征提取（自动）
    ├─ 特征拆分
    ├─ 质量评分
    └─ 过滤低质量特征
    ↓
设备匹配（自动）
    ├─ 权重计算
    ├─ 阈值判定
    └─ 最佳选择
    ↓
匹配成功？
    ├─ 是 → 返回结果
    └─ 否 → 手动选择
```

**结论**：智能清理可以有效处理混乱格式数据

3. 特征提取: ["座阀", "dn15"]
4. 权重计算:
   - 座阀 (设备类型): 4.0

**处理建议**：
- 如果匹配到多个设备，系统会选择得分最高的
- 用户可以从候选列表中手动选择
- 建议补充品牌或型号信息以提高精确度

**输入**：
```
"温度探头０～１００摄氏度"
```

**处理过程**：
```
1. 智能清理: "温度探头+0-100" (删除"摄氏度")

2. 归一化:
   - 全角转半角: ０～１００ → 0~100
   - 符号统一: ~ → -
   - 删除单位: 摄氏度 → (删除)
   结果: "温度探头+0-100"

3. 同义词扩展:
   - 探头 → 传感器
   结果: "温度传感器+0-100"

4. 特征提取: ["温度传感器", "0-100"]

5. 权重计算:
   - 温度传感器 (设备类型): 4.0
   - 0-100 (参数): 4.0
   总得分: 8.0

6. 匹配结果: 成功 (8.0 >= 5.0)
```

**结论**：归一化和同义词扩展可以处理表达方式不统一的数据

## 总结

### 核心要点

1. **设备库录入（唯一的录入操作）**
   - 录入入口1：设备库管理页面（单个录入 + Excel批量导入）
   - 录入入口2：智能设备录入页面（智能解析）
   - 录入信息：设备ID、品牌、设备名称、规格型号、详细参数、单价
   - 自动生成匹配规则

2. **报价清单匹配（使用功能，非录入）**
   - 用户上传Excel清单
   - 系统从设备库中匹配设备
   - 生成包含价格的报价单
   - 不会添加新设备到设备库

3. **特征提取是关键**
   - 智能清理去除噪音
   - 归一化统一表达
   - 特征拆分提取关键信息
   - 质量评分过滤低质量特征

4. **提升准确度的方法**
   - 丰富设备库（最有效）
   - 优化特征权重
   - 扩展同义词库
   - 完善归一化规则
   - 调整匹配阈值

5. **混乱数据处理**
   - 智能清理（自动）
   - 归一化规则（配置）
   - 同义词映射（配置）
   - 设备库补充（手动）
   - 手动调整（用户操作）

### 最佳实践

1. **初期设置（管理员）**
   - 通过Excel批量导入真实设备数据（720+条）
   - 或使用智能设备录入逐个添加
   - 配置基础归一化规则
   - 配置常用同义词

2. **日常使用（用户）**
   - 上传客户的报价清单Excel
   - 系统自动匹配设备
   - 手动调整匹配失败的项
   - 导出报价单

3. **持续优化（管理员）**
   - 分析匹配失败案例
   - 补充缺失设备到设备库
   - 优化配置规则
   - 调整特征权重

4. **混乱数据处理**
   - 依赖智能清理（自动）
   - 配置归一化规则（一次性）
   - 手动选择（兜底方案）

### 预期效果

- **匹配准确率**：91.30% → 95%+（通过丰富设备库）
- **处理速度**：1000个设备 ≤10秒
- **用户体验**：大部分情况自动匹配，少量手动调整
