# 前端配置测试接口参数修复

## 问题

配置管理页面的实时预览功能报错：
- 前端提示：测试失败: 请求中缺少 test_text 参数
- 后台提示：400错误

## 原因

前端和后端的参数名不匹配：
- **前端发送**：`{ text: "...", config: {...} }`
- **后端期望**：`{ test_text: "...", config: {...} }`

## 修复

**文件**：`frontend/src/api/config.js`

**修改**：
```javascript
// 修复前
export const testConfig = (text, config) => {
  return api.post('/config/test', { text, config })
}

// 修复后
export const testConfig = (text, config) => {
  return api.post('/config/test', { test_text: text, config })
}
```

## 结果

✅ 前端现在发送正确的参数名 `test_text`  
✅ 配置管理页面的实时预览功能可以正常工作
