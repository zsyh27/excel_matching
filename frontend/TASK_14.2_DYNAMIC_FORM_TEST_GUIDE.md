# Task 14.2 - 动态设备表单测试指南

## 测试目标

验证动态设备表单功能是否正确实现，包括：
- 设备类型选择
- 动态参数表单渲染
- 设备类型切换逻辑
- 参数数据绑定
- 表单样式优化

## 前置条件

1. 后端服务运行在 `http://localhost:5000`
2. 前端服务运行在 `http://localhost:3001`
3. 数据库已完成Schema迁移（包含device_type、key_params等字段）
4. device_params.yaml配置文件存在且包含设备类型配置

## 测试步骤

### 测试 14.2.1 - 设备类型选择组件

**验证需求**: 36.1

1. 访问设备库管理页面: `http://localhost:3001/#/database/devices`
2. 点击"添加设备"按钮
3. 验证表单中显示"设备类型"下拉选择框
4. 点击设备类型下拉框
5. **预期结果**:
   - 显示所有可用的设备类型（CO2传感器、座阀、温度传感器等）
   - 支持搜索过滤功能
   - 可以清空选择

### 测试 14.2.2 - 动态参数表单渲染

**验证需求**: 36.1-36.5

1. 在添加设备表单中，选择设备类型"CO2传感器"
2. **预期结果**:
   - 表单下方显示"设备参数"分隔线
   - 显示CO2传感器的参数字段：
     - 量程（必填，单位：ppm）
     - 输出信号（必填，单位：mA）
     - 精度（可选）
   - 必填字段标签显示红色星号
   - 输入框右侧显示单位提示

3. 尝试不填写必填参数直接提交
4. **预期结果**:
   - 显示验证错误："请输入量程"、"请输入输出信号"

### 测试 14.2.3 - 设备类型切换逻辑

**验证需求**: 36.2

1. 在设备类型下拉框中选择"CO2传感器"
2. 填写参数：
   - 量程: 0-2000 ppm
   - 输出信号: 4-20 mA
3. 切换设备类型为"温度传感器"
4. **预期结果**:
   - 之前填写的CO2传感器参数被清空
   - 显示温度传感器的参数字段：
     - 量程（必填，单位：℃）
     - 输出信号（必填）
     - 精度（可选）
   - 其他字段（设备ID、品牌、名称等）保持不变

### 测试 14.2.4 - 参数数据绑定（创建模式）

**验证需求**: 36.6

1. 填写完整的设备信息：
   - 设备ID: TEST_CO2_001
   - 品牌: 霍尼韦尔
   - 设备类型: CO2传感器
   - 设备名称: CO2浓度传感器
   - 规格型号: QAA2061
   - 量程: 0-2000 ppm
   - 输出信号: 4-20 mA
   - 精度: ±50 ppm
   - 单价: 450
2. 勾选"自动生成规则"
3. 点击"创建"按钮
4. **预期结果**:
   - 显示成功提示
   - 设备列表刷新，显示新创建的设备
   - 设备的key_params字段包含规范化的JSON数据

5. 在后端验证数据：
```bash
curl http://localhost:5000/api/devices/TEST_CO2_001
```

6. **预期JSON结构**:
```json
{
  "device_id": "TEST_CO2_001",
  "device_type": "CO2传感器",
  "key_params": {
    "量程": {
      "value": "0-2000 ppm",
      "raw_value": "0-2000 ppm",
      "data_type": "range",
      "unit": "ppm",
      "confidence": 1.0
    },
    "输出信号": {
      "value": "4-20 mA",
      "raw_value": "4-20 mA",
      "data_type": "string",
      "unit": "mA",
      "confidence": 1.0
    },
    "精度": {
      "value": "±50 ppm",
      "raw_value": "±50 ppm",
      "data_type": "string",
      "unit": null,
      "confidence": 1.0
    }
  }
}
```

### 测试 14.2.4 - 参数数据绑定（编辑模式）

**验证需求**: 36.7

1. 在设备列表中找到刚创建的设备 TEST_CO2_001
2. 点击"编辑"按钮
3. **预期结果**:
   - 表单正确回填所有字段
   - 设备类型显示为"CO2传感器"
   - 动态参数正确回填：
     - 量程: 0-2000 ppm
     - 输出信号: 4-20 mA
     - 精度: ±50 ppm

4. 修改参数：
   - 量程: 0-5000 ppm
   - 精度: ±30 ppm
5. 勾选"重新生成规则"
6. 点击"保存"按钮
7. **预期结果**:
   - 显示成功提示
   - 设备信息更新
   - 规则重新生成

### 测试 14.2.5 - 表单样式优化

1. 观察动态参数区域的样式
2. **预期结果**:
   - 动态参数区域有浅灰色背景（#f5f7fa）
   - 参数区域有圆角边框
   - "设备参数"分隔线文字为蓝色
   - 参数提示信息显示为灰色小字
   - 表单布局清晰，间距合理

3. 调整浏览器窗口大小
4. **预期结果**:
   - 表单响应式布局正常
   - 在不同屏幕尺寸下显示正常

### 测试 14.2.6 - 向后兼容性测试

1. 创建一个不选择设备类型的设备：
   - 设备ID: TEST_OLD_001
   - 品牌: 西门子
   - 设备类型: （不选择）
   - 设备名称: 旧式传感器
   - 规格型号: OLD-123
   - 详细参数: 测量范围: 0-100℃, 输出: 4-20mA
   - 单价: 300
2. 点击"创建"按钮
3. **预期结果**:
   - 设备创建成功
   - 不显示动态参数表单
   - 使用detailed_params字段存储参数

4. 编辑该设备，选择设备类型"温度传感器"
5. **预期结果**:
   - 显示温度传感器的动态参数表单
   - 可以填写key_params
   - 保存后同时保留detailed_params和key_params
