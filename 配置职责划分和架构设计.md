# 配置职责划分和架构设计

## 核心设计原则

**设备录入阶段**和**匹配阶段**应该使用不同的配置，职责清晰分离。

## 配置职责划分

### 📋 设备参数配置（device_params）

**配置位置**: 配置管理 → 设备参数配置

**用途**: 设备录入阶段 - 定义设备结构

**职责**:
1. 定义每种设备类型有哪些参数
2. 指定参数的提取规则
3. 这些参数会被提取为特征

**示例**:
```json
{
  "温度传感器": {
    "params": [
      {
        "name": "温度量程",
        "type": "range",
        "required": true
      },
      {
        "name": "温度信号类型",
        "type": "text",
        "required": true
      },
      {
        "name": "安装位置",
        "type": "text",
        "required": false
      }
    ]
  }
}
```

**特征来源**:
- `device.device_type` → "温度传感器"
- `device.brand` → "霍尼韦尔"
- `device.spec_model` → "HST-RA"
- `device.key_params` → {"温度量程": "-20~60℃", "温度信号类型": "NTC 10K", ...}

---

### ⚖️ 特征权重配置（feature_weight_config）

**配置位置**: 配置管理 → 特征权重

**用途**: 设备录入阶段 - 分配特征权重

**职责**:
1. 定义不同类型特征的权重
2. 用于规则生成时的权重分配
3. 不涉及特征识别和拆分

**示例**:
```json
{
  "device_type_weight": 20.0,    // device.device_type字段的权重
  "brand_weight": 10.0,           // device.brand字段的权重
  "model_weight": 5.0,            // device.spec_model字段的权重
  "key_params_weight": 15.0,     // device.key_params中参数的权重
  "parameter_weight": 1.0         // 其他通用参数的权重
}
```

**权重分配逻辑**:
```
特征来源                    → 权重
device.device_type         → 20分（最高）
device.key_params          → 15分（次高）
device.brand               → 10分（中高）
device.spec_model          → 5分（中）
其他特征                    → 1分（低）
```

---

### 📦 设备类型关键词（device_type_keywords）

**配置位置**: 配置管理 → 设备类型

**用途**: 匹配阶段 - 识别用户输入

**职责**:
1. 识别用户输入中的设备类型
2. 智能拆分混乱的输入
3. 提高匹配召回率

**示例**:
```json
[
  "温度传感器",
  "传感器",
  "控制器",
  "DDC",
  "阀门",
  "执行器"
]
```

**使用场景**:
- 用户输入: "室内温度传感器"
- 智能拆分: ["室内", "温度传感器"]
- 匹配时可以匹配到"温度传感器"设备

---

### 🔧 高级配置（metadata_keywords + whitelist）

**配置位置**: 配置管理 → 高级配置

**用途**: 匹配阶段 - 处理混乱输入

**职责**:
1. 元数据关键词: 删除字段名前缀（如"型号:"）
2. 特征白名单: 保护重要但简短的特征（如"水"、"气"）

**示例**:
```json
{
  "metadata_keywords": ["型号", "通径", "品牌", "规格"],
  "whitelist_features": ["水", "气", "阀", "泵"]
}
```

## 当前实现的问题

### ❌ 问题1: 设备录入阶段使用了device_type_keywords

**当前代码** (`rule_generator.py`):
```python
def _assign_feature_weight(self, feature, strategy, device, key_params_features):
    # ...
    
    # 4. 设备类型关键词：高权重
    if self._is_device_type_keyword(feature_lower):
        weight = strategy.get('device_type_weight', 15.0)  # ❌ 错误！
        return weight
```

**问题**:
- 在设备录入阶段判断特征是否包含device_type_keywords
- 这会导致"传感器"等词被赋予高权重
- 但这些词应该只在匹配阶段使用

### ❌ 问题2: 权重配置名称混淆

**当前配置**:
```json
{
  "device_type_weight": 15.0  // ❌ 这个名称有歧义
}
```

**混淆点**:
- 这个权重是给device.device_type字段用的？
- 还是给device_type_keywords判断用的？

## 正确的架构设计

### ✅ 设备录入阶段

**特征提取**:
1. 从设备字段提取特征
   - `device.device_type` → "温度传感器"
   - `device.brand` → "霍尼韦尔"
   - `device.spec_model` → "HST-RA"
   - `device.key_params` → {"温度量程": "-20~60℃", ...}

2. 预处理特征
   - 使用`mode='device'`
   - 不使用元数据关键词
   - 不使用特征白名单
   - 不使用device_type_keywords
   - 不进行智能拆分

**权重分配**:
```python
def _assign_feature_weight(self, feature, strategy, device, key_params_features):
    # 1. device.device_type字段 → 使用device_type_weight
    if device and device.device_type and feature == device.device_type:
        return self.device_type_weight  # 20分
    
    # 2. device.spec_model字段 → 使用model_weight
    if device and device.spec_model and feature == device.spec_model:
        return self.model_weight  # 5分
    
    # 3. device.key_params参数 → 使用key_params_weight
    if key_params_features and feature in key_params_features:
        return self.key_params_weight  # 15分
    
    # 4. device.brand字段 → 使用brand_weight
    if device and device.brand and feature == device.brand:
        return self.brand_weight  # 10分
    
    # 5. 其他特征 → 使用parameter_weight
    return self.parameter_weight  # 1分
```

**不使用**:
- ❌ device_type_keywords判断
- ❌ 元数据关键词处理
- ❌ 特征白名单判断
- ❌ 智能拆分

### ✅ 匹配阶段

**特征提取**:
1. 预处理用户输入
   - 使用`mode='matching'`
   - 使用元数据关键词删除前缀
   - 使用特征白名单保护重要特征
   - 使用device_type_keywords智能拆分

2. 智能拆分
   - "室内温度传感器" → ["室内", "温度传感器"]
   - "型号:QAA2061" → ["qaa2061"]

**匹配逻辑**:
- 使用规则中的特征和权重
- 计算相似度得分
- 应用匹配阈值

## 配置重命名建议

为了避免混淆，建议重命名配置项：

### 当前配置（有歧义）

```json
{
  "device_type_weight": 20.0,  // ❌ 不清楚是给哪个用的
  "brand_weight": 10.0,
  "model_weight": 5.0
}
```

### 建议配置（清晰）

```json
{
  // 设备字段权重（用于设备录入阶段）
  "field_weights": {
    "device_type": 20.0,    // device.device_type字段
    "brand": 10.0,          // device.brand字段
    "spec_model": 5.0,      // device.spec_model字段
    "key_params": 15.0      // device.key_params参数
  },
  
  // 通用权重
  "default_weights": {
    "parameter": 1.0        // 其他通用参数
  }
}
```

## 实施方案

### 方案1: 最小修改（推荐）

**修改点**:
1. 在`rule_generator.py`的`_assign_feature_weight`方法中
2. 移除device_type_keywords判断
3. 只使用设备字段和key_params判断

**优势**:
- ✅ 修改最小
- ✅ 不需要重命名配置
- ✅ 向后兼容

**代码修改**:
```python
def _assign_feature_weight(self, feature, strategy, device, key_params_features):
    feature_lower = feature.lower()
    
    # 1. device.device_type字段
    if device and device.device_type:
        if feature == device.device_type or feature_lower == device.device_type.lower():
            return self.device_type_weight
    
    # 2. device.spec_model字段
    if device and device.spec_model:
        if feature == device.spec_model or feature_lower == device.spec_model.lower():
            return self.model_weight
    
    # 3. device.key_params参数
    if key_params_features and feature in key_params_features:
        return self.key_params_weight
    
    # 4. device.brand字段
    if device and device.brand:
        if feature == device.brand or feature_lower == device.brand.lower():
            return self.brand_weight
    
    # ❌ 移除: 不再使用device_type_keywords判断
    # if self._is_device_type_keyword(feature_lower):
    #     return strategy.get('device_type_weight', 15.0)
    
    # 5. 其他特征
    return self.parameter_weight
```

## 总结

### 核心原则

1. ✅ **设备录入阶段**: 只使用设备字段和device_params配置
2. ✅ **权重分配**: 只使用feature_weight_config配置
3. ✅ **匹配阶段**: 使用device_type_keywords、元数据关键词、特征白名单

### 配置职责

| 配置项 | 用途 | 阶段 |
|--------|------|------|
| device_params | 定义设备结构 | 设备录入 |
| feature_weight_config | 分配特征权重 | 设备录入 |
| device_type_keywords | 识别设备类型 | 匹配 |
| metadata_keywords | 处理混乱输入 | 匹配 |
| whitelist_features | 保护重要特征 | 匹配 |
