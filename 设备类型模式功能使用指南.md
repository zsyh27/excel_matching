# 设备类型模式功能使用指南

## 页面概述

**位置**：配置管理 → 智能特征提取 → 设备类型模式

**作用**：配置智能设备类型识别系统，用于从Excel设备描述中自动识别设备类型。

**应用场景**：当用户上传Excel文件进行设备匹配时，系统会使用这些配置来识别每行描述的设备类型。

---

## 功能一：基础设备类型

### 功能说明

管理系统能够识别的所有设备类型列表。这是设备类型识别的基础。

### 当前默认设备类型

系统预置了8个基础设备类型：
1. 传感器
2. 探测器
3. 变送器
4. 控制器
5. 执行器
6. 阀门
7. 流量计
8. 能量计

### 如何使用

#### 1. 添加设备类型

**操作步骤**：
1. 点击"添加设备类型"按钮
2. 在弹出的对话框中输入设备类型名称
   - 例如："压力变送器"、"温湿度传感器"、"电动球阀"
3. 点击"确定"
4. 设备类型立即显示在列表中

**命名建议**：
- ✅ 使用标准的设备类型名称（如"温度传感器"）
- ✅ 可以添加具体的子类型（如"压力变送器"）
- ✅ 保持命名简洁明了
- ❌ 避免使用过于宽泛的名称（如"设备"）

**示例**：
```
添加设备类型：
- 压力变送器
- 流量变送器
- 温湿度传感器
- CO2探测器
- 电动球阀
- 蝶阀
```

#### 2. 编辑设备类型

**操作步骤**：
1. 在设备类型列表中找到要编辑的类型
2. 点击该行的"编辑"按钮
3. 修改设备类型名称
4. 点击"确定"保存

**使用场景**：
- 修正拼写错误
- 调整命名规范
- 更新设备类型名称

#### 3. 删除设备类型

**操作步骤**：
1. 在设备类型列表中找到要删除的类型
2. 点击该行的"删除"按钮
3. 确认删除

**注意事项**：
- ⚠️ 删除后，系统将无法识别该设备类型
- ⚠️ 如果该类型已被前缀关键词或主类型映射引用，建议先更新相关配置

## 功能二：前缀关键词

### 功能说明

配置前缀关键词与设备类型的映射关系。当输入文本包含前缀关键词时，系统会推断可能的设备类型。

### 工作原理

**识别逻辑**：
```
输入文本："压力变送器 0-10Bar"
↓
检测到前缀："压力"
↓
查找映射：压力 → ['传感器', '探测器', '变送器']
↓
结合文本中的"变送器"
↓
识别结果：压力变送器（主类型：变送器）
```

### 当前默认前缀关键词

系统预置了11个前缀关键词：

| 前缀 | 关联设备类型 |
|------|-------------|
| 温度 | 传感器、探测器、变送器 |
| 湿度 | 传感器、探测器、变送器 |
| 压力 | 传感器、探测器、变送器 |
| 压差 | 传感器、探测器、变送器 |
| CO | 传感器、探测器、变送器 |
| CO2 | 传感器、探测器、变送器 |
| PM2.5 | 传感器、探测器、变送器 |
| PM10 | 传感器、探测器、变送器 |
| 室内 | 传感器、探测器、变送器 |
| 室外 | 传感器、探测器、变送器 |
| 风管 | 传感器、探测器、变送器 |

### 如何使用

#### 1. 添加前缀关键词

**操作步骤**：
1. 点击"添加前缀词"按钮
2. 在弹出的对话框中：
   - **前缀词**：输入前缀关键词（如"流量"）
   - **关联设备类型**：从下拉列表中选择一个或多个设备类型
3. 点击"确定"

**配置示例**：
```
前缀词：流量
关联设备类型：['变送器', '流量计']

前缀词：液位
关联设备类型：['传感器', '变送器']

前缀词：电动
关联设备类型：['阀门', '执行器']
```

**配置建议**：
- ✅ 前缀词应该是设备描述中常见的特征词
- ✅ 一个前缀可以关联多个设备类型
- ✅ 关联的设备类型应该是实际可能出现的类型
- ❌ 避免使用过于通用的前缀（如"设备"、"系统"）

#### 2. 编辑前缀关键词

**操作步骤**：
1. 在前缀关键词列表中找到要编辑的前缀
2. 点击该行的"编辑"按钮
3. 修改前缀词或关联的设备类型
4. 点击"确定"保存

**使用场景**：
- 调整关联的设备类型
- 修正前缀词拼写
- 优化识别准确率

#### 3. 删除前缀关键词

**操作步骤**：
1. 在前缀关键词列表中找到要删除的前缀
2. 点击该行的"删除"按钮
3. 确认删除

## 功能三：实时测试

### 功能说明

在保存配置前，实时测试设备类型识别效果，验证配置是否正确。

### 如何使用

#### 1. 输入测试文本

在测试框中输入设备描述，例如：
```
压力变送器 0-10Bar
温度传感器 -20~60℃
CO2探测器 0-2000ppm 输出4-20mA
电动球阀 DN50 PN16
```

#### 2. 点击测试按钮

点击"测试"按钮或按回车键执行测试。

#### 3. 查看测试结果

系统会显示识别结果：

**结果字段说明**：

| 字段 | 说明 | 示例 |
|------|------|------|
| 主类型 | 设备的主要类别 | 变送器、传感器、探测器 |
| 子类型 | 具体的设备类型 | 压力变送器、温度传感器 |
| 置信度 | 识别的可信程度（0-100%） | 100%、90%、80% |
| 匹配模式 | 使用的识别方法 | exact、fuzzy、keyword、inference |

**匹配模式说明**：

| 模式 | 置信度 | 说明 | 示例 |
|------|--------|------|------|
| exact | 100% | 精确匹配：完整的设备类型名称 | "压力变送器" 完全匹配 |
| fuzzy | 90% | 模糊匹配：前缀+类型组合 | "压力.*?变送器" 模式匹配 |
| keyword | 80% | 关键词匹配：前缀和类型分别匹配 | "压力" 和 "变送器" 都存在 |
| inference | 70% | 类型推断：根据前缀推断 | 只有"压力"，推断为"压力传感器" |
| none | 0% | 无法识别 | 未找到任何匹配 |

#### 4. 根据结果调整配置

**如果识别成功**：
- ✅ 置信度 ≥ 90%：配置良好，可以保存
- ⚠️ 置信度 70-90%：可以使用，但建议优化配置
- ❌ 置信度 < 70%：建议调整配置

**如果识别失败**（置信度0%）：
1. 检查设备类型是否已添加
2. 检查前缀关键词是否已配置
3. 检查主类型映射是否正确
4. 调整配置后重新测试

### 测试示例

#### 示例1：精确匹配（最佳）
```
输入：压力变送器 0-10Bar
结果：
  主类型：变送器
  子类型：压力变送器
  置信度：100%
  匹配模式：exact
```

#### 示例2：模糊匹配（良好）
```
输入：室内温度传感器 -20~60℃
结果：
  主类型：传感器
  子类型：温度传感器
  置信度：90%
  匹配模式：fuzzy
```

#### 示例3：关键词匹配（可用）
```
输入：压力 变送器 量程0-10Bar
结果：
  主类型：变送器
  子类型：压力变送器
  置信度：80%
  匹配模式：keyword
```

#### 示例4：类型推断（需优化）
```
输入：压力设备 0-10Bar
结果：
  主类型：传感器
  子类型：压力传感器
  置信度：70%
  匹配模式：inference
```

#### 示例5：识别失败（需调整配置）
```
输入：流量变送器 DN50
结果：
  主类型：未识别
  子类型：未识别
  置信度：0%
  匹配模式：none

原因：
  - "流量变送器"不在设备类型列表中
  - "流量"前缀未配置

解决方案：
  1. 添加设备类型："流量变送器"
  2. 添加前缀关键词："流量" → ['变送器', '流量计']
  3. 重新测试
```

## 功能四：保存配置

### 功能说明

将所有配置更改保存到数据库，使配置生效。

### 如何使用

#### 1. 点击保存配置按钮

在页面底部点击"保存配置"按钮。

#### 2. 等待保存完成

系统会显示"配置保存成功"的提示消息。

#### 3. 配置立即生效

保存后，配置会立即生效，无需重启后端服务。

### 注意事项

⚠️ **重要**：
- 所有配置更改（添加、编辑、删除）都必须点击"保存配置"才会持久化
- 如果不保存就刷新页面，所有更改都会丢失
- 保存成功后，实时测试会立即使用新配置

---

## 功能五：重置配置

### 功能说明

放弃当前所有未保存的更改，恢复到上次保存的状态。

### 如何使用

点击页面底部的"重置"按钮。

### 使用场景

- 误操作后想要撤销
- 测试配置效果不理想，想要恢复
- 放弃当前的配置更改

## 高级使用技巧

### 技巧1：批量添加相关设备类型

**场景**：添加一系列相关的变送器类型

   - 压力变送器
   - 温度变送器

   - 压力 → [变送器, 传感器]
   - 温度 → [变送器, 传感器]
   - 湿度 → [变送器, 传感器]
   - 流量 → [变送器, 流量计]
   - 液位 → [变送器, 传感器]

3. 批量测试：
   ```
   压力变送器 0-10Bar
   温度变送器 -20~60℃
   湿度变送器 0-100%RH
   流量变送器 DN50
   液位变送器 0-5m
   ```

### 技巧2：优化识别准确率

**方法1：添加更具体的设备类型**
```
不够具体：传感器
更具体：温度传感器、压力传感器、湿度传感器
```

**方法2：完善前缀关键词映射**
```
基础配置：
  温度 → [传感器]

优化配置：
  温度 → [传感器, 探测器, 变送器]
  室内温度 → [传感器]
  室外温度 → [传感器]
```

**方法3：使用实时测试验证**
- 测试各种可能的输入格式
- 确保常见的设备描述都能正确识别
- 根据测试结果调整配置

### 技巧3：处理组合设备

**场景**：识别"蝶阀+电动执行器"这类组合设备

**配置方法**：
1. 添加设备类型：
   - 蝶阀
   - 电动执行器
   - 蝶阀+电动执行器（组合类型）

2. 添加前缀关键词：
   - 蝶阀 → [阀门]
   - 电动 → [执行器, 阀门]

3. 测试：
   ```
   蝶阀+电动执行器 DN100
   电动蝶阀 DN100
   蝶阀 DN100 配电动执行器
   ```

## 常见问题

### Q1：添加设备类型后，测试仍然无法识别？

**可能原因**：
1. 没有点击"保存配置"
2. 前缀关键词未配置
3. 输入文本格式不匹配

**解决方案**：
1. 确保点击"保存配置"
2. 添加相应的前缀关键词
3. 使用实时测试验证

### Q2：置信度很低（70%），如何提高？

**解决方案**：
1. 添加更精确的设备类型名称
2. 完善前缀关键词映射
3. 确保输入文本包含完整的设备类型名称

### Q3：一个前缀应该关联哪些设备类型？

**建议**：
- 关联所有可能出现的设备类型
- 例如"温度"可以关联：传感器、探测器、变送器
- 系统会根据文本中的其他信息选择最合适的类型

### Q4：如何知道配置是否正确？

**验证方法**：
1. 使用实时测试功能
2. 测试各种可能的输入格式
3. 确保置信度 ≥ 90%
4. 在实际Excel匹配中验证效果

### Q5：配置保存后多久生效？

**答案**：立即生效！
- 保存成功后，配置会自动重新加载
- 实时测试会立即使用新配置
- 无需重启后端服务
