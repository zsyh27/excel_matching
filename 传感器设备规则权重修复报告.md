# 传感器设备规则权重修复报告

## 问题描述

用户报告设备详情显示问题:
- 设备ID: `霍尼韦尔_HST-RA_20260306113921944861`
- 设备类型: 温度传感器
- 问题: 规则中显示"传感器"权重1,而不是"温度传感器"权重20

## 根本原因

通过深入调试,发现问题出在`backend/modules/text_preprocessor.py`的`_extract_bracket_features`方法中:

1. **metadata_keywords配置问题**: "温度"被错误地添加到了`metadata_keywords`列表中
2. **前缀删除逻辑**: `_extract_bracket_features`方法会删除元数据关键词前缀
3. **错误拆分**: "温度传感器"被识别为"温度:传感器"的形式,删除了"温度"前缀,只保留了"传感器"

### 问题追踪过程

```
输入: "温度传感器"
↓
_extract_bracket_features方法:
  - 检查metadata_keywords: "温度" in metadata_keywords → True
  - 删除前缀: "温度传感器"[len("温度"):] → "传感器"
↓
输出: ["传感器"]
```

## 解决方案

### 1. 修复metadata_keywords配置

从`metadata_keywords`配置中删除可能是设备类型一部分的词:

**删除的关键词:**
- 温度 (影响"温度传感器")
- 压力 (影响"压力传感器")
- 功率 (可能影响其他设备类型)

**修复脚本:** `fix_metadata_keywords.py`

**修复前:** 29个关键词
**修复后:** 26个关键词

### 2. 重新生成设备规则

使用修复后的配置重新生成受影响设备的规则。

**修复脚本:** `fix_sensor_device_rule.py`

### 如何在前端管理元数据关键词

1. 打开系统,进入"配置管理"页面
2. 点击"高级配置"标签
3. 找到"元数据关键词"部分

### 注意事项

⚠️ **不要添加设备类型的组成部分**

以下词汇不应该添加到元数据关键词列表:
- ❌ 温度 (会导致"温度传感器"被拆分)
- ❌ 压力 (会导致"压力传感器"被拆分)
- ❌ 湿度 (会导致"湿度传感器"被拆分)
- ❌ 流量 (会导致"流量计"被拆分)

✅ **应该添加的是字段名称**

例如:
- 型号、品牌、规格、参数、名称、类型、尺寸、材质等

详细说明请参考: [元数据关键词配置使用指南](./元数据关键词配置使用指南.md)
