## 📝 如何访问配置

### 旧方法（已废弃）

```python
# ❌ 不再可用
import yaml
with open('backend/config/device_params.yaml', 'r') as f:
    device_params = yaml.safe_load(f)

# ❌ 不再可用
import json
with open('backend/config/intelligent_extraction_config.json', 'r') as f:
    extraction_config = json.load(f)
```

### 新方法（推荐）

```python
# ✅ 从数据库读取
import sys
sys.path.insert(0, 'backend')

from modules.database import DatabaseManager
from modules.database_loader import DatabaseLoader

# 初始化
db_manager = DatabaseManager("sqlite:///data/devices.db")
db_loader = DatabaseLoader(db_manager)

# 访问配置
device_params = db_loader.get_config_by_key('device_params')
extraction_config = db_loader.get_config_by_key('intelligent_extraction')
```

## ❓ 常见问题

### Q1: 我的旧代码还能运行吗？

如果你的代码直接读取 YAML 或 JSON 配置文件，需要更新为数据库访问。参考上面的"新方法"示例。

### Q2: 如何恢复删除的文件？

备份文件在 `.backup/config/` 目录，但建议使用数据库配置，不要恢复旧文件。

### Q3: 测试脚本怎么办？

测试脚本可以继续使用 `data/static_config.json`，该文件保留作为副本。

### Q4: 如何同步配置？

运行 `python sync_config_db_to_json.py` 可以将数据库配置同步到 JSON 文件。
