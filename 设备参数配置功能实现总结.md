# 设备参数配置功能实现总结

## 工作概述

根据用户需求，实现了设备参数配置的可视化界面，允许用户为每种设备类型配置对应的参数字段。该功能暂不包含正则表达式编辑，专注于简单直观的参数配置。

## 完成的工作

### 1. 前端组件开发

#### 1.1 创建 DeviceParamsEditor 组件
**文件**：`frontend/src/components/ConfigManagement/DeviceParamsEditor.vue`

**功能特性**：
- 左右分栏布局
- 左侧：设备类型列表
  - 显示所有设备类型
  - 显示每个类型的参数数量
  - 支持添加/删除设备类型
- 右侧：参数编辑器
  - 参数名称输入
  - 必填/可选开关
  - 数据类型选择（字符串/数字/范围）
  - 单位输入
  - 提示信息输入
  - 支持添加/删除参数

**技术实现**：
- Vue 3 Composition API
- Element Plus UI组件
- 响应式数据绑定
- v-model 双向绑定

#### 1.2 集成到配置管理页面
**文件**：`frontend/src/views/ConfigManagementView.vue`

**修改内容**：
1. 导入 `DeviceParamsEditor` 组件
2. 在 components 中注册组件
3. 在菜单项中添加"设备参数配置"（📋图标）
4. 在编辑器映射中添加 `device_params: 'DeviceParamsEditor'`

**菜单位置**：
- 位于"匹配配置阶段"
- 在"设备类型"和"特征权重"之间

### 2. 后端数据配置

#### 2.1 数据库配置更新
**文件**：`data/devices.db` (configs 表)

**操作**：
- 在 `configs` 表中添加 `device_params` 配置项
- 配置键：`device_params`
- 配置值：JSON格式的设备参数配置
- 描述：设备参数配置，用于动态表单生成

#### 2.2 初始数据
添加了4种常见设备类型的参数配置：

1. **CO2传感器**
   - 量程（必填，范围，ppm）
   - 输出信号（必填，字符串，mA）
   - 精度（可选，字符串）

2. **温度传感器**
   - 量程（必填，范围，℃）
   - 输出信号（必填，字符串）
   - 精度（可选，字符串）

3. **座阀**
   - 通径（必填，数字，mm）
   - 压力等级（可选，数字，bar）
   - 流量系数（可选，数字）

4. **执行器**
   - 扭矩（可选，数字，N·m）
   - 行程时间（可选，数字，s）
   - 控制信号（可选，字符串）

#### 2.3 JSON配置文件同步
**文件**：`data/static_config.json`

也在JSON配置文件中添加了相同的 `device_params` 配置，保持数据一致性。

## 数据结构

### device_params 配置格式

```json
{
  "设备类型名称": {
    "params": [
      {
        "name": "参数名称",
        "required": true/false,
        "data_type": "string/number/range",
        "unit": "单位",
        "hint": "提示信息"
      }
    ]
  }
}
```

### 示例

```json
{
  "CO2传感器": {
    "params": [
      {
        "name": "量程",
        "required": true,
        "data_type": "range",
        "unit": "ppm",
        "hint": "例如：0-2000ppm"
      }
    ]
  }
}
```

## 技术要点

### 1. 组件通信
- 使用 `v-model` 实现父子组件双向数据绑定
- 通过 `emit` 触发 `update:modelValue` 和 `change` 事件
- 父组件通过 `config[activeTab]` 传递数据

### 2. 数据持久化
- 配置保存到数据库 `configs` 表
- 使用 SQLAlchemy 的 JSON 类型自动序列化/反序列化
- 支持配置历史记录和回滚

### 3. 响应式更新
- 使用 Vue 3 的 `ref` 和 `computed` 实现响应式
- 使用 `watch` 监听外部数据变化
- 深拷贝确保数据独立性
