## 一、用户反馈问题

### 问题1：key_params权重未在前端展示
用户指出：key_params权重设置为15.0，但在特征权重配置页面上没有展示。

**问题分析**：
- key_params_weight 存储在 `feature_weight_strategy` 配置中
- 前端只展示和保存 `feature_weight_config` 配置
- 导致用户无法通过前端页面修改 key_params 权重

### 问题2：菜单顺序不合理
用户指出：配置管理页面上的品牌关键词、设备参数配置、特征权重在设备信息录入阶段就需要使用，应该放在菜单树的最上面。

**问题分析**：
- 原菜单顺序：设备行识别 → 智能清理 → ... → 品牌关键词 → 设备参数配置 → 特征权重
- 不符合业务流程：应该先配置基础数据，再进行设备录入
- 用户体验不佳：需要滚动到底部才能找到最重要的配置

#### 1.2 后端修改

**文件**：`backend/modules/rule_generator.py`

**修改内容**：
1. 从 `feature_weight_config` 读取 `key_params_weight`
2. 在 `_assign_feature_weight` 方法中优先使用 `self.key_params_weight`

**代码修改**：
```python
# 初始化时读取配置
feature_weight_config = self.config.get('feature_weight_config', {})
self.key_params_weight = feature_weight_config.get('key_params_weight', 15.0)

# 分配权重时使用配置值
if key_params_features and feature in key_params_features:
    weight = self.key_params_weight if hasattr(self, 'key_params_weight') and self.key_params_weight else strategy.get('key_params_weight', 15.0)
    return weight
```

## 五、使用指南

### 1. 配置权重

**步骤**：
1. 打开浏览器：`http://localhost:3000`
2. 导航到：配置管理（左侧菜单第一项就是"品牌关键词"）
3. 点击"特征权重"（第3项）
4. 调整权重值：
   - 设备类型权重：20.0（高亮显示）
   - 关键参数权重：15.0（高亮显示）
   - 品牌权重：10.0
   - 型号权重：5.0
   - 通用参数权重：1.0
5. 点击"保存"按钮

### 2. 添加设备

**必填字段**：
- ✅ 品牌（brand）
- ✅ 设备类型（device_type）- **最重要**
- ✅ 设备名称（device_name）
- ✅ 规格型号（spec_model）

**推荐填写**：
- ✅ 关键参数（key_params）- **次重要**
  - 格式：`{"参数名": "参数值"}`
  - 示例：`{"量程": "0-2000ppm", "输出": "4-20mA"}`

**示例**：
```json
{
  "brand": "霍尼韦尔",
  "device_type": "温度传感器",
  "device_name": "温度传感器",
  "spec_model": "QAA2061",
  "key_params": {
    "量程": "0-50℃",
    "输出": "4-20mA"
  }
}
```

### 3. 重新生成规则

**当需要重新生成规则时**：
- 修改了权重配置
- 添加了新的设备
- 更新了设备信息

**执行命令**：
```bash
cd backend
python simple_regenerate_rules.py
```

## 八、常见问题

### Q1：为什么要将 key_params_weight 从 feature_weight_strategy 移到 feature_weight_config？

**A**：
- `feature_weight_config` 是前端页面展示和保存的配置
- `feature_weight_strategy` 是后端内部使用的策略配置
- 用户只能通过前端页面修改 `feature_weight_config`
- 将 `key_params_weight` 移到 `feature_weight_config` 后，用户可以通过前端页面修改

### Q2：为什么要调整菜单顺序？

**A**：
- 符合业务流程：先配置基础数据，再进行设备录入
- 提高效率：最重要的配置放在最上面，无需滚动
- 改善体验：用户打开配置管理页面，第一眼就能看到最重要的配置

### Q3：为什么要高亮显示设备类型和key_params权重？

**A**：
- 强调重要性：这两个是最核心的匹配参数
- 引导用户：提醒用户重点关注这两个配置
- 视觉区分：让用户一眼就能看到核心配置

### Q4：修改权重后需要做什么？

**A**：
1. 点击"保存"按钮保存配置
2. 运行 `python backend/simple_regenerate_rules.py` 重新生成规则
3. 重启后端服务（如果正在运行）
4. 测试匹配效果
