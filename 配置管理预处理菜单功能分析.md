## 详细功能分析

### 3.1 文本清理阶段

#### 3.1.1 噪音过滤 (noise-filter)
- **配置键**: `ignore_keywords`
- **数据类型**: 数组 `string[]`
- **执行时机**: 预处理第一步
- **主要功能**: 删除无关关键词（如"施工要求"、"验收"、"含税"等）
- **典型场景**: 
  - "霍尼韦尔温度传感器 施工要求：按图纸" → "霍尼韦尔温度传感器"
- **编辑器**: `IgnoreKeywordsEditor`

#### 3.1.2 元数据处理 (metadata)
- **配置键**: `text_cleaning.metadata_rules`
- **数据类型**: 数组 `string[]`
- **执行时机**: 文本清理阶段
- **主要功能**: 删除元数据标签（如"型号："、"品牌："），只保留值
- **典型场景**: 
  - "型号：ML-5000" → "ML-5000"
  - "品牌：霍尼韦尔" → "霍尼韦尔"
- **编辑器**: `MetadataRulesEditor`

#### 3.1.3 分隔符统一 (separator-unify)
- **配置键**: `text_cleaning.separator_mappings`
- **数据类型**: 数组 `[{from: string, to: string}]`
- **执行时机**: 文本清理阶段
- **主要功能**: 将各种分隔符统一为标准分隔符
- **典型场景**: 
  - "；" → ";"
  - "，" → ","
  - "　"（全角空格） → " "（半角空格）
- **编辑器**: `SeparatorMappingEditor`

---

### 3.2 归一化映射

#### 3.2.1 归一化映射 (normalization)
- **配置键**: `normalization_map`
- **数据类型**: 数组 `[{from: string, to: string}]`
- **执行时机**: 文本清理之后、特征提取之前
- **主要功能**: 字符级别的精准映射（删除单位符号、统一括号格式）
- **典型场景**: 
  - "℃" → ""
  - "°C" → ""
  - "（" → "("
  - "）" → ")"
- **编辑器**: `NormalizationEditor`

---

### 3.3 特征提取阶段

#### 3.3.1 处理分隔符 (separator-process)
- **配置键**: `feature_split_chars`
- **数据类型**: 数组 `string[]`
- **执行时机**: 特征提取阶段
- **主要功能**: 定义用于拆分特征的分隔符列表
- **典型场景**: 
  - 按 "+"、"|"、"；"、"\n" 拆分文本为特征列表
- **编辑器**: `SplitCharsEditor`
- **特殊说明**: 该编辑器还包含"智能拆分"配置（`intelligent_splitting`）

#### 3.3.2 复杂参数分解 (param-decompose)
- **配置键**: `intelligent_extraction.complex_parameter_decomposition`
- **数据类型**: 对象 `{enabled: boolean, complex_parameter_decomposition: {}}`
- **执行时机**: 特征提取阶段
- **主要功能**: 将组合参数拆分为独立特征
- **典型场景**: 
  - "4-20mA" → ["4-20", "mA"]
  - "温度+湿度传感器" → ["温度传感器", "湿度传感器"]
  - "DN15" → ["DN", "15"]
- **编辑器**: `ComplexParamEditor`

#### 3.3.3 智能拆分 (smart-split)
- **配置键**: `intelligent_splitting`
- **数据类型**: 对象 `{enabled: boolean, split_compound_words: boolean, ...}`
- **执行时机**: **匹配阶段**（注意：不是预处理阶段！）
- **主要功能**: 在匹配阶段自动拆分复合词和技术规格
- **典型场景**: 
  - "室内墙装" → ["室内", "墙装"]
  - "ntc 10k" → ["ntc", "10k"]
- **编辑器**: `IntelligentCleaningEditor`
- **重要说明**: 
  - 设备录入阶段：保持数据完整性，不拆分
  - 匹配阶段：智能拆分，提高灵活性

#### 3.3.4 单位删除 (unit-remove)
- **配置键**: `unit_removal`
- **数据类型**: 对象 `{enabled: boolean, units: string[]}`
- **执行时机**: **匹配阶段**（注意：不是预处理阶段！）
- **主要功能**: 在匹配阶段自动删除特征末尾的单位符号
- **典型场景**: 
  - "0-2000ppm" → "0-2000"
  - "4-20ma" → "4-20"
- **编辑器**: `AdvancedConfigEditor`
- **重要说明**: 仅在匹配阶段生效，设备录入阶段保持数据完整性

---

### 3.4 特征质量阶段

#### 3.4.1 质量评分 (quality-score)
- **配置键**: `feature_quality_scoring` 或 `intelligent_extraction.feature_quality_scoring`
- **数据类型**: 对象 `{enabled: boolean, scoring_rules: {}}`
- **执行时机**: 特征提取之后
- **主要功能**: 配置特征质量评分规则，过滤低质量特征
- **典型场景**: 
  - 过滤过短的特征（长度 < 2）
  - 过滤无意义的特征
- **编辑器**: `AdvancedConfigEditor`

#### 3.4.2 白名单 (whitelist)
- **配置键**: `feature_whitelist` 或 `intelligent_extraction.feature_quality_scoring.whitelist_features`
- **数据类型**: 数组 `string[]`
- **执行时机**: 特征质量评分阶段
- **主要功能**: 白名单中的特征即使质量评分低也会保留
- **典型场景**: 
  - 保护重要的短特征：如"水"、"气"、"阀"、"co"、"co2"
- **编辑器**: `AdvancedConfigEditor`

## 功能重复问题分析

### ❌ 问题1：分隔符配置重复

**涉及菜单**：
- 3.1.3 **分隔符统一** (separator-unify)
- 3.3.1 **处理分隔符** (separator-process)

**重复内容**：
- 两者都涉及分隔符的配置
- 用户容易混淆两者的区别

**实际区别**：
- **分隔符统一**：将各种分隔符**转换**为标准分隔符（如 "；" → ";"）
- **处理分隔符**：定义用于**拆分**文本的分隔符列表

**建议**：
- 保留"处理分隔符"，因为它是核心功能
- 将"分隔符统一"合并到"归一化映射"中，因为它们都是字符替换操作

---

### ❌ 问题2：智能拆分和复杂参数分解功能重叠

**涉及菜单**：
- 3.3.2 **复杂参数分解** (param-decompose)
- 3.3.3 **智能拆分** (smart-split)

**重复内容**：
- 两者都涉及文本拆分
- 功能有部分重叠

**实际区别**：
- **复杂参数分解**：针对特定模式的参数拆分（如"4-20mA"、"DN15"）
- **智能拆分**：更通用的复合词拆分（如"室内墙装"）

**执行时机差异**：
- **复杂参数分解**：特征提取阶段
- **智能拆分**：匹配阶段

**建议**：
- 可以考虑合并为一个"智能拆分"功能
- 或者明确区分：一个用于"参数拆分"，一个用于"词语拆分"

---

### ❌ 问题3：单位删除位置错误

**涉及菜单**：
- 3.3.4 **单位删除** (unit-remove)

**问题**：
- 该功能放在"预处理配置 → 特征提取"下
- 但实际执行时机是**匹配阶段**，不是预处理阶段

**建议**：
- 将"单位删除"移动到"匹配配置阶段"
- 或者在界面上明确标注"（匹配阶段生效）"

---

### ⚠️ 问题4：智能拆分位置错误

**涉及菜单**：
- 3.3.3 **智能拆分** (smart-split)

**问题**：
- 该功能放在"预处理配置 → 特征提取"下
- 但实际执行时机是**匹配阶段**，不是预处理阶段

**建议**：
- 将"智能拆分"移动到"匹配配置阶段"
- 或者在界面上明确标注"（匹配阶段生效）"

---

### ⚠️ 问题5：编辑器组件复用过度

**涉及菜单**：
- 3.3.4 **单位删除** (unit-remove) → 使用 `AdvancedConfigEditor`
- 3.4.1 **质量评分** (quality-score) → 使用 `AdvancedConfigEditor`
- 3.4.2 **白名单** (whitelist) → 使用 `AdvancedConfigEditor`

**问题**：
- 一个 `AdvancedConfigEditor` 组件同时处理3个不同的功能
- 组件内部逻辑复杂，难以维护
- 用户界面混乱，一个页面显示多个不相关的配置

**建议**：
- 为每个功能创建专用的编辑器组件
- 或者至少将"单位删除"独立出来（因为它属于匹配阶段）

## 总结

### 主要问题
1. ✅ **分隔符配置重复**："分隔符统一"和"处理分隔符"容易混淆
2. ✅ **执行阶段错位**："智能拆分"和"单位删除"实际在匹配阶段执行，但放在预处理配置下
3. ✅ **功能重叠**："复杂参数分解"和"智能拆分"有部分功能重叠
4. ✅ **组件复用过度**：`AdvancedConfigEditor` 同时处理多个不相关的功能

### 优化目标
1. 按实际执行阶段组织菜单
2. 消除功能重复和混淆
3. 为每个功能提供专用的编辑器
4. 提供清晰的功能说明和示例

### 推荐方案
采用**方案A**，按执行阶段重新组织菜单，将"智能拆分"和"单位删除"移动到"匹配配置阶段"。
