### 问题2：spec_model权重错误

**根本原因：**
- 在`rule_generator.py`的`_assign_feature_weight`方法中，spec_model检查在key_params检查之后
- 导致spec_model特征被key_params逻辑捕获，分配了15的权重而不是5

**修复方案：**
- 调整权重分配优先级，将spec_model检查移到key_params之前
- 确保spec_model特征使用`model_weight`配置值（5.0）

**修改文件：**
- `backend/modules/rule_generator.py` (第240-280行)

**权重分配优先级（修复后）：**
1. 设备类型（device_type字段）：20.0
2. **规格型号（spec_model字段）：5.0** ← 提前到这里
3. key_params参数：15.0
4. 设备类型关键词：15.0
5. 品牌（brand字段）：10.0
6. 品牌关键词：10.0
7. 型号特征：5.0
8. 重要参数：3.0
9. 通用参数：1.0

## 技术细节

### spec_model权重分配逻辑
```python
# 修复前（错误）
# spec_model检查在key_params之后，导致被key_params捕获

4. 品牌特征：10.0
5. 规格型号特征：15.0  ← 永远不会执行到这里

# 修复后（正确）
# spec_model检查在key_params之前，优先匹配
2. 规格型号特征：5.0  ← 优先检查
3. key_params参数特征：15.0
4. 设备类型关键词：15.0
5. 品牌特征：10.0
```

### "水"特征提取流程
```
1. 输入: key_params = {'介质': '水', ...}
2. 规则生成器遍历key_params
3. 对每个参数值调用preprocessor.preprocess()
4. 预处理器识别"介质"为metadata_keyword
5. 删除"介质:"前缀，保留"水"
6. 检查"水"是否在whitelist_features中
7. 由于在白名单中，跳过质量评分检查
8. "水"被添加到特征列表
9. 分配权重15.0（来自key_params）
```

### 白名单工作原理
```python
# 在text_preprocessor.py的_extract_features_with_detail方法中
whitelist_features = quality_scoring_config.get('whitelist_features', [])

for feature in enhanced_features:
    is_whitelisted = feature in whitelist_features
    quality_score = self._calculate_feature_quality(feature)
    
    # 白名单特征跳过质量评分检查
    if quality_enabled and quality_score < min_quality_score and not is_whitelisted:
        filter_reason = 'low_quality'
        # 过滤掉
    else:
        # 保留特征
```
