## 🔧 可选优化（如果匹配效果不理想）

### 优化1：调整特征权重

如果匹配准确率低于预期：

1. 进入"配置管理" → "特征权重配置"

2. 调整传感器相关特征的权重：
   ```
   品牌（霍尼韦尔）：10-15
   设备类型：20-25
   规格型号：5-10
   检测对象：15-20
   输出信号：10-15
   温度信号类型：10-15
   湿度信号类型：10-15
   ```

3. 保存配置

4. 重新生成规则：
   - 进入"规则管理"
   - 选择需要更新的规则
   - 点击"批量重新生成"

### 优化2：补充同义词

如果发现某些表述无法匹配：

1. 进入"配置管理" → "高级配置" → "同义词映射"

2. 添加新的同义词对，例如：
   ```json
   {
     "温湿度": ["温度湿度", "温湿"],
     "传感器": ["探头", "探测器", "检测器"]
   }
   ```

3. 保存配置

### 优化3：更新白名单

如果重要特征被过滤：

1. 进入"配置管理" → "高级配置" → "特征白名单"

2. 添加新的白名单特征

3. 保存配置后重新生成规则

## ⚠️ 常见问题

### Q1: 导入时提示"不是数据库模式"？

**A**: 确认系统配置：
1. 检查 `backend/config.py` 中的 `DATABASE_TYPE`
2. 应该设置为 `'database'` 而不是 `'json'`
3. 修改后重启后端服务

### Q2: 导入后设备数量不对？

**A**: 检查导入日志：
1. 查看后端控制台输出
2. 查找失败的设备记录
3. 检查失败原因（通常是数据格式问题）

### Q3: 匹配规则没有自动生成？

**A**: 确认导入选项：
1. 导入时必须勾选"自动生成匹配规则"
2. 如果忘记勾选，可以手动批量生成：
   - 进入"规则管理"
   - 选择所有设备
   - 点击"批量生成规则"

### Q4: 部分设备匹配不准确？

**A**: 逐步优化：
1. 先调整特征权重（最有效）
2. 补充同义词映射
3. 更新白名单特征
4. 手动调整个别规则

### Q5: 如何批量更新设备信息？

**A**: 重新导入：
1. 修改Excel文件中的数据
2. 重新执行批量导入
3. 系统会自动更新已存在的设备（根据规格型号匹配）
