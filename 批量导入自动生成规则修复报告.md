## 问题诊断

通过代码检查发现，`backend/app.py` 中有多处使用了旧的 `RuleGenerator` 构造函数签名：

```python
# 旧的构造函数（错误）
rule_generator = RuleGenerator(preprocessor, default_threshold=default_threshold, config=config)
rule_generator = RuleGenerator(config=config, preprocessor=preprocessor)
```

但是在之前的架构改进中，我们已经修改了 `RuleGenerator` 的构造函数，移除了 `preprocessor` 参数：

```python
# 新的构造函数（正确）
rule_generator = RuleGenerator(config=config, default_threshold=default_threshold)
```

## 用户操作指南

### 批量导入设备

1. 准备Excel文件，包含以下列：
   - 品牌（必需）
   - 设备类型（可选）
   - 设备名称（必需）
   - 规格型号（必需）
   - 单价（必需）
   - 其他列作为关键参数

2. 在设备管理页面点击"批量导入"

3. 选择Excel文件并上传

4. 系统会自动：
   - 解析Excel文件
   - 导入设备到数据库
   - 为每个设备生成匹配规则

### 为单个设备生成规则

1. 在设备列表中找到没有规则的设备

2. 点击该设备行的"生成规则"按钮

3. 系统会自动生成规则并刷新列表

### 批量生成规则

1. 在配置管理页面点击"重新生成规则"按钮

2. 确认操作

3. 系统会为所有设备重新生成规则
