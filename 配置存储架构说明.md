## 🗂️ 配置存储位置

### 1. 数据库配置（主要存储）

**位置**：`data/devices.db` → `configs` 表

**访问方式**：
```python
from modules.database_loader import DatabaseLoader
from modules.database import DatabaseManager

db_manager = DatabaseManager("sqlite:///data/devices.db")
db_loader = DatabaseLoader(db_manager)

# 获取单个配置
config = db_loader.get_config_by_key('device_params')

# 获取所有配置
all_configs = db_loader.load_config()
```

**存储的配置键**（31个）：
- `device_params` - 设备参数配置（原 device_params.yaml）
- `intelligent_extraction` - 智能提取配置
- `device_type_keywords` - 设备类型关键词
- `brand_keywords` - 品牌关键词
- `metadata_keywords` - 元数据关键词
- `whitelist_features` - 特征白名单
- `feature_weight_config` - 特征权重配置
- `synonym_map` - 同义词映射
- 等等...

### 2. 静态配置文件（运行时配置）

**位置**：`data/static_config.json`

**用途**：
- ✅ **运行时配置**：系统启动时加载，提供默认配置
- ✅ **配置模板**：包含所有配置项的完整结构
- ✅ **向后兼容**：支持旧代码直接读取JSON文件

**访问方式**：
```python
import json

with open('data/static_config.json', 'r', encoding='utf-8') as f:
    config = json.load(f)
```

**包含的配置**：
- 所有数据库配置的副本
- 全局配置（global_config）
- 性能配置（performance_config）
- UI配置（ui_config）
- 等等...

## 🔄 配置优先级和使用场景

### 场景1：前端配置管理页面

**流程**：前端 → API → 数据库

```
用户修改配置
  ↓
前端发送 POST /api/config/save
  ↓
后端保存到数据库 configs 表
  ↓
前端刷新显示（从数据库读取）
```

**代码示例**：
```python
# 后端 API
@app.route('/api/config/save', methods=['POST'])
def save_config():
    data = request.json
    config_key = data['config_key']
    config_value = data['config_value']
    
    db_loader.update_config(config_key, config_value)
    return jsonify({'success': True})
```

### 场景2：设备录入和规则生成

**流程**：代码 → 数据库

```python
# 1. 初始化
db_manager = DatabaseManager("sqlite:///data/devices.db")
db_loader = DatabaseLoader(db_manager)

# 2. 加载配置
config = db_loader.load_config()  # 从数据库加载所有配置

# 3. 使用配置
feature_extractor = DeviceFeatureExtractor(config)

```

### 场景3：测试脚本和工具

**流程**：代码 → static_config.json

```python
# 快速测试，直接读取JSON文件
with open('data/static_config.json', 'r', encoding='utf-8') as f:
    config = json.load(f)
```

**为什么使用JSON文件？**
- ✅ 简单快速，不需要初始化数据库

- ✅ 向后兼容旧代码

## 📊 配置同步机制

### 数据库 ↔ static_config.json

**问题**：两个地方存储配置，如何保持同步？

**答案**：
1. **数据库是主存储**：前端修改配置时，保存到数据库
2. **static_config.json是副本**：定期从数据库导出更新
3. **手动同步**：修改数据库配置后，需要手动更新JSON文件

**同步脚本**（建议创建）：
```python
# sync_config_to_json.py
import json
from modules.database import DatabaseManager
from modules.database_loader import DatabaseLoader

db_manager = DatabaseManager("sqlite:///data/devices.db")
db_loader = DatabaseLoader(db_manager)

# 从数据库加载所有配置
db_config = db_loader.load_config()

# 读取现有JSON文件
with open('data/static_config.json', 'r', encoding='utf-8') as f:
    json_config = json.load(f)

# 更新JSON配置（保留JSON中独有的配置）
json_config.update(db_config)

# 保存回JSON文件
with open('data/static_config.json', 'w', encoding='utf-8') as f:
    json.dump(json_config, f, ensure_ascii=False, indent=2)

print("✅ 配置同步完成")
```

3. ✅ **定期同步**：修改数据库后，同步到JSON文件
4. ✅ **删除遗留文件**：`intelligent_extraction_config.json` 可以删除
5. ✅ **统一访问方式**：通过 `DatabaseLoader` 访问配置
