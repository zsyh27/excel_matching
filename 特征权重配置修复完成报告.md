# 特征权重配置修复完成报告

## 📋 问题描述

用户发现蝶阀设备的特征提取有问题，设备有7个关键参数（key_params），但特征只提取了3个：
- v8bfw16-050 (型号) - 权重5
- 霍尼韦尔 (品牌) - 权重1
- dn50 蝶阀 对夹式 (设备名称) - 权重1

**缺失的特征**：
- 设备类型（蝶阀）
- 7个关键参数（公称通径、公称压力、连接方式、阀体材质、密封材质、适用介质、介质温度）

---

## 🔍 问题分析

### 根本原因

数据库中的 `feature_weight_config` 配置错误：

**错误的配置**：
```json
{
  "brand_weight": 1,           // ❌ 应该是10
  "device_type_weight": 15,    // ❌ 应该是20
  "key_params_weight": 5,      // ❌ 应该是15
  "model_weight": 5,           // ✅ 正确
  "parameter_weight": 1        // ✅ 正确
}
```

**正确的配置**：
```json
{
  "brand_weight": 10,          // ✅ 品牌字段权重
  "device_type_weight": 20,    // ✅ 设备类型字段权重（最高）
  "key_params_weight": 15,     // ✅ key_params参数权重（次高）
  "model_weight": 5,           // ✅ 规格型号字段权重
  "parameter_weight": 1        // ✅ 通用参数权重
}
```

### 影响范围

- ✅ 特征提取器代码正确（`DeviceFeatureExtractor`）
- ❌ 权重配置错误（数据库 `feature_weight_config`）
- ❌ 所有606个蝶阀设备的规则权重错误

---

## ✅ 修复方案

### 步骤1：修复权重配置

运行 `fix_feature_weight_config.py` 脚本，更新数据库中的权重配置：

```bash
python fix_feature_weight_config.py
```

**修复结果**：
- brand_weight: 1 → 10 ✅
- device_type_weight: 15 → 20 ✅
- key_params_weight: 5 → 15 ✅

### 步骤2：清除Python缓存

```bash
rm -r backend/__pycache__
rm -r backend/modules/__pycache__
```

### 步骤3：重新生成所有规则

运行 `regenerate_all_butterfly_rules.py` 脚本，重新生成606个蝶阀设备的规则：

```bash
python regenerate_all_butterfly_rules.py
```

**生成结果**：
- 总设备数: 606
- 成功生成: 606
- 失败: 0
- 成功率: 100%
