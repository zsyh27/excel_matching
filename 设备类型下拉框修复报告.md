# 设备类型下拉框修复报告

## 问题描述
用户反馈：添加设备页面中的设备类型下拉框只有15个设备类型可以选择，而配置管理页面显示有36个设备类型。

## 问题分析

### 数据流程
1. **配置管理页面**：调用 `/api/config` → 从数据库读取 `device_params` → 显示36个设备类型 ✅
2. **添加设备页面**：调用 `/api/device-types` → 从YAML文件读取 → 只显示15个设备类型 ❌

### 根本原因
后端 `/api/device-types` API 端点从 `backend/config/device_params.yaml` 文件读取设备类型配置，而该YAML文件只定义了15个设备类型。

数据库中的 `device_params` 配置有36个设备类型，但该API没有使用数据库数据。

### YAML文件中的15个设备类型
1. CO2传感器
2. 座阀
3. 温度传感器
4. 压力传感器
5. 执行器
6. 湿度传感器
7. 流量传感器
8. 液位传感器
9. 电动阀
10. 变频器
11. 控制器
12. 风机盘管
13. 水泵
14. 差压传感器
15. 电磁阀

### 数据库中的36个设备类型
包含上述15个，另外还有：
- DDC、传感器、压差开关、变送器、开关、探测器、控制柜、模块
- 水阀、流量计、液位开关、温控器、球阀、电源、继电器、网关
- 蝶阀、调节阀、阀门、风机、风阀

## 解决方案

### 修改内容
修改 `backend/app.py` 中的 `/api/device-types` 端点，从数据库读取配置而不是从YAML文件读取。

**修改前**：
```python
@app.route('/api/device-types', methods=['GET'])
def get_device_types():
    try:
        import yaml
        config_path = 'config/device_params.yaml'
        # ... 读取YAML文件
        with open(config_path, 'r', encoding='utf-8') as f:
            config = yaml.safe_load(f)
        device_types = list(config['device_types'].keys())
        # ... 返回15个设备类型
```

**修改后**：
```python
@app.route('/api/device-types', methods=['GET'])
def get_device_types():
    try:
        # 从数据库读取完整配置
        config = data_loader.load_config()
        device_params = config.get('device_params', {})
        device_types = list(device_params.keys())
        # ... 返回36个设备类型
```
