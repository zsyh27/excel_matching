## 权重配置策略

### 当前配置
```
device_type_weight:  20.0  (最高 - 设备类型)
key_params_weight:   15.0  (次高 - 关键参数)
brand_weight:        10.0  (中高 - 品牌)
model_weight:         5.0  (中等 - 型号)
parameter_weight:     1.0  (最低 - 普通参数)
```

### 设计理念

1. **设备类型最重要**（20.0）
   - 确保设备大类匹配正确
   - 避免将阀门匹配成传感器

2. **关键参数次之**（15.0）
   - 通径、通数、阀体类型等核心参数
   - 这些参数决定设备的具体规格

3. **品牌适中**（10.0）
   - 品牌识别重要，但不是决定性因素
   - 允许一定的品牌灵活性

4. **型号较低**（5.0）
   - 避免过度依赖具体型号
   - 支持同系列产品匹配

5. **普通参数最低**（1.0）
   - 作为辅助匹配条件
   - 不影响主要匹配逻辑

## 技术要点

### 1. 权重计算机制

**规则生成时**（后端）：
```python
# 智能分配权重
for feature in features:
    if feature == device.device_type:
        weight = 20.0
    elif feature in key_params:
        weight = 15.0
    elif feature == device.brand:
        weight = 10.0
    # ...
    feature_weights[feature] = weight
```

**总权重计算**（前端+后端）：
```python
# 简单相加
total_weight = sum(feature_weights.values())
```

### 2. 前端权重配置加载

```javascript
// 从后端API加载
const response = await getConfig()
const config = response.data.config.feature_weight_config

// 更新本地状态
weightConfig.value = {
  device_type_weight: config.device_type_weight,
  key_params_weight: config.key_params_weight,
  // ...
}
```

### 3. 自动权重调整

```javascript
// 特征类型改变时
const onFeatureTypeChange = (feature) => {
  // 自动设置为标准权重
  feature.weight = getStandardWeight(feature.type)
}
```

## 注意事项

1. **通数参数的使用**
   - 优先填写在"通数"字段（key_params）
   - 不要填写在"详细参数"中
   - 这样可以享受15.0的高权重

2. **设备类型必填**
   - 所有新设备必须选择设备类型
   - 旧设备可能没有设备类型（兼容性）

3. **权重配置**
   - 可以在"配置管理 → 特征权重配置"中调整
   - 调整后需要重新生成规则

4. **阈值设置**
   - 建议设置为总权重的40%-50%
   - 过低容易误匹配，过高容易匹配失败
