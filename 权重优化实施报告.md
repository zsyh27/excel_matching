## 一、实施内容

### 1. 代码修改

#### 文件：`backend/modules/rule_generator.py`

**修改方法**：`_assign_feature_weight`

**修改内容**：
- ✅ 设备类型（device_type字段）：硬编码3.0 → 使用配置值（默认20.0）
- ✅ 品牌（brand字段）：硬编码3.0 → 使用配置值（默认10.0）
- ✅ 型号：硬编码6.0 → 使用配置值（默认5.0）
- ✅ key_params参数：硬编码2.5 → 使用策略配置（默认15.0）
- ✅ 通用参数：硬编码1.0 → 使用配置值（默认1.0）

**关键改进**：
```python
# 修改前（硬编码）
if device and device.device_type:
    if feature == device.device_type:
        return 3.0  # ❌ 硬编码

# 修改后（使用配置）
if device and device.device_type:
    if feature == device.device_type:
        weight = self.device_type_weight if hasattr(self, 'device_type_weight') and self.device_type_weight else strategy.get('device_type_weight', 20.0)
        return weight  # ✅ 使用配置值
```

## 二、权重优先级

### 最终权重体系

| 特征类型 | 权重值 | 优先级 | 说明 |
|---------|--------|--------|------|
| 设备类型（device_type字段） | **20.0** | 最高 | 核心匹配参数 |
| key_params参数 | **15.0** | 次高 | 核心匹配参数 |
| 设备类型关键词 | **15.0** | 高 | 识别设备类型 |
| 品牌（brand字段） | **10.0** | 中高 | 重要识别特征 |
| 品牌关键词 | **10.0** | 中高 | 识别品牌 |
| 型号 | **5.0** | 中 | 唯一标识 |
| 重要参数 | **3.0** | 中等 | 带单位的数值范围 |
| 通用参数 | **1.0** | 低 | 区分度较低 |

### 权重比例

```
设备类型 : key_params : 品牌 : 型号 : 参数
  20.0   :   15.0     : 10.0 : 5.0  : 1.0
   20    :    15      :  10  :  5   :  1
```

## 三、预期效果

### 1. 匹配准确率提升

**示例1：温度传感器匹配**

**Excel输入**：`霍尼韦尔 温度传感器 QAA2061 0-50℃`

**提取特征**：
- "霍尼韦尔"（品牌）
- "温度传感器"（设备类型）
- "qaa2061"（型号）
- "0-50"（参数）

**匹配计算（修改前）**：
```
规则1（霍尼韦尔温度传感器QAA2061）：
- "霍尼韦尔"：1.0
- "温度传感器"：3.0
- "qaa2061"：1.0
- "0-50"：3.0
总得分：8.0
```

**匹配计算（修改后）**：
```
规则1（霍尼韦尔温度传感器QAA2061）：
- "霍尼韦尔"：10.0
- "温度传感器"：20.0
- "qaa2061"：5.0
- "0-50"：1.0
总得分：36.0
```

**效果**：
- 得分提升：8.0 → 36.0（提升4.5倍）
- 设备类型权重占比：37.5% → 55.6%（提升18.1%）
- 核心参数（设备类型+品牌）占比：50% → 83.3%（提升33.3%）

### 2. 错误匹配减少

**场景**：Excel输入 `西门子 温度传感器`，但参数不匹配

**修改前**：
- 参数权重高（3.0），可能因为参数匹配而误匹配
- 设备类型权重低（3.0），核心参数权重不足

**修改后**：
- 设备类型权重高（20.0），确保设备类型优先匹配
- 参数权重低（1.0），减少参数误匹配的影响
- 即使参数不完全匹配，只要设备类型和品牌匹配，也能正确识别

### 3. key_params参数重要性提升

**场景**：设备库中填写了 key_params（如通径、量程等）

**修改前**：
- key_params权重：2.5（较低）
- 与普通参数区分度不大

**修改后**：
- key_params权重：15.0（次高）
- 与设备类型同等重要
- 确保核心参数在匹配中起决定性作用

## 六、风险评估

### 潜在风险

1. **已有规则需要重新生成**
   - 风险：旧规则使用旧权重，可能导致匹配不一致
   - 缓解：立即执行批量重新生成规则

2. **权重过高可能导致过拟合**
   - 风险：设备类型权重过高，可能忽略其他重要特征
   - 缓解：通过测试验证，必要时微调

3. **配置不兼容**
   - 风险：旧版本代码可能不支持新配置
   - 缓解：代码已修改为向后兼容，使用默认值兜底

### 回滚方案

如果出现问题，可以回滚配置：

```python
# backend/rollback_weight_config.py
import sqlite3
import json

db_path = '../data/devices.db'
conn = sqlite3.connect(db_path)
cursor = conn.cursor()

# 回滚到旧配置
old_config = {
    "brand_weight": 1,
    "model_weight": 1,
    "device_type_weight": 3,
    "parameter_weight": 3
}

cursor.execute("""
    UPDATE configs 
    SET config_value = ? 
    WHERE config_key = 'feature_weight_config'
""", (json.dumps(old_config),))

conn.commit()
conn.close()
```
