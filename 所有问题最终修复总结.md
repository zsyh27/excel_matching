## 修复方案

### 修复1：批量导入API

**文件：** `backend/app.py` 第1950行

```python
# 如果需要自动生成规则
if auto_generate_rules:
    try:
        from modules.rule_generator import RuleGenerator
        from modules.text_preprocessor import TextPreprocessor
        
        # 初始化预处理器和规则生成器
        preprocessor = TextPreprocessor(config=config)
        rule_gen = RuleGenerator(config=config, preprocessor=preprocessor)
        rule = rule_gen.generate_rule(device)
        
        if rule:
            data_loader.loader.save_rule(rule)  # ✅ 修复：使用save_rule
            generated_rules.append(device_id)
            logger.info(f"规则生成成功: {device_id}")
        else:
            logger.warning(f"规则生成失败: {device_id} - generate_rule返回None")
    except Exception as e:
        logger.error(f"生成规则失败 {device_id}: {e}")
        import traceback
        logger.error(traceback.format_exc())
```

## 关于"水"被过滤的问题

**实际情况：** "水"**不会**被过滤

检查预处理器配置：
```python
min_feature_length: 1  # 最小长度为1，"水"（长度1）不会被过滤
```

如果您的设备关键参数中有"介质: 水"，它应该会出现在规则中。

**可能的原因：**
1. 您查看的设备关键参数中是"介质: 蒸汽"，不是"水"
2. 前端显示的是旧数据（缓存）
3. 后端服务没有重启
