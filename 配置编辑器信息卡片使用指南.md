# 配置编辑器信息卡片使用指南

## 概述

为了帮助用户更好地理解每个配置项的作用和使用方法，我们为所有配置编辑器添加了统一的信息卡片组件 `ConfigInfoCard`。

该组件会显示：
1. **流程位置**：该配置在整个流程中的哪个阶段生效
2. **使用说明**：如何使用该配置
3. **配置示例**：具体的配置示例
4. **注意事项**：使用时需要注意的事项

## 组件位置

- **组件文件**：`frontend/src/components/ConfigManagement/ConfigInfoCard.vue`
- **配置信息映射**：`frontend/src/config/configInfoMap.js`

## 使用方法

### 方法一：使用配置信息映射（推荐）

1. 在编辑器组件中导入 `ConfigInfoCard` 和配置信息：

```vue
<script>
import { ref, watch } from 'vue'
import ConfigInfoCard from './ConfigInfoCard.vue'
import { getConfigInfo } from '../../config/configInfoMap'

export default {
  name: 'YourEditor',
  components: {
    ConfigInfoCard
  },
  setup() {
    // 获取配置信息
    const configInfo = getConfigInfo('your-config-id')
    
    return {
      configInfo
    }
  }
}
</script>
```

2. 在模板中使用：

```vue
<template>
  <div class="your-editor">
    <div class="editor-header">
      <h2>配置标题</h2>
      <p class="description">配置描述</p>
      
      <ConfigInfoCard
        v-if="configInfo"
        :stage="configInfo.stage"
        :stage-icon="configInfo.stageIcon"
        :stage-name="configInfo.stageName"
        :stage-description="configInfo.stageDescription"
        :usage-text="configInfo.usageText"
        :examples="configInfo.examples"
        :notes="configInfo.notes"
      />
    </div>
    
    <!-- 编辑器内容 -->
  </div>
</template>
```

### 方法二：直接传递属性

如果不想使用配置信息映射，可以直接传递属性：

```vue
<template>
  <div class="your-editor">
    <div class="editor-header">
      <h2>配置标题</h2>
      <p class="description">配置描述</p>
      
      <ConfigInfoCard
        stage="preprocessing"
        stage-icon="🔍"
        stage-name="预处理配置"
        stage-description="此配置在特征提取前生效..."
        usage-text="配置使用说明..."
        :examples="['示例1', '示例2', '示例3']"
        :notes="['注意事项1', '注意事项2']"
      />
    </div>
  </div>
</template>
```

### 方法三：使用插槽（最灵活）

如果需要更复杂的内容，可以使用插槽：

```vue
<template>
  <ConfigInfoCard
    stage="preprocessing"
    stage-icon="🔍"
    stage-name="预处理配置"
    stage-description="此配置在特征提取前生效..."
  >
    <template #usage>
      <ul>
        <li><strong>功能1</strong>：详细说明</li>
        <li><strong>功能2</strong>：详细说明</li>
      </ul>
    </template>
    
    <template #examples>
      <div class="custom-examples">
        <!-- 自定义示例内容 -->
      </div>
    </template>
    
    <template #notes>
      <div class="custom-notes">
        <!-- 自定义注意事项 -->
      </div>
    </template>
  </ConfigInfoCard>
</template>
```

## 流程阶段说明

### 1. 设备信息录入前配置 (pre-entry)
- **阶段图标**：📝
- **生效时机**：在设备信息录入前
- **包含配置**：
  - 品牌关键词 (brand-keywords)
  - 设备参数配置 (device-params)
  - 特征权重 (feature-weights)

### 2. 数据导入阶段 (import)
- **阶段图标**：📥
- **生效时机**：Excel数据导入时
- **包含配置**：
  - 设备行识别 (device-row)

### 3. 预处理配置 (preprocessing)
- **阶段图标**：🔍
- **生效时机**：特征提取和处理阶段
- **包含配置**：
  - 文本清理：噪音过滤 (noise-filter)、元数据处理 (metadata)
  - 归一化映射 (normalization)
  - 特征提取：处理分隔符 (separator-process)、复杂参数分解 (param-decompose)
  - 特征质量：质量评分 (quality-score)、白名单 (whitelist)

### 4. 匹配配置阶段 (matching)
- **阶段图标**：🎯
- **生效时机**：设备匹配阶段
- **包含配置**：
  - 同义词映射 (synonym-map)
  - 设备类型关键词 (device-type)
  - 匹配增强：智能拆分 (smart-split)、单位删除 (unit-remove)
  - 匹配阈值 (match-threshold)

### 5. 全局配置 (global)
- **阶段图标**：⚙️
- **生效时机**：影响整个系统
- **包含配置**：
  - 全局配置 (global-settings)

## 配置信息映射文件

所有配置项的信息都定义在 `frontend/src/config/configInfoMap.js` 文件中。

### 添加新配置项

如果需要添加新的配置项，在 `configInfoMap.js` 中添加：

```javascript
export const CONFIG_INFO_MAP = {
  // ... 其他配置
  
  'your-new-config': {
    stage: 'preprocessing',  // 流程阶段
    stageIcon: '🔍',         // 阶段图标
    stageName: '预处理配置',  // 阶段名称
    stageDescription: '此配置在特征提取前生效...',  // 阶段描述
    usageText: '配置使用说明...',  // 使用说明
    examples: [  // 配置示例
      '示例1',
      '示例2',
      '示例3'
    ],
    notes: [  // 注意事项
      '注意事项1',
      '注意事项2'
    ]
  }
};
```

## 更新步骤

对于每个编辑器，按以下步骤更新：

1. **导入组件和配置信息**：
   ```javascript
   import ConfigInfoCard from './ConfigInfoCard.vue'
   import { getConfigInfo } from '../../config/configInfoMap'
   ```

2. **注册组件**：
   ```javascript
   components: {
     ConfigInfoCard
   }
   ```

3. **获取配置信息**：
   ```javascript
   setup() {
     const configInfo = getConfigInfo('config-id')
     return { configInfo }
   }
   ```

4. **在模板中添加信息卡片**：
   ```vue
   <ConfigInfoCard
     v-if="configInfo"
     v-bind="configInfo"
   />
   ```

5. **删除旧的信息展示代码**（如果有）

## 样式说明

`ConfigInfoCard` 组件已经包含完整的样式，不需要在编辑器中添加额外的样式。

如果需要自定义样式，可以在编辑器的 `<style>` 中覆盖：

```vue
<style scoped>
/* 自定义信息卡片样式 */
:deep(.config-info-card) {
  margin-bottom: 30px;
}
</style>
```

## 总结

通过使用 `ConfigInfoCard` 组件和配置信息映射，我们可以：

1. **统一信息展示**：所有编辑器使用相同的信息卡片样式
2. **集中管理信息**：所有配置信息集中在 `configInfoMap.js` 中
3. **易于维护**：修改配置信息只需更新映射文件
4. **提升用户体验**：用户可以清楚地了解每个配置的作用和使用方法
