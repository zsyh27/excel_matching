# 配置管理页面信息卡片功能实现报告

## 实现概述

为配置管理页面的所有编辑器添加了统一的信息卡片组件，帮助用户了解：
1. 该配置在整个流程中的位置
2. 该配置的使用方法
3. 配置示例和注意事项

## 实现内容

### 1. 创建信息卡片组件

**文件**：`frontend/src/components/ConfigManagement/ConfigInfoCard.vue`

**功能**：
- 显示流程位置（5个阶段，不同颜色徽章）
- 显示使用说明
- 显示配置示例
- 显示注意事项
- 支持插槽自定义内容

**流程阶段徽章颜色**：
- 📝 设备信息录入前配置：蓝色 (#e3f2fd)
- 📥 数据导入阶段：紫色 (#f3e5f5)

- 🎯 匹配配置阶段：绿色 (#e8f5e9)
- ⚙️ 全局配置：粉色 (#fce4ec)

## 使用方法

### 方法一：使用配置信息映射（推荐）

```vue
<script>
import ConfigInfoCard from './ConfigInfoCard.vue'
import { getConfigInfo } from '../../config/configInfoMap'

export default {
  components: { ConfigInfoCard },
  setup() {
    const configInfo = getConfigInfo('your-config-id')
    return { configInfo }
  }
}
</script>

<template>
  <ConfigInfoCard v-if="configInfo" v-bind="configInfo" />
</template>
```

### 方法二：直接传递属性

```vue
<ConfigInfoCard
  stage="preprocessing"
  stage-icon="🔍"
  stage-name="预处理配置"
  stage-description="此配置在特征提取前生效..."
  usage-text="配置使用说明..."
  :examples="['示例1', '示例2']"
  :notes="['注意1', '注意2']"
/>
```

### 方法三：使用插槽（最灵活）

```vue
<ConfigInfoCard
  stage="preprocessing"
  stage-icon="🔍"
  stage-name="预处理配置"
  stage-description="此配置在特征提取前生效..."
>
  <template #usage>
    <ul>
      <li><strong>功能1</strong>：详细说明</li>
      <li><strong>功能2</strong>：详细说明</li>
    </ul>
  </template>
</ConfigInfoCard>
```

## 技术要点

### 组件设计

1. **Props 设计**：
   - 必需属性：stage, stageName, stageDescription
   - 可选属性：stageIcon, usageText, examples, notes
   - 支持插槽：usage, examples, notes

2. **样式设计**：
   - 使用不同颜色区分流程阶段
   - 清晰的视觉层次
   - 响应式布局

3. **可扩展性**：
   - 支持插槽自定义内容
   - 配置信息集中管理
   - 易于添加新配置项

### 配置信息管理

1. **集中管理**：
   - 所有配置信息在 `configInfoMap.js` 中
   - 统一的数据结构
   - 易于维护和更新

2. **工具函数**：
   - `getConfigInfo(configId)` - 获取单个配置信息
   - `getAllConfigIds()` - 获取所有配置ID

3. **类型安全**：
   - 使用 JSDoc 注释
   - 清晰的数据结构定义
