# 品牌关键词统一配置完成报告

## 问题描述

在配置管理页面中，品牌关键词在两个地方维护：

1. **品牌关键词配置页面** (`/config-management?section=brand-keywords`)
   - 配置键：`brand_keywords`
   - 数据类型：`list`（数组）
   - 用途：设备信息录入前配置，用于特征识别和质量评分

2. **辅助信息模式页面** (`/config-management?section=auxiliary-info`)
   - 配置路径：`intelligent_extraction.auxiliary_extraction.brand.keywords`
   - 用途：智能特征提取阶段的品牌识别

这导致了以下问题：
- ❌ 重复维护，容易不一致
- ❌ 用户困惑，不知道应该在哪里维护
- ❌ 配置冗余，增加维护成本

## 解决方案

### 统一配置架构

**单一数据源原则**：`brand_keywords` 作为唯一的品牌关键词配置

```
brand_keywords (list)  ← 唯一数据源
    ↓
    ├─→ 品牌关键词配置页面（可编辑）
    ├─→ 辅助信息模式页面（只读显示）
    └─→ 后端辅助信息提取器（读取使用）
```

#### 2. 后端配置（api_handler.py）

后端已经正确实现，从 `brand_keywords` 读取：

```python
# 为辅助信息提取器准备配置
auxiliary_config = ie_config.get('auxiliary_extraction', {})
# 如果没有auxiliary_extraction配置，从其他配置中获取
if not auxiliary_config:
    auxiliary_config = {
        'brand': {
            'enabled': True,
            'keywords': config.get('brand_keywords', [])  # ← 从 brand_keywords 读取
        },
        # ...
    }
```

#### 3. 配置保存逻辑

**修改后的保存逻辑**：

  brand: {

    // keywords 字段不保存，由 brand_keywords 配置统一管理
  },
  medium: config.value.medium,
  model: config.value.model

## 用户指南

### 如何管理品牌关键词？

1. **添加/删除品牌**：
   - 进入"配置管理" → "品牌关键词配置"
   - 在该页面添加或删除品牌
   - 保存配置

2. **查看品牌使用情况**：
   - 进入"配置管理" → "辅助信息模式"
   - 查看当前使用的品牌列表（只读）
   - 点击链接可跳转到品牌关键词配置页面

3. **启用/禁用品牌识别**：
   - 进入"配置管理" → "辅助信息模式"
   - 使用"品牌关键词"开关启用或禁用品牌识别功能
