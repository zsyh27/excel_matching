# 元数据关键词配置使用指南

## 配置位置

**配置管理 → 高级配置 → 元数据关键词**

## 功能说明

元数据关键词用于识别字段名称,避免将其作为匹配特征。

### 工作原理

当系统遇到以下格式的文本时:
- `型号:QAA2061` → 提取 "QAA2061"
- `品牌:霍尼韦尔` → 提取 "霍尼韦尔"
- `温度:25℃` → 提取 "25℃"

系统会识别冒号前的部分(如"型号"、"品牌"、"温度")是否在元数据关键词列表中:
- 如果在列表中,则删除该前缀,只保留值部分
- 如果不在列表中,则保留完整文本

## ⚠️ 重要更新 (2026-03-06)

### 配置分离已实现

系统已经实现了设备录入和匹配阶段的配置分离:

- **设备录入阶段** (`mode='device'`): 跳过元数据关键词处理,保持设备类型完整性
- **匹配阶段** (`mode='matching'`): 应用元数据关键词处理,智能清理混乱输入

### 现在可以安全添加的关键词

✅ **现在可以放心添加以下关键词**,它们只会在匹配阶段生效:
- `温度` - 用于处理"温度:25℃"这样的混乱输入
- `压力` - 用于处理"压力:1.6MPa"这样的混乱输入
- `湿度` - 用于处理"湿度:0-100%"这样的混乱输入
- `流量` - 用于处理"流量:0-100L/h"这样的混乱输入
- `功率` - 用于处理"功率:100W"这样的混乱输入

这些关键词**不会影响设备录入阶段**,设备类型如"温度传感器"、"压力传感器"等会被完整保留。

详细说明请参考: [设备录入与匹配阶段配置分离完成报告](./设备录入与匹配阶段配置分离完成报告.md)

## 使用建议

### 1. 添加新关键词前的检查清单

在添加新的元数据关键词之前,请确认:

- [ ] 这个词是字段名称,而不是字段值
- [ ] 这个词不是设备类型的组成部分
- [ ] 这个词不是设备名称的组成部分
- [ ] 这个词不是品牌名称的组成部分
- [ ] 这个词不是型号的组成部分

### 2. 测试新关键词的影响

添加新关键词后,建议:

1. 重新生成受影响设备的规则
2. 检查设备类型是否被正确提取
3. 验证匹配权重是否合理

### 3. 定期审查配置

建议每季度审查一次元数据关键词配置:
- 检查是否有不应该在列表中的词汇
- 检查是否有新的字段名称需要添加
- 验证配置是否影响了设备类型的提取

## 常见问题

### Q1: 为什么"温度"不应该在元数据关键词列表中?

A: 因为"温度"是"温度传感器"的组成部分。如果"温度"在元数据关键词列表中,系统会将"温度传感器"识别为"温度:传感器"的格式,删除"温度"前缀,只保留"传感器",导致设备类型被错误拆分。

### Q2: 如何判断一个词是否应该添加到元数据关键词列表?

A: 问自己两个问题:
1. 这个词是否总是作为字段名称出现?(如"型号:"、"品牌:")
2. 这个词是否可能是设备类型、设备名称或其他重要特征的一部分?

如果第1个问题的答案是"是",第2个问题的答案是"否",则可以添加。

### Q3: 如果误添加了不应该添加的关键词,会有什么影响?

A: 会导致:
1. 设备类型被错误拆分
2. 匹配权重计算错误
3. 匹配准确度下降
4. 需要重新生成所有受影响设备的规则

### Q4: 修改元数据关键词配置后,需要做什么?

A: 需要:
1. 重新生成所有受影响设备的规则
2. 验证设备类型是否被正确提取
3. 检查匹配权重是否合理
