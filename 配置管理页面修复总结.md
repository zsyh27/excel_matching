# 配置管理页面修复总结

## 发现的问题

### 1. 重复页面问题

#### 问题 A：分隔符统一 vs 处理分隔符
- **状态**：✅ 已修复
- **问题**：两个页面都使用 `SplitCharsEditor` 组件和 `feature_split_chars` 配置
- **解决方案**：创建新的 `SeparatorMappingEditor` 组件用于"分隔符统一"

#### 问题 B：元数据处理 vs 复杂参数分解
- **状态**：⚠️ 待修复
- **问题**：两个页面都使用 `AdvancedConfigEditor` 组件和 `metadata_keywords` 配置
- **正确映射**：
  - 元数据处理 → `text_cleaning.metadata_rules`
  - 复杂参数分解 → `intelligent_extraction.complex_parameter_decomposition`

### 2. 页面无法打开问题

#### 问题 C：智能拆分页面报错
- **状态**：✅ 已修复
- **错误信息**：`Invalid prop: type check failed for prop "modelValue". Expected Object, got Undefined`
- **原因**：配置键映射错误
  - 错误：`smart-split` → `intelligent_extraction`（不存在）
  - 正确：`smart-split` → `intelligent_splitting`
- **解决方案**：更新配置键映射

#### 问题 D：单位删除页面报错
- **状态**：✅ 已修复
- **错误信息**：同上
- **原因**：配置键映射错误
  - 错误：`unit-remove` → `metadata_keywords`
  - 正确：`unit-remove` → `unit_removal`
- **解决方案**：更新配置键映射

### 3. 其他配置键映射问题

#### 问题 E：质量评分
- **状态**：✅ 已修复
- **原因**：配置键映射错误
  - 错误：`quality-score` → `metadata_keywords`
  - 正确：`quality-score` → `feature_quality_scoring`

#### 问题 F：白名单
- **状态**：✅ 已修复
- **原因**：配置键映射错误
  - 错误：`whitelist` → `metadata_keywords`
  - 正确：`whitelist` → `feature_whitelist`

#### 问题 G：匹配阈值
- **状态**：✅ 已修复
- **原因**：配置键映射错误
  - 错误：`match-threshold` → `metadata_keywords`
  - 正确：`match-threshold` → `match_threshold_config`

### 2. 添加默认值处理（getEditorValue 函数）

为避免 `undefined` 错误，为不存在的配置返回合适的默认值：

```javascript
if (value === undefined) {
  if (menuId === 'smart-split') {
    return { enabled: false, split_compound_words: false, split_technical_specs: false, split_by_space: false }
  }
  if (menuId === 'unit-remove') {
    return { enabled: false, units: [] }
  }
  if (menuId === 'param-decompose') {
    return { enabled: false, complex_parameter_decomposition: {} }
  }
  if (menuId === 'quality-score') {
    return { enabled: false, scoring_rules: {} }
  }
  if (menuId === 'whitelist') {
    return []
  }
  if (menuId === 'match-threshold') {
    return { value: 0.6 }
  }
  return null
}
```

## 待完成的工作

### 1. 创建元数据处理编辑器

需要创建 `MetadataRulesEditor.vue` 组件，用于编辑 `text_cleaning.metadata_rules`。

**功能需求**：
- 配置元数据标签模式（如"型号："、"通径："等）
- 删除这些标签，只保留值
- 示例：
  - 输入："型号：ML-5000"
  - 输出："ML-5000"

**数据结构**：
```json
{
  "text_cleaning": {
    "metadata_rules": [
      { "pattern": "型号：", "action": "remove_label" },
      { "pattern": "通径：", "action": "remove_label" },
      { "pattern": "品牌：", "action": "remove_label" }
    ]
  }
}
```

### 2. 创建复杂参数分解编辑器

需要创建 `ComplexParamEditor.vue` 组件，用于编辑 `intelligent_extraction.complex_parameter_decomposition`。

**功能需求**：
- 配置复杂参数的分解规则
- 识别并拆分复合参数
- 示例：
  - 输入："温度+湿度传感器"
  - 输出：["温度传感器", "湿度传感器"]

**数据结构**：
```json
{
  "intelligent_extraction": {
    "complex_parameter_decomposition": {
      "enabled": true,
      "patterns": [
        { "pattern": "\\+", "split_type": "additive" },
        { "pattern": "/", "split_type": "alternative" }
      ]
    }
  }
}
```
