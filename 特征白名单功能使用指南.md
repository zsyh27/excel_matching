# 特征白名单功能使用指南

## 功能概述

特征白名单功能允许您指定一些特殊的特征，即使它们的质量评分低于阈值，也不会被过滤掉。这对于一些重要但简短的特征（如"水"、"气"、"阀"等）非常有用。

## 问题背景

在特征提取过程中，系统会对每个特征进行质量评分，评分低于阈值（默认50分）的特征会被过滤掉。但是有些特征虽然简短（如单字），但对匹配非常重要，不应该被过滤。

例如：
- **"水"**：介质类型，质量评分65分（通过）
- **"气"**：介质类型，质量评分65分（通过）
- **"阀"**：设备类型，质量评分30分（被过滤）

虽然"水"和"气"因为在 `medium_keywords` 中而获得了额外的15分加分，但"阀"这样的单字设备类型仍然会被过滤。

## 解决方案

### 方案1：使用白名单（推荐）

将重要的特征添加到白名单中，即使质量评分低也不会被过滤。

**优点**：
- 灵活性高，可以针对特定特征进行保护
- 不影响其他特征的质量评分机制
- 易于管理和维护

**缺点**：
- 需要手动添加每个需要保护的特征

### 方案2：降低质量评分阈值

将 `min_quality_score` 从50降低到更低的值（如30）。

**优点**：
- 一次性解决所有低分特征的问题

**缺点**：
- 可能会保留一些真正无意义的特征
- 影响整个系统的特征质量控制

## 使用方法

### 1. 通过配置管理页面（推荐）

1. 打开配置管理页面
2. 点击左侧菜单的"高级配置"
3. 在"特征白名单"部分：
   - 在输入框中输入要添加的特征（如"阀"）
   - 按回车或点击"添加"按钮
   - 要删除特征，点击特征标签上的"×"按钮
4. 点击右上角的"保存配置"按钮

### 2. 直接修改配置文件

编辑 `data/static_config.json` 文件：

```json
{
  "intelligent_extraction": {
    "feature_quality_scoring": {
      "enabled": true,
      "min_quality_score": 50,
      "whitelist_features": [
        "水",
        "气",
        "阀",
        "泵",
        "表"
      ],
      "scoring_rules": {
        ...
      }
    }
  }
}
```

**注意**：修改配置文件后需要重启后端服务才能生效。

## 配置说明

### whitelist_features 字段

- **类型**：字符串数组
- **默认值**：`["水", "气", "阀"]`
- **说明**：白名单中的特征会跳过质量评分检查

### 建议添加到白名单的特征

1. **介质类型**（单字）：
   - 水、气、油、汽（蒸汽的简称）

2. **设备类型**（单字）：
   - 阀、泵、表、柜、箱

3. **其他重要的短词**：
   - 根据实际业务需求添加

### 不建议添加到白名单的特征

1. **量词**：个、台、套、只等
2. **无意义的单字**：的、了、是等
3. **纯数字**：1、2、3等
4. **纯标点**：、。！等

## 工作原理

### 特征质量评分机制

系统对每个特征进行质量评分（0-100分）：

**基础分**：50分

**加分项**：
- 是技术术语：+20分
- 包含数字：+10分
- 包含单位：+10分
- 在设备关键词库中：+15分
- 长度适中(3-20)：+5分

**减分项**：
- 是元数据标签：-30分
- 是常见词：-20分
- 太短(<2)：-20分
- 纯数字：-15分
- 纯标点：-30分

### 白名单保护机制

在过滤阶段，系统会检查特征是否在白名单中：

```python
# 伪代码
if feature in whitelist:
    # 跳过质量评分检查，直接保留
    keep_feature(feature)
elif quality_score >= min_quality_score:
    # 质量评分达标，保留
    keep_feature(feature)
else:
    # 质量评分不达标，过滤
    filter_feature(feature)
```

## 常见问题

### Q1: 添加到白名单后，为什么规则中还是没有这个特征？

**A**: 需要执行以下步骤：
1. 确认配置文件已保存
2. 重启后端服务（让新配置生效）
3. 重新生成规则（更新数据库中的规则数据）

### Q2: 白名单和 medium_keywords 有什么区别？

**A**: 
- **medium_keywords**：介质关键词列表，会给特征加15分
- **whitelist_features**：白名单，直接跳过质量评分检查

如果一个特征在 medium_keywords 中，它会获得额外的15分加分，可能就不需要加入白名单了。

### Q3: 应该添加多少个特征到白名单？

**A**: 建议只添加确实需要保护的特征，避免过度使用白名单。一般来说：
- 5-10个特征是合理的
- 超过20个可能说明质量评分机制需要调整

### Q4: 白名单会影响匹配性能吗？

**A**: 不会。白名单只是在过滤阶段跳过质量评分检查，不影响匹配算法的性能。

## 总结

特征白名单功能提供了一种灵活的方式来保护重要但质量评分低的特征。通过合理使用白名单，可以在保持整体特征质量控制的同时，确保关键特征不会被过滤掉。

建议：
1. 优先使用 `medium_keywords`、`device_type_keywords` 等关键词列表来提高特征评分
2. 只有在评分机制无法解决的情况下，才使用白名单
3. 定期审查白名单，移除不再需要的特征
