## 三、最终权重体系

### 权重优先级（从高到低）

| 排名 | 特征类型 | 权重值 | 配置来源 | 说明 |
|------|---------|--------|---------|------|
| 1 | 设备类型（device_type字段） | **20.0** | feature_weight_config | 最核心的匹配参数 |
| 2 | key_params参数 | **15.0** | feature_weight_strategy | 次核心的匹配参数 |
| 3 | 设备类型关键词 | **15.0** | feature_weight_strategy | 识别设备类型 |
| 4 | 品牌（brand字段） | **10.0** | feature_weight_config | 重要识别特征 |
| 5 | 品牌关键词 | **10.0** | feature_weight_strategy | 识别品牌 |
| 6 | 型号 | **5.0** | feature_weight_config | 唯一标识 |
| 7 | 重要参数 | **3.0** | feature_weight_strategy | 带单位的数值范围 |
| 8 | 通用参数 | **1.0** | feature_weight_config | 区分度较低 |

### 权重比例

```
设备类型 : key_params : 品牌 : 型号 : 参数
  20     :    15      :  10  :  5   :  1
```

**核心参数占比**：
- 设备类型 + key_params = 35
- 总权重 = 20 + 15 + 10 + 5 + 1 = 51
- 核心参数占比 = 35 / 51 = **68.6%**

## 四、配置生效机制详解

### 完整流程

```
步骤1：用户添加设备
  ↓
步骤2：后端接收请求（POST /api/devices）
  ↓
步骤3：DataLoader 加载配置
  ├─ 读取 feature_weight_config（品牌、型号、设备类型、参数权重）
  └─ 读取 feature_weight_strategy（key_params、重要参数等权重）
  ↓
步骤4：初始化 RuleGenerator
  ├─ self.brand_weight = 10.0
  ├─ self.model_weight = 5.0
  ├─ self.device_type_weight = 20.0
  └─ self.parameter_weight = 1.0
  ↓
步骤5：调用 generate_rule(device)
  ├─ extract_features(device) - 提取特征
  │   ├─ 从 brand 提取
  │   ├─ 从 device_type 提取（高权重）
  │   ├─ 从 key_params 提取（高权重）
  │   └─ 从 detailed_params 提取（低权重）
  └─ assign_weights(features, device) - 分配权重
      └─ _assign_feature_weight(feature, strategy, device, key_params_features)
          ├─ 设备类型字段 → 20.0
          ├─ key_params特征 → 15.0
          ├─ 品牌字段 → 10.0
          ├─ 型号特征 → 5.0
          └─ 通用参数 → 1.0
  ↓
步骤6：保存规则到数据库
  ↓
步骤7：用户上传Excel匹配
  ↓
步骤8：使用已生成的规则进行匹配
  └─ 计算匹配得分（使用规则中的权重）
```

### 配置优先级

```python
# 权重值的优先级（从高到低）
1. self.device_type_weight（从 feature_weight_config 读取）
2. strategy['device_type_weight']（从 feature_weight_strategy 读取）
3. 默认值（20.0）
```

## 九、常见问题

### Q1：为什么前端页面还显示旧值？

**A**：可能的原因：
1. 浏览器缓存：按 Ctrl+F5 强制刷新
2. 后端未重启：重启后端服务
3. 前端未重新编译：重新运行 `npm run dev`

### Q2：修改配置后需要重新生成规则吗？

**A**：是的。修改权重配置后，需要批量重新生成所有设备的规则，新权重才能生效。

**执行命令**：
```bash
cd backend
python simple_regenerate_rules.py
```

### Q3：如何验证权重配置是否生效？

**A**：可以通过以下方式验证：

1. 检查配置：
```bash
python check_weight_config.py
```

2. 添加测试设备，查看生成的规则权重：
   - 在设备详情页面查看"匹配规则"标签
   - 检查特征权重是否符合预期

3. 上传Excel测试匹配，检查匹配得分：
   - 查看匹配详情中的得分计算
   - 验证设备类型和key_params的权重是否最高

### Q4：如果效果不理想，如何调整权重？

**A**：可以通过以下方式调整：

1. 通过前端配置管理页面修改
2. 通过脚本修改数据库配置
3. 重新生成规则
4. 测试验证
5. 根据实际效果微调

### Q5：key_params和detailed_params有什么区别？

**A**：
- **key_params**：关键参数，权重高（15.0），用于核心匹配
- **detailed_params**：详细参数，权重低（1.0），用于补充信息

建议将最重要的参数（如量程、输出信号）放在key_params中。
