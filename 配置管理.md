# 配置管理系统文档

## 概述

本文档详细描述了设备匹配系统的配置管理页面结构、各功能模块及其对应的数据库存储结构。配置管理系统按照业务处理流程组织，分为5个主要工作流阶段，每个阶段包含多个配置功能点。

## 系统架构

### 数据存储

所有配置数据存储在数据库的 `configs` 表中：

```sql
CREATE TABLE configs (
    config_key VARCHAR(100) PRIMARY KEY,
    config_value JSON NOT NULL,
    description TEXT
);
```

每个配置项以 `config_key` 作为唯一标识，配置内容以 JSON 格式存储在 `config_value` 字段中。

### 配置历史

系统支持配置版本管理，历史记录存储在 `config_history` 表中：

```sql
CREATE TABLE config_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    version INTEGER NOT NULL UNIQUE,
    config_data TEXT NOT NULL,  -- JSON格式的完整配置数据
    remark TEXT,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

## 工作流阶段详解


### 阶段一：设备信息录入前配置 📝

此阶段的配置在用户录入设备信息之前生效，用于定义系统的基础数据和规则。

#### 1.1 品牌关键词

**功能描述**：配置系统识别的设备品牌列表，用于在设备录入和匹配过程中识别品牌信息。

**数据库字段**：`config_key = 'brand_keywords'`

**数据结构**：
```json
{
  "brand_keywords": [
    "霍尼韦尔",
    "西门子",
    "江森自控",
    "施耐德",
    "通用品牌"
  ]
}
```

**使用场景**：
- 智能设备录入时自动识别品牌
- 特征提取阶段识别品牌特征
- 匹配阶段计算品牌权重得分

**相关代码**：
- `backend/modules/text_preprocessor.py` - `_extract_features_with_detail()` 方法
- `backend/modules/match_engine.py` - `calculate_weight_score()` 方法

#### 1.2 设备参数配置

**功能描述**：为不同设备类型定义可配置的参数字段，支持动态表单生成。

**数据库字段**：`config_key = 'device_params'`

**数据结构**：
```json
{
  "device_params": {
    "减压阀": {
      "keywords": ["减压阀"],
      "params": [
        {
          "name": "通径",
          "data_type": "string",
          "required": true,
          "unit": "",
          "hint": "DN15/DN20/DN25/DN32/DN40/DN50/DN65/DN80/DN100"
        },
        {
          "name": "阀体类型",
          "data_type": "string",
          "required": false,
          "unit": "",
          "hint": "直接作用式/先导式/薄膜式/活塞式"
        }
      ]
    },
    "温度传感器": {
      "params": [
        {
          "name": "检测对象",
          "type": "text",
          "required": false
        },
        {
          "name": "温度量程",
          "type": "string",
          "required": false,
          "default": "-20~60℃"
        }
      ]
    }
  }
}
```

**字段说明**：
- `keywords`: 设备类型的识别关键词
- `params`: 参数列表
  - `name`: 参数名称
  - `data_type/type`: 数据类型（string/text/number）
  - `required`: 是否必填
  - `unit`: 单位
  - `hint`: 提示信息（可选值）
  - `default`: 默认值

**使用场景**：
- 设备录入表单动态生成
- 设备编辑时显示对应参数字段
- 智能解析时提取关键参数

**相关代码**：
- `frontend/src/components/ConfigManagement/DeviceParamsEditor.vue`
- `backend/config/device_params.yaml`

#### 1.3 特征权重

**功能描述**：配置匹配阶段各特征的权重值，用于计算设备匹配得分。

**数据库字段**：`config_key = 'feature_weight_config'` 和 `config_key = 'feature_weight_strategy'`

**数据结构**：
```json
{
  "feature_weight_config": {
    "brand_weight": 1,
    "device_type_weight": 15,
    "key_params_weight": 5,
    "model_weight": 5,
    "parameter_weight": 1
  },
  "feature_weight_strategy": {
    "brand_weight": 10,
    "device_type_weight": 15,
    "key_params_weight": 15,
    "model_weight": 5,
    "common_param_weight": 1,
    "important_param_weight": 3
  }
}
```

**权重说明**：
- `brand_weight`: 品牌特征权重
- `device_type_weight`: 设备类型特征权重（最高）
- `key_params_weight`: 关键参数特征权重
- `model_weight`: 型号规格特征权重
- `parameter_weight`: 普通参数特征权重
- `common_param_weight`: 通用参数权重
- `important_param_weight`: 重要参数权重

**使用场景**：
- 匹配引擎计算加权得分
- 规则生成时分配特征权重
- 优化建议生成时调整权重

**相关代码**：
- `backend/modules/match_engine.py` - `calculate_weight_score()` 方法

### 阶段二：数据导入阶段 📥

此阶段的配置用于Excel文件导入时识别和分类数据行。

#### 2.1 设备行识别

**功能描述**：配置设备行识别规则，用于从Excel文件中自动识别哪些行是设备信息行。

**数据库字段**：`config_key = 'device_row_recognition'`

**数据结构**：
```json
{
  "device_row_recognition": {
    "industry_keywords": {
      "brands": [
        "霍尼韦尔", "西门子", "江森自控", "施耐德", "明纬",
        "Honeywell", "Siemens", "Johnson", "Schneider"
      ],
      "device_types": [
        "传感器", "控制器", "DDC", "阀门", "执行器",
        "温度传感器", "湿度传感器", "CO2传感器"
      ],
      "model_patterns": [
        "[A-Z]{2,}-[A-Z0-9]+",
        "[A-Z]+\\d{3,}",
        "\\d{4}[A-Z]+",
        "[A-Z]+-\\d+"
      ],
      "parameters": [
        "PPM", "ppm", "℃", "Pa", "%RH", "m/s", "MPa", "mA",
        "4-20mA", "0-10V", "2-10V", "AC220V", "DC24V", "DN"
      ]
    },
    "probability_thresholds": {
      "high": 50,
      "medium": 30
    },
    "scoring_weights": {
      "industry": 0.35,
      "structure": 0.35,
      "data_type": 0.3
    }
  }
}
```

**字段说明**：
- `industry_keywords`: 行业关键词库
  - `brands`: 品牌关键词列表
  - `device_types`: 设备类型关键词列表
  - `model_patterns`: 型号正则表达式模式
  - `parameters`: 参数单位和格式关键词
- `probability_thresholds`: 概率阈值
  - `high`: 高概率阈值（≥50分）
  - `medium`: 中等概率阈值（≥30分）
- `scoring_weights`: 评分权重
  - `industry`: 行业关键词权重（35%）
  - `structure`: 结构特征权重（35%）
  - `data_type`: 数据类型权重（30%）

**使用场景**：
- Excel导入时自动识别设备行
- 过滤标题行、合计行、备注行
- 提供识别概率评分

**相关代码**：
- `backend/modules/device_row_classifier.py` - `DeviceRowClassifier` 类

3. 层次2：精准映射（normalization_map）
4. 层次3：通用归一化（全角转半角、删除空格、统一大小写）

**字段说明**：
- `pattern`: 匹配模式（正则表达式）
- `extraction_logic`: 提取逻辑标识

**使用场景**：
- 分解"4-20mA"为"4-20"和"mA"
- 分解"温度+湿度"为"温度"和"湿度"

```json
    "阀门",
    "风机",
    "风阀"
  ]
}
```

**使用场景**：
- 智能设备录入时识别设备类型
- 特征提取时识别设备类型特征
- 匹配时计算设备类型权重（权重最高）

**相关代码**：
- `backend/modules/text_preprocessor.py` - `_extract_features_with_detail()` 方法
- `backend/modules/match_engine.py` - `calculate_weight_score()` 方法

#### 4.3 匹配阈值

**功能描述**：配置默认匹配阈值，用于判断匹配是否成功。

**数据库字段**：`config_key = 'default_match_threshold'` 和 `config_key = 'global_config'`

**数据结构**：
```json
{
  "default_match_threshold": {
    "value": 0.6
  },
  "global_config": {
    "default_match_threshold": 5
  }
}
```

**字段说明**：
- `default_match_threshold.value`: 相似度阈值（0-1之间，旧版本）
- `global_config.default_match_threshold`: 加权得分阈值（新版本，推荐使用）

**匹配流程**：
1. 第一轮匹配：使用规则自己的 `match_threshold`
2. 第二轮匹配：使用 `default_match_threshold` 兜底
3. 选择得分最高的匹配结果

**使用场景**：
- 判断匹配是否成功
- 过滤低分匹配结果
- 提供兜底匹配机制

**相关代码**：
- `backend/modules/match_engine.py` - `match()` 方法

**使用场景**：

## 配置优先级

当多个配置项存在冲突时，按以下优先级处理：

1. **白名单** > 质量评分（白名单特征不受质量评分影响）
2. **精准映射** > 同义词映射（归一化在同义词之前执行）
3. **规则阈值** > 默认阈值（规则自己的阈值优先）
4. **设备类型权重** > 其他权重（设备类型权重最高）

---

## 配置最佳实践

### 1. 品牌关键词配置

- 包含常见品牌的中英文名称
- 包含品牌缩写和全称
- 定期更新新增品牌

### 2. 同义词映射配置

- 优先配置常用缩写（如 co2、co）
- 配置行业术语的多种表达方式
- 避免过度映射导致误匹配

### 3. 特征权重配置

- 设备类型权重应最高（推荐15）
- 关键参数权重次之（推荐5-15）
- 品牌和型号权重适中（推荐1-10）
- 普通参数权重最低（推荐1-3）

### 4. 匹配阈值配置

- 默认阈值建议设置为5-10
- 过低会导致误匹配
- 过高会导致匹配失败
- 根据实际匹配效果调整

### 5. 白名单配置

- 包含所有重要的短特征
- 包含专业术语和缩写
- 定期审查和更新

---

## 配置变更影响分析

### 修改品牌关键词

**影响范围**：
- 智能设备录入的品牌识别
- 特征提取的品牌拆分
- 匹配阶段的品牌权重计算

**建议**：
- 添加新品牌：影响小，可直接添加
- 删除品牌：影响大，需评估现有设备

---

### 修改同义词映射

**影响范围**：
- 匹配阶段的同义词扩展
- 匹配召回率和准确率

**建议**：
- 添加新映射：影响小，可直接添加
- 修改现有映射：影响大，需测试验证
- 删除映射：影响大，可能导致匹配失败

---

### 修改特征权重

**影响范围**：
- 匹配得分计算
- 匹配结果排序
- 规则生成的权重分配

**建议**：
- 小幅调整（±20%）：影响小
- 大幅调整（>50%）：影响大，需全面测试
- 调整后需重新生成规则

---

### 修改匹配阈值

**影响范围**：
- 匹配成功率
- 误匹配率

**建议**：
- 提高阈值：减少误匹配，可能增加匹配失败
- 降低阈值：提高匹配成功率，可能增加误匹配
- 建议通过测试接口验证效果

---

## 故障排查

### 配置保存失败

**可能原因**：
1. 配置格式错误
2. 数据库连接失败
3. 权限不足

**解决方法**：
1. 使用验证接口检查配置格式
2. 检查数据库连接状态
3. 检查用户权限

---

### 配置不生效

**可能原因**：
1. 配置未保存
2. 系统组件未重新加载
3. 配置被缓存

**解决方法**：
1. 确认配置已保存成功
2. 重启后端服务
3. 清除浏览器缓存

---

### 匹配结果异常

**可能原因**：
1. 同义词映射配置错误
2. 特征权重配置不合理
3. 匹配阈值设置不当

**解决方法**：
1. 使用测试接口验证配置效果
2. 检查匹配详情日志
3. 逐步调整配置参数
