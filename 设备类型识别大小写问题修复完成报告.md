#### 修改2：更新 `recognize()` 方法

```python
def recognize(self, text: str) -> DeviceTypeInfo:
    """识别设备类型"""
    if not text or not text.strip():
        return DeviceTypeInfo(...)
    
    # 新增：应用文本归一化（如果预处理器可用）
    normalized_text = text
    if self.preprocessor:
        try:
            normalized_text = self.preprocessor.normalize_text(text, mode='matching')
            logger.debug(f"文本归一化: '{text}' -> '{normalized_text}'")
        except Exception as e:
            logger.warning(f"文本归一化失败: {e}，使用原始文本")
            normalized_text = text
    
    # 使用归一化后的文本进行匹配
    result = self._exact_match(normalized_text)
    if result and result.confidence >= 0.95:
        return result
    
    result = self._fuzzy_match(normalized_text)
    if result and result.confidence >= 0.85:
        return result
    
    result = self._keyword_match(normalized_text)
    if result and result.confidence >= 0.75:
        return result
    
    result = self._type_inference(normalized_text)
    return result if result else DeviceTypeInfo(...)
```

**关键变化**：
- ✅ 在匹配前调用 `normalize_text()`
- ✅ 使用归一化后的文本进行所有匹配
- ✅ 添加调试日志，便于追踪
- ✅ 添加异常处理，确保稳定性

```
原始文本: "CO浓度探测器 量程0~250ppm"

全角转半角: "CO浓度探测器 量程0~250ppm"
    ↓
删除空格: "CO浓度探测器量程0~250ppm"
    ↓

    ↓

```

## ⚠️ 注意事项

### 1. 配置要求

确保全局配置中启用了文本归一化选项：

```python
'global_config': {
    'unify_lowercase': True,        # 必须启用
    'remove_whitespace': True,      # 推荐启用
    'fullwidth_to_halfwidth': True  # 推荐启用
}
```

### 2. 性能影响

- 每次调用 `recognize()` 都会进行文本归一化
- 对于单次调用，性能影响可忽略（< 1ms）
- 对于批量调用，总体性能影响仍然很小

### 3. 日志输出

启用 DEBUG 日志可以看到归一化过程：

```python
logger.debug(f"文本归一化: '{text}' -> '{normalized_text}'")
```

### 4. 异常处理

如果文本归一化失败，识别器会：
- 记录警告日志
- 使用原始文本继续识别
- 不会中断识别流程
