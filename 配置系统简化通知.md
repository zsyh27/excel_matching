# 配置系统简化通知

## 📢 重要变更通知

**日期**: 2026-03-08  
**变更**: 配置系统简化 - 移除YAML配置文件

---

## 🎯 变更内容

系统配置管理已简化，**不再使用YAML配置文件**，统一使用数据库作为配置源。

### 删除的文件

- ❌ `backend/config/device_params.yaml`

### 变更原因

之前系统有两个配置源（YAML文件 + 数据库），导致：
- 数据冗余
- 需要手动同步
- 容易不一致

现在统一使用数据库，更简单、更可靠。

---

## ✅ 对用户的影响

### 好消息：大部分用户无影响！

如果你只是使用系统（不修改代码），**完全不受影响**：

- ✅ 前端配置管理页面正常使用
- ✅ 设备录入功能正常使用
- ✅ 设备匹配功能正常使用
- ✅ 所有配置数据完整保留

### 需要注意的情况

**如果你有以下情况，需要采取行动**：

#### 1. 你修改过 `device_params.yaml` 文件

**行动**: 
1. 备份你的YAML文件（如果还存在）
2. 通过前端配置管理页面重新应用你的修改
3. 或运行迁移脚本（如果YAML文件还存在）：
   ```bash
   python sync_yaml_config_to_database.py
   ```

#### 2. 你的脚本使用了 `ConfigurationManager`

**行动**: 更新你的脚本

**修改前**:
```python
from modules.intelligent_device.configuration_manager import ConfigurationManager

config_path = 'backend/config/device_params.yaml'
config_manager = ConfigurationManager(config_path)
```

**修改后**:
```python
from modules.intelligent_device.configuration_manager import ConfigurationManager
from modules.database import DatabaseManager

db_manager = DatabaseManager('sqlite:///data/devices.db')
config_manager = ConfigurationManager(db_manager)
```

## 📖 配置管理方式

### 查看和修改配置

**推荐方式：使用前端界面**

1. 访问配置管理页面：`http://localhost:5000/config`
2. 选择"设备参数配置"菜单
3. 查看或修改配置
4. 点击"保存"按钮

**优点**：
- 可视化界面，操作简单
- 自动验证配置格式
- 支持版本历史和回滚

### 备份配置

**方法1：备份数据库文件**（推荐）
```bash
cp data/devices.db data/devices_backup.db
```

**方法2：通过前端导出**
- 点击配置管理页面的"导出"按钮
- 保存JSON文件

## ❓ 常见问题

### Q1: 我的配置会丢失吗？

**A**: 不会！所有配置都已经在数据库中，完整保留。

### Q2: 我需要做什么吗？

**A**: 大部分用户不需要做任何事情。只有修改过YAML文件或自定义脚本的用户需要采取行动。

### Q3: 如何添加新的设备类型？

**A**: 
1. 访问前端配置管理页面
2. 选择"设备参数配置"
3. 添加新的设备类型
4. 保存配置

### Q4: 配置修改后需要重启服务吗？

**A**: 不需要。配置保存后立即生效。

### Q5: 如何回滚到之前的配置？

**A**: 
1. 访问配置管理页面
2. 点击"版本历史"按钮
3. 选择要回滚的版本
4. 点击"回滚"按钮
