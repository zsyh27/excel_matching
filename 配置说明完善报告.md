# 配置说明完善报告

## 修复时间
2026-03-07

## 问题描述

用户反馈：智能特征提取中的4个配置页面（设备类型模式、参数提取模式、辅助信息模式、同义词映射）没有实现配置说明功能，页面顶部缺少配置说明卡片。

## 问题分析

### 1. 现有实现

其他配置页面（如品牌关键词）使用了完整的 `ConfigInfoCard` 组件，通过传递详细的 props 来显示配置说明：

```vue
<ConfigInfoCard
  stage="pre-entry"
  stage-icon="📝"
  stage-name="设备信息录入前配置"
  stage-description="此配置在设备信息录入前生效..."
  usage-text="品牌关键词用于识别设备描述中的品牌信息..."
  :examples="[...]"
  :notes="[...]"
/>
```

### 2. 智能提取页面的问题

智能提取的4个编辑器使用了简化的 `ConfigInfoCard`，但传递了错误的 props：

```vue
<!-- 错误的用法 -->
<ConfigInfoCard
  title="设备类型模式配置"
  description="配置设备类型识别的模式和规则"
  :last-updated="lastUpdated"
  :editor-name="editorName"
/>
```

问题：
- `ConfigInfoCard` 组件不接受 `title`、`description`、`last-updated`、`editor-name` 这些 props
- 没有传递 `config-id` 属性，导致无法从 `configInfoMap` 加载配置说明
- 结果：配置说明卡片显示为空白

### 3. 正确的实现方式

`ConfigInfoCard` 组件支持两种使用方式：

**方式1：使用 config-id（推荐）**
```vue
<ConfigInfoCard config-id="device-type-patterns" />
```
- 自动从 `configInfoMap.js` 加载配置信息
- 简洁、易维护
- 配置信息集中管理

**方式2：手动传递 props**
```vue
<ConfigInfoCard
  stage="pre-entry"
  stage-icon="📝"
  stage-name="设备信息录入前配置"
  stage-description="..."
  usage-text="..."
  :examples="[...]"
  :notes="[...]"
/>
```
- 适用于特殊情况
- 需要手动维护所有信息

## ConfigInfoCard 组件功能

### 简洁视图（默认显示）

- 显示工作流阶段徽章（带图标和名称）
- 显示简短的使用说明
- 提供"查看详情"按钮
- 占用空间小（约80-100px高度）

### 详细信息对话框（点击"查看详情"）

包含以下部分：

1. **流程位置**
   - 工作流阶段徽章
   - 阶段描述

2. **使用说明**
   - 详细的使用说明文本
   - 支持自定义内容（通过 slot）

3. **配置示例**（可选）
   - 具体的配置示例列表
   - 帮助用户理解如何配置

4. **注意事项**（可选）
   - 重要的注意事项列表
   - 黄色背景突出显示

## 配置说明的维护

### 集中管理

所有配置说明都在 `configInfoMap.js` 中集中管理：

**优点：**
- 易于维护和更新
- 保证信息一致性
- 便于国际化扩展

**使用方式：**
```javascript
// 1. 在 configInfoMap.js 中添加配置信息
export const CONFIG_INFO_MAP = {
  'my-config': {
    stage: 'pre-entry',
    stageIcon: '📝',
    stageName: '...',
    stageDescription: '...',
    usageText: '...',
    examples: [...],
    notes: [...]
  }
};

// 2. 在编辑器组件中使用
<ConfigInfoCard config-id="my-config" />
```

### 更新流程

1. 修改 `configInfoMap.js` 中的配置信息
2. 前端自动重新构建
3. 所有使用该配置的页面自动更新
