# 设备名称权重说明

## 问题

用户发现设备名称（如"室内温度传感器"）的权重为1，但在"配置管理-特征权重"中没有看到"设备名称"这个配置项。

## 当前实现

### 权重配置

在数据库的 `feature_weight_config` 中，当前有以下权重配置：

```json
{
  "brand_weight": 10.0,           // 品牌权重
  "model_weight": 5.0,            // 型号权重
  "device_type_weight": 20.0,     // 设备类型权重
  "parameter_weight": 1.0,        // 参数权重
  "key_params_weight": 15.0       // 关键参数权重
}
```

### 特征提取逻辑

在 `DeviceFeatureExtractor` 中，设备名称使用的是 `parameter_weight`：

```python
# 3. 提取设备名称
if device.device_name:
    feature = self._normalize(device.device_name)
    if feature:
        features.append(DeviceFeature(
            feature=feature,
            type='device_name',
            weight=self.parameter_weight,  # 使用参数权重(1.0)
            source='device_name'
        ))
```

## 设计原理

### 为什么设备名称权重较低？

设备名称（如"室内温度传感器"）通常比较通用，不够具体，因此权重设置较低：

1. **品牌（权重10）** - 非常重要，能区分不同厂商
2. **设备类型（权重20）** - 最重要，决定设备的大类
3. **规格型号（权重5）** - 重要，是设备的唯一标识
4. **关键参数（权重15）** - 很重要，描述设备的具体特性
5. **设备名称（权重1）** - 较低，因为通常比较通用

### 示例说明

以"霍尼韦尔 室内温度传感器 HST-RA"为例：

| 特征 | 类型 | 权重 | 说明 |
|------|------|------|------|
| 霍尼韦尔 | brand | 10.0 | 品牌，能区分厂商 |
| 温度传感器 | device_type | 20.0 | 设备类型，最重要 |
| 室内温度传感器 | device_name | 1.0 | 设备名称，较通用 |
| hst-ra | model | 5.0 | 规格型号，唯一标识 |
| 温度 | parameter | 15.0 | 关键参数 |
| 室内墙装 | parameter | 15.0 | 关键参数 |

**为什么"室内温度传感器"权重低？**
- 这个名称太通用了，很多设备都可能叫这个名字
- 真正能区分设备的是：品牌（霍尼韦尔）、型号（HST-RA）、设备类型（温度传感器）
- 如果设备名称权重太高，可能导致误匹配

## 是否需要独立的设备名称权重？

### 当前设计的优点

1. **简化配置** - 减少配置项数量
2. **符合实际** - 设备名称确实应该权重较低
3. **统一管理** - 所有通用描述性文本都使用参数权重

### 可能的改进方案

如果需要更精细的控制，可以添加独立的 `device_name_weight` 配置：

```json
{
  "brand_weight": 10.0,
  "device_type_weight": 20.0,
  "device_name_weight": 1.0,      // 新增：设备名称权重
  "model_weight": 5.0,
  "key_params_weight": 15.0,
  "parameter_weight": 1.0
}
```

## 建议

### 保持当前设计

建议保持当前设计，原因：

1. **设备名称权重应该较低** - 这是合理的设计
2. **避免配置复杂化** - 太多配置项会让用户困惑
3. **实际效果良好** - 当前权重分配已经能很好地工作

### 如果需要调整

如果用户确实需要调整设备名称的权重，有两个选择：

#### 选择1：调整 parameter_weight（影响所有参数）

在"配置管理-特征权重"中调整"参数权重"，但这会影响所有使用参数权重的特征。

#### 选择2：添加独立的设备名称权重配置

需要修改以下文件：

1. **数据库配置** - 添加 `device_name_weight` 到 `feature_weight_config`
2. **DeviceFeatureExtractor** - 添加 `self.device_name_weight` 并使用它
3. **前端配置界面** - 添加"设备名称权重"配置项
