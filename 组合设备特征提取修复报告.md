# 组合设备特征提取修复报告

## 问题描述

用户反馈"蝶阀+开关型执行器"和"蝶阀+调节型执行器"的特征提取数量偏少，没有达到阀门与执行器特征之和。

## 问题分析

经过检查发现，部分组合设备的 `key_params` 中缺少"公称通径"参数。这是因为：

1. 原始数据中，部分设备的规格型号包含通径信息（如 V8BFW16-050），但没有显式存储在 `key_params` 中
2. 特征提取器只从 `key_params` 中提取参数特征，导致缺少公称通径的设备特征数量偏少

## 解决方案

### 1. 从规格型号中提取公称通径

创建脚本 `extract_diameter_from_spec.py`，从规格型号中提取公称通径并补充到 `key_params`：

```python
# 公称通径映射表
diameter_map = {
    '050': 'DN50',
    '065': 'DN65',
    '080': 'DN80',
    '100': 'DN100',
    '125': 'DN125',
    '150': 'DN150',
    '200': 'DN200',
    '250': 'DN250',
    '300': 'DN300'
}
```

### 2. 重新生成规则

使用 `regenerate_combined_device_rules.py` 重新生成所有组合设备的规则，确保特征提取包含新增的公称通径参数。
