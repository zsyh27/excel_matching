# 文本预处理流程详解

## 您的问题回答

### 问题1：原始文本是不是应该先归一化之后再进行清理？

**答案：不是的，当前的处理顺序是正确的。**

**当前处理顺序**：
```
原始文本 → 智能清理 → 归一化 → 特征提取
```

**为什么这个顺序是正确的？**

1. **智能清理需要识别原始格式**
   - 智能清理需要识别"3.施工要求:"这样的元数据标签
   - 如果先归一化，会把"："转换为其他字符，导致无法识别
   - 噪音分隔符（如"施工要求"、"备注"）也需要在原始格式下识别

2. **归一化会改变文本结构**
   - 归一化会删除空格、转换大小写、替换字符
   - 这些操作会破坏原始的文本结构，影响智能清理的准确性

3. **实际案例**
   ```
   原始文本: "1.名称:CO浓度探测器 3.施工要求:按照图纸..."
   
   如果先归一化:
   "1名称co浓度探测器3施工要求按照图纸..." 
   → 无法识别"施工要求"作为截断点
   
   正确顺序（先清理）:
   "1.名称:CO浓度探测器" 
   → 成功在"施工要求"处截断
   → 然后归一化为"co浓度探测器"
   ```

### 问题2：归一化映射中的全局配置是什么意思？

**全局配置（Global Config）** 是指应用于所有文本的通用归一化规则。

**两种归一化规则**：

1. **精准映射（Normalization Map）**
   - 特定字符的精确替换
   - 例如：`"℃" → ""`, `"°C" → ""`, `"度" → ""`
   - 需要在配置文件中明确定义每一个映射

2. **全局配置（Global Config）**
   - 通用的文本处理规则
   - 自动应用于所有字符，无需逐个定义

**全局配置包含的规则**：

| 规则名称 | 说明 | 示例 |
|---------|------|------|
| `fullwidth_to_halfwidth` | 全角字符转半角 | "１２３" → "123" |
| `unify_lowercase` | 统一转小写 | "CO" → "co" |
| `remove_whitespace` | 删除所有空格 | "CO 浓度" → "CO浓度" |

**您看到的输出**：
```
归一化映射: 0 个映射
全局配置: fullwidth_to_halfwidth, unify_lowercase
```

**含义**：
- **0 个映射**：您的文本中没有触发精准映射规则（如℃、°C等）
- **fullwidth_to_halfwidth**：应用了全角转半角规则
- **unify_lowercase**：应用了统一小写规则

### 问题3：智能清理的应用规则是什么？为什么删除57个字符？

**智能清理包含5个阶段**：

#### 阶段0：过滤行号/序号（可选）
- 删除"1."、"2."、"3."等序号前缀

#### 阶段1：截断（Truncation）
- **目的**：在噪音分隔符处截断文本，删除后面的无关内容
- **噪音分隔符**：
  - "施工要求"
  - "备注"
  - "说明"
  - "注意事项"
  - 等等...

**您的案例**：
```
原始文本: "1.名称:CO浓度探测器 2.规格参数：... 3.施工要求:按照图纸..."
                                              ↑ 在这里截断
截断后: "1.名称:CO浓度探测器 2.规格参数：..."
删除了: "3.施工要求:按照图纸、规范及清单要求配置并施工调试到位，并达到验收使用要求。"
```

#### 阶段2：删除噪音段落（Noise Pattern）
- **目的**：删除匹配噪音模式的整个段落
- **噪音模式**：
  - 重复的标点符号
  - 无意义的填充文本
  - 等等...

#### 阶段3：删除元数据标签（Metadata Tag）
- **目的**：删除字段名称前缀，只保留值
- **元数据标签**：
  - "名称:"
  - "型号:"
  - "规格:"
  - "参数:"
  - 等等...

**您的案例**：
```
清理前: "1.名称:CO浓度探测器 2.规格参数：工作原理：..."
清理后: "CO浓度探测器+工作原理：..."
删除了: "1.", "名称:", "2.", "规格参数："
```

#### 阶段4：删除无关关键词（Ignore Keywords）
- **目的**：删除配置中指定的无关词汇
- **无关关键词**：
  - 可能包括一些常见的无意义词汇

#### 阶段5：统一分隔符（Separator Unification）
- **目的**：将各种分隔符统一为标准分隔符（通常是"+"）
- **分隔符**：
  - 空格 → +
  - 逗号 → +
  - 分号 → +
  - 换行 → +
  - 等等...

**您的案例**：
```
清理前: "CO浓度探测器 工作原理：电化学式； 量程 0~ 250ppm"
清理后: "CO浓度探测器+工作原理：电化学式++量程+0~250ppm"
```

## 为什么删除57个字符？

**计算过程**：

```
原始文本长度: 约 200 字符
清理后文本长度: 约 143 字符
删除长度: 200 - 143 = 57 字符
```

**删除的内容包括**：

1. **截断删除**（最主要）
   - "3.施工要求:按照图纸、规范及清单要求配置并施工调试到位，并达到验收使用要求。"
   - 这部分约 40-45 字符

2. **元数据标签删除**
   - "1.", "名称:", "2.", "规格参数："
   - 这部分约 10-12 字符

3. **其他清理**
   - 可能还有一些空格、标点符号等
   - 约 2-5 字符

**总计**：约 57 字符

## 完整处理流程示例

### 输入文本
```
1.名称:CO浓度探测器 2.规格参数：工作原理：电化学式； 量程 0~ 250ppm ；输出信号 4~20mA / 2~10VDC；精度±5% @25C. 50% RH（0~100 ppm） 3.施工要求:按照图纸、规范及清单要求配置并施工调试到位，并达到验收使用要求。
```

### 步骤0：智能清理

#### 阶段1：截断
```
应用规则: truncation
在 "3.施工要求" 处截断
删除: "3.施工要求:按照图纸、规范及清单要求配置并施工调试到位，并达到验收使用要求。"
结果: "1.名称:CO浓度探测器 2.规格参数：工作原理：电化学式； 量程 0~ 250ppm ；输出信号 4~20mA / 2~10VDC；精度±5% @25C. 50% RH（0~100 ppm）"
```

#### 阶段3：删除元数据标签
```
应用规则: metadata_tag
删除: "1.", "名称:", "2.", "规格参数："
结果: "CO浓度探测器 工作原理：电化学式； 量程 0~ 250ppm ；输出信号 4~20mA / 2~10VDC；精度±5% @25C. 50% RH（0~100 ppm）"
```

#### 阶段5：统一分隔符
```
应用规则: separator_unification
将空格、分号、逗号等统一为 "+"
结果: "CO浓度探测器+工作原理：电化学式++量程+0~250ppm++输出信号+4~20mA+/+2~10VDC+精度±5%++@25C.+50%+RH（0~100++ppm）"
```

**智能清理总结**：
- 应用规则: truncation, metadata_tag, separator_unification
- 删除长度: 57 字符

### 步骤1：归一化

#### 全角转半角
```
无全角字符，跳过
```

#### 统一小写
```
应用规则: unify_lowercase
结果: "co浓度探测器+电化学式++量程+0~250ppm++输出信号+4~20ma+/+2~10vdc+精度±5%++@25c.+50%+rh（0~100++ppm）"
```

**归一化总结**：
- 归一化映射: 0 个
- 全局配置: fullwidth_to_halfwidth, unify_lowercase

### 步骤2：特征提取

#### 按分隔符拆分
```
按 "+" 拆分
结果: ["co浓度探测器", "电化学式", "", "量程", "0~250ppm", "", "输出信号", "4~20ma", "/", "2~10vdc", "精度±5%", "", "@25c.", "50%", "rh（0~100", "", "ppm）"]
```

#### 过滤无效特征
```
删除空字符串、单字符、低质量特征
保留: ["co浓度探测器", "电化学式", "0~250", "4~20", "2~10vd", "±5%", "25", "50%", "0~100"]
```

**特征提取总结**：
- 提取: 12 个
- 过滤: 6 个（空字符串、单字符等）
- 最终: 9 个有效特征

## 总结

1. **处理顺序是正确的**：先清理 → 再归一化 → 最后提取特征
2. **全局配置**：是通用的归一化规则，自动应用于所有字符
3. **智能清理规则**：
   - `truncation`：在噪音分隔符处截断
   - `metadata_tag`：删除字段名称前缀
   - `separator_unification`：统一分隔符
4. **删除57字符**：主要是截断删除的"施工要求"部分和元数据标签

这个设计确保了：
- 准确识别和删除无关内容
- 保留所有有价值的技术参数
- 提取出干净、标准化的特征用于匹配
