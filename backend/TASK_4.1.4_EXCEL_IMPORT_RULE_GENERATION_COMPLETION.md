## 实施内容

### 1. 修改 DeviceImporter 类

在 `backend/scripts/import_devices_from_excel.py` 中增强了设备导入器,添加了自动规则生成功能。

#### 1.1 更新 import_to_database 方法

**修改内容:**
- 添加 `auto_generate_rules` 参数(默认为 True)
- 在导入设备后自动触发规则生成

**验证需求:** 2.1, 3.1-3.5

```python
def import_to_database(self, devices: List[Dict], batch_size: int = 100, auto_generate_rules: bool = True) -> None:
    """
    批量导入设备到数据库
    
    Args:
        devices: 设备数据列表
        batch_size: 批量大小
        auto_generate_rules: 是否自动生成规则
    """
```

#### 1.2 更新 _import_batch 方法

**修改内容:**
- 在设备导入成功后调用 `_generate_rules_for_batch`
- 确保设备数据先提交,再生成规则
- 规则生成失败不影响设备导入

**验证需求:** 2.5, 2.6, 3.1-3.5

#### 1.3 新增 _generate_rules_for_batch 方法

**功能:**
- 为批次中的设备生成匹配规则
- 使用 RuleGenerator 自动提取特征和分配权重
- 检查规则是否已存在,避免重复生成
- 将生成的规则保存到数据库
- 记录规则生成统计信息

**验证需求:** 3.1-3.5

**实现细节:**
```python
def _generate_rules_for_batch(self, session, batch: List[Dict]) -> None:
    """
    为批次中的设备生成匹配规则
    
    1. 导入必要的模块(RuleGenerator, TextPreprocessor等)
    2. 加载配置文件
    3. 初始化预处理器和规则生成器
    4. 遍历批次中的设备:
       - 检查规则是否已存在
       - 创建设备对象
       - 生成规则
       - 保存规则到数据库
    5. 提交规则数据
    6. 更新统计信息
    """
```

### 3. 添加命令行参数

#### 3.1 新增 --no-rules 参数

**功能:**
- 允许用户选择是否自动生成规则
- 默认情况下自动生成规则
- 使用 `--no-rules` 标志可以禁用规则生成

**使用示例:**
```bash

# 只导入设备,不生成规则

```

#### 4.1 规则生成错误处理

- 规则生成失败不会导致设备导入回滚
- 每个设备的规则生成错误都会被记录
- 批次规则生成失败会记录错误但继续处理

#### 4.2 日志记录

- 记录规则生成开始和完成信息
- 记录每个规则的生成状态(成功/失败)

## 技术实现细节

### 1. 规则生成流程

```
1. 设备导入到数据库
   ↓
2. 提交设备数据
   ↓
3. 检查是否启用自动生成规则
   ↓
4. 为每个设备:
   - 检查规则是否已存在
   - 创建设备对象
   - 调用 RuleGenerator.generate_rule()
   - 保存规则到数据库
   ↓
5. 提交规则数据
   ↓
6. 更新统计信息
```

### 2. 依赖模块

- `modules.rule_generator.RuleGenerator`: 规则生成器
- `modules.text_preprocessor.TextPreprocessor`: 文本预处理器
- `modules.data_classes.Device`: 设备数据类
- `modules.models.Rule`: 规则 ORM 模型

### 3. 配置加载

从 `data/static_config.json` 加载配置:
- `default_match_threshold`: 默认匹配阈值
- 其他预处理和规则生成相关配置
