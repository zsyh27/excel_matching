## 问题描述

用户在测试页面输入设备描述时，"步骤2：参数候选提取" 显示的是旧的参数提取模式的参数值，例如：

```
量程参数: 0~ 250ppm (80.0%)
输出信号: 4~20mA (90.0%)
精度参数: ±5% (90.0%)
```

而不是新的参数候选列表格式。

### 测试用例

**输入文本**:
```
1.名称:CO浓度探测器 
2.规格参数：工作原理：电化学式； 量程 0~ 250ppm ；输出信号 4~20mA / 2~10VDC；精度±5%  @25C. 50% RH（0~100  ppm） 
3.施工要求:按照图纸、规范及清单要求配置并施工调试到位，并达到验收使用要求。
```

**期望输出**: 显示参数候选列表，包含所有提取到的候选项（量程、输出信号、精度、温度等）

## 根本原因

### 数据流分析

1. **后端 API** (`/api/intelligent-extraction/preview`):
   - ✅ 已经在 `api_handler.py` 中提取 `parameter_candidates`
   - ✅ 已经在返回数据中包含 `parameter_candidates` 字段
   - ❌ 但前端没有正确接收和传递这个数据

2. **前端组件** (`FiveStepPreview.vue`):
   - ❌ 在处理 API 响应时，没有将 `parameter_candidates` 添加到 `previewResult` 对象中
   - 结果：`ParameterStep.vue` 组件无法访问 `parameter_candidates` 数据

3. **参数显示组件** (`ParameterStep.vue`):
   - ✅ 已经实现了新的参数候选显示逻辑
   - ✅ 有回退机制：如果没有 `parameter_candidates`，显示旧格式
   - ❌ 由于数据未传递，一直使用旧格式显示

## 数据结构对比

### 旧格式 (step2_parameters)

```json
{
  "range": {
    "value": "0~ 250ppm",
    "confidence": 0.8
  },
  "output": {
    "value": "4~20mA",
    "confidence": 0.9
  },
  "accuracy": {
    "value": "±5%",
    "confidence": 0.9
  },
  "specs": []
}
```

**特点**:
- 每种参数类型只保留一个值
- 结构固定（range, output, accuracy, specs）
- 信息丢失（例如：`2~10VDC` 输出信号被忽略）

### 新格式 (parameter_candidates)

```json
[
  {
    "param_type": "range",
    "value": "0~ 250ppm",
    "confidence": 0.9,
    "source": "量程 0~ 250ppm"
  },
  {
    "param_type": "output",
    "value": "4~20mA",
    "confidence": 0.9,
    "source": "输出信号 4~20mA"
  },
  {
    "param_type": "output",
    "value": "2~10V",
    "confidence": 0.9,
    "source": "2~10VDC"
  },
  ...
]
```

**特点**:
- 保留所有提取到的候选项
- 每个候选项独立显示
- 信息完整（所有输出信号都被保留）
- 更灵活的数据结构
