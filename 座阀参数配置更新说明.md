# 座阀参数配置更新说明

## 问题描述

前端配置管理页面显示：
- 座阀：3个参数
- 执行器：3个参数

但根据导入的Excel文件，座阀应该有12个参数字段。

## 问题原因

### YAML配置文件中的参数不完整

**原始配置**（只有3个参数）：
```yaml
座阀:
  keywords: ["座阀", "调节阀", "control valve", "座式调节阀"]
  params:
    - name: "通径"
    - name: "压力等级"
    - name: "流量系数"
```

**Excel文件中的参数**（12个）：
1. Cv值
2. Kvs值
3. 介质
4. 动作方式
5. 压力等级
6. 接口类型
7. 温度范围
8. 英制尺寸
9. 通径
10. 通数
11. 配套执行器
12. 阀门类型

### 为什么会不一致？

1. **YAML配置是通用配置**：用于智能解析和参数提取
2. **Excel导入使用的是动态参数**：直接从Excel表头读取
3. **两者没有同步**：导入脚本中定义了完整的参数，但YAML文件没有更新

## 解决方案

### 更新YAML配置文件

将座阀的参数从3个扩展到12个，与Excel文件保持一致：

```yaml
座阀:
  keywords: ["座阀", "调节阀", "control valve", "座式调节阀", "水阀", "蒸汽阀"]
  params:
    - name: "通径"
      pattern: "DN\\s*([0-9]+)"
      required: false
      data_type: "string"
      unit: "mm"
    - name: "通数"
      pattern: "([0-9]+)通"
      required: false
      data_type: "string"
      unit: null
    - name: "英制尺寸"
      pattern: "([0-9/]+)\""
      required: false
      data_type: "string"
      unit: null
    - name: "Cv值"
      pattern: "Cv\\s*([0-9.]+)"
      required: false
      data_type: "number"
      unit: null
    - name: "Kvs值"
      pattern: "Kvs\\s*([0-9.]+)"
      required: false
      data_type: "number"
      unit: null
    - name: "介质"
      pattern: null
      required: false
      data_type: "string"
      unit: null
    - name: "阀门类型"
      pattern: null
      required: false
      data_type: "string"
      unit: null
    - name: "接口类型"
      pattern: null
      required: false
      data_type: "string"
      unit: null
    - name: "动作方式"
      pattern: null
      required: false
      data_type: "string"
      unit: null
    - name: "压力等级"
      pattern: "PN\\s*([0-9]+)"
      required: false
      data_type: "string"
      unit: "bar"
    - name: "温度范围"
      pattern: "(-?[0-9]+)[-~]([0-9]+)°?C"
      required: false
      data_type: "string"
      unit: "℃"
    - name: "配套执行器"
      pattern: null
      required: false
      data_type: "string"
      unit: null
```

**关键变更**：
1. 添加了9个新参数
2. 更新了关键词，包含"水阀"和"蒸汽阀"（因为Excel中这些类型统一归类为"座阀"）
3. 所有参数都设置为`required: false`（允许空值）

## 执行器参数说明

### 为什么执行器只有3个参数？

**Excel数据分析**：
```
执行器:
  (所有参数字段都是空的)
```

执行器在Excel文件中没有使用任何参数字段，所有36个执行器设备的参数列都是空的。因此，执行器保持3个基础参数（扭矩、行程时间、控制信号）是合理的。

**执行器配置**（保持不变）：
```yaml
执行器:
  keywords: ["执行器", "电动执行器", "actuator", "风阀执行器"]
  params:
    - name: "扭矩"
      pattern: "扭矩[:：]?\\s*([0-9.]+)\\s*[Nn]·?[Mm]"
      required: false
      data_type: "number"
      unit: "N·m"
    - name: "行程时间"
      pattern: "行程时间[:：]?\\s*([0-9.]+)\\s*[秒sS]"
      required: false
      data_type: "number"
      unit: "s"
    - name: "控制信号"
      pattern: "控制[:：]?\\s*([0-9]+-[0-9]+\\s*[Vv]|开关量)"
      required: false
      data_type: "string"
      unit: null
```
