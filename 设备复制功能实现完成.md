# 设备复制功能实现完成报告

## 执行时间
2026-03-05

## 功能需求

在设备库管理页面中，单条添加设备比较慢，同类型仅有部分参数修改的设备，为了加快添加设备的效率，需要实现设备复制功能。

## 实现方案

### 功能描述

在设备列表的操作列添加"复制"按钮，点击后：
1. 复制当前设备的所有信息
2. 清空设备ID（让系统自动生成新ID）
3. 设备名称添加"(副本)"后缀
4. 规格型号添加"-COPY"后缀
5. 保留所有其他字段（品牌、设备类型、关键参数、详细参数、单价等）
6. 打开设备表单，用户可以修改需要更改的字段
7. 提交后创建新设备并自动生成规则

```javascript
const initForm = () => {
  if (props.deviceData) {
    // 判断是编辑还是复制（复制时device_id为空）
    isEdit.value = !!props.deviceData.device_id
    
    Object.assign(formData, {
      device_id: props.deviceData.device_id || '',
      brand: props.deviceData.brand,
      device_type: props.deviceData.device_type || '',
      device_name: props.deviceData.device_name,
      spec_model: props.deviceData.spec_model,
      key_params: props.deviceData.key_params || {},
      detailed_params: props.deviceData.detailed_params || '',
      unit_price: props.deviceData.unit_price,
      regenerate_rule: false,
      auto_generate_rule: !isEdit.value // 复制时默认生成规则
    })
    
    // 如果是复制（device_id为空），生成新的device_id
    if (!isEdit.value && formData.spec_model) {
      formData.device_id = generateDeviceId()
    }
    
    // 如果有device_type但没有key_params，初始化参数结构
    if (formData.device_type && Object.keys(formData.key_params).length === 0) {
      onDeviceTypeChange(formData.device_type)
    }
  } else {
    // 新增设备
    isEdit.value = false
    Object.assign(formData, {
      device_id: '',
      brand: '',
      device_type: '',
      device_name: '',
      spec_model: '',
      key_params: {},
      detailed_params: '',
      unit_price: 0,
      input_method: 'manual',
      auto_generate_rule: true
    })
  }
  
  // 清除验证
  if (formRef.value) {
    formRef.value.clearValidate()
  }
}
```

## 使用流程

### 1. 复制设备

1. 在设备库管理页面，找到要复制的设备
2. 点击该设备行的"复制"按钮（橙色按钮）
3. 系统自动打开设备表单对话框

### 2. 修改参数

表单中会预填充原设备的所有信息，但有以下变化：
- 设备ID：自动生成新ID
- 设备名称：原名称 + " (副本)"
- 规格型号：原型号 + "-COPY"
- 其他字段：保持原值

用户可以修改任何需要更改的字段，例如：
- 修改设备名称（去掉"(副本)"后缀）
- 修改规格型号（去掉"-COPY"后缀）
- 修改关键参数（如通径、通数等）
- 修改单价

### 3. 提交创建

1. 修改完成后，点击"创建"按钮
2. 系统创建新设备
3. 自动生成匹配规则（默认勾选）
4. 设备列表自动刷新，显示新设备

## 技术细节

### 1. 设备ID生成

```javascript
const generateDeviceId = () => {
  const specModel = formData.spec_model || ''
  const brand = formData.brand || ''
  const timestamp = Date.now().toString()
  const random = Math.random().toString(36).substring(2, 8).toUpperCase()
  
  // 基于品牌+规格型号+时间戳+随机数生成30位ID
  let base = `${brand}_${specModel}_${timestamp}_${random}`
  // 移除特殊字符，只保留字母数字和下划线
  base = base.replace(/[^a-zA-Z0-9]/g, '_')
  
  // 确保长度为30位
  if (base.length > 30) {
    base = base.substring(0, 30)
  } else if (base.length < 30) {
    base = base.padEnd(30, '0')
  }
  
  return base
}
```

### 2. 深拷贝key_params

```javascript
key_params: device.key_params ? JSON.parse(JSON.stringify(device.key_params)) : null
```

使用`JSON.parse(JSON.stringify())`确保key_params是深拷贝，避免修改副本时影响原设备。

### 3. 判断编辑还是复制

```javascript
isEdit.value = !!props.deviceData.device_id
```

通过检查device_id是否存在来判断：
- 有device_id：编辑模式
- 无device_id：新增模式（包括复制）

## 优点

1. **快速录入**：复制后只需修改少量字段
2. **减少错误**：避免重复输入相同信息时的错误
3. **保持一致性**：确保同系列设备的配置一致
4. **提高效率**：特别适合批量录入同类型设备

## 注意事项

1. **设备名称和型号**
   - 系统会自动添加"(副本)"和"-COPY"后缀
   - 建议用户修改这些后缀，使用更有意义的名称

2. **设备ID**
   - 系统会自动生成新的设备ID
   - 确保每个设备都有唯一的ID

3. **规则生成**
   - 复制的设备默认会自动生成规则
   - 如果不需要规则，可以取消勾选

4. **关键参数**
   - key_params会被完整复制
   - 确保修改需要更改的参数
