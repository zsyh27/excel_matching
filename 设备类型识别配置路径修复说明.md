# 设备类型识别配置路径修复说明

## 问题根源（最终确认）

经过深入调查，发现了真正的问题：**配置路径不匹配**！

### 问题分析

1. **IntelligentExtractionAPI 使用的配置路径**：
   ```python
   extraction_config = config.get('extraction_rules', {})
   device_type_config = extraction_config.get('device_type', {})
   # 完整路径: config['extraction_rules']['device_type']
   ```

2. **实际配置存储的路径**：
   ```python
   ie_config = config.get('intelligent_extraction', {})
   device_type_config = ie_config.get('device_type_recognition', {})
   # 完整路径: config['intelligent_extraction']['device_type_recognition']
   ```

3. **结果**：
   - ❌ `extraction_rules` 配置键在数据库中**根本不存在**
   - ❌ `DeviceTypeRecognizer` 初始化时得到的是**空配置** `{}`
   - ❌ 无论添加多少设备类型，识别器都无法识别，因为它的配置是空的

### 为什么之前的修复没有效果

之前我修复了配置保存后重新加载 `intelligent_extraction_api` 的问题，但由于配置路径错误，即使重新加载，识别器仍然得到的是空配置。
