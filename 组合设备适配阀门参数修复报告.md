# 组合设备"适配阀门"参数修复报告

## 问题描述

用户发现组合设备类型的参数数量不正确：

| 设备类型 | 期望参数数量 | 实际参数数量 | 差异 |
|---------|------------|------------|-----|
| 蝶阀 | 7 | 7 | ✅ 正确 |
| 开关型执行器 | 8 | 8 | ✅ 正确 |
| 调节型执行器 | 9 | 9 | ✅ 正确 |
| **蝶阀+开关型执行器** | **15 (7+8)** | **14** | ❌ 缺少1个 |
| **蝶阀+调节型执行器** | **16 (7+9)** | **15** | ❌ 缺少1个 |

## 根本原因

通过分析发现，缺失的参数是**"适配阀门"**：

1. **蝶阀**本身没有"适配阀门"参数（合理，因为蝶阀本身就是阀门）
2. **开关型执行器**有"适配阀门"参数（用于说明该执行器适配哪些阀门）
3. **调节型执行器**有"适配阀门"参数（用于说明该执行器适配哪些阀门）
4. **组合设备**在创建时，只复制了蝶阀的7个参数和执行器的前7/8个参数，遗漏了最后的"适配阀门"参数

### 为什么会遗漏？

可能的原因：
- 在批量导入组合设备时，某些设备的"适配阀门"字段为空
- 配置生成脚本在合并参数时，过滤掉了空值参数
- 导致"适配阀门"参数没有被添加到组合设备类型的参数配置中

## 修复方案

为组合设备类型添加"适配阀门"参数，配置为非必填项（允许为空）：

```python
{
    'name': '适配阀门',
    'type': 'string',
    'required': False,  # 非必填，允许为空
    'options': ['蝶阀/风阀', '球阀', '座阀', '其他']
}
```
