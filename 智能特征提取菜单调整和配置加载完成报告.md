### 3. 技术实现细节

#### 3.1 配置加载流程

```javascript
const loadConfig = async () => {
  try {
    // 1. 从API获取完整配置
    const response = await fetch('/api/config')
    const data = await response.json()
    
    // 2. 提取智能提取配置
    if (data.success && data.config) {
      const ieConfig = data.config.intelligent_extraction || {}
      const specificConfig = ieConfig.device_type_recognition || {}
      
      // 3. 更新本地配置
      config.value = specificConfig
      lastUpdated.value = data.config.updated_at || null
    }
  } catch (error) {
    ElMessage.error('加载配置失败: ' + error.message)
  }
}
```

#### 3.2 配置保存流程

```javascript
const saveConfig = async () => {
  saving.value = true
  try {
    // 1. 获取当前完整配置
    const response = await fetch('/api/config')
    const data = await response.json()
    
    // 2. 更新智能提取配置
    const fullConfig = data.config
    if (!fullConfig.intelligent_extraction) {
      fullConfig.intelligent_extraction = {}
    }
    fullConfig.intelligent_extraction.device_type_recognition = config.value
    
    // 3. 保存配置
    const saveResponse = await fetch('/api/config/save', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        config: fullConfig,
        remark: '更新设备类型模式配置'
      })
    })
    
    // 4. 处理响应
    const saveData = await saveResponse.json()
    if (saveData.success) {
      ElMessage.success('配置保存成功')
      lastUpdated.value = new Date().toISOString()
    }
  } catch (error) {
    ElMessage.error('保存配置失败: ' + error.message)
  } finally {
    saving.value = false
  }
}
```

#### 3.3 组件生命周期

所有编辑器都在 `onMounted` 钩子中自动加载配置：

```javascript
onMounted(() => {
  loadConfig()
})
```

## 配置管理流程

### 完整工作流

```
1. 用户打开配置管理页面
   ↓
2. 选择"智能特征提取"阶段
   ↓
3. 选择具体编辑器（如"设备类型模式"）
   ↓
4. 编辑器自动从API加载配置
   ↓
5. 用户修改配置
   ↓
6. 点击"保存配置"按钮
   ↓
7. 编辑器获取完整配置
   ↓
8. 更新对应的配置部分
   ↓
9. 调用API保存配置
   ↓
10. 显示保存成功消息
```

### API端点

| 端点 | 方法 | 用途 |

| `/api/config` | GET | 获取完整配置 |
| `/api/config/save` | POST | 保存配置（支持历史记录） |
